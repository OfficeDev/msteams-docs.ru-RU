# <a name="support-and-feedback"></a>Поддержка и обратная связь

## <a name="product-support-and-service-issues"></a>Проблемы с поддержкой продуктов и службами

Используйте **Центр администрирования Azure** **или Microsoft 365 Admin Для** решения критически важных  для бизнеса проблем, которые требуют поддержки на основе соглашения об уровне обслуживания.

| **Поддержка продукта** | **Контакт** |
|:------------|:------------|
| **Microsoft Teams проблемы и сбои продукта** | Если у вас есть контракт на поддержку Premier :::image type="icon" source="assets/icons/microsoft-icon.png" border="false"::: для Microsoft 365, посетите центр Microsoft 365 Admin и откройте меню поддержки, чтобы [открыть запрос на обслуживание](https://admin.microsoft.com/). |
| **Справка и поддержка Azure** | Если у вас есть платная подписка Azure, посетите Центр администрирования Azure, [чтобы открыть запрос](https://ms.portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportreq). |
| **Общие вопросы о Microsoft Teams** | Если у вас есть общие вопросы по Microsoft Teams, отправьте свои [вопросы на Microsoft Teams Community](https://answers.microsoft.com/msteams/forum):::image type="icon" source="assets/icons/microsoft-icon.png" border="false":::.|

## <a name="developer-community-help"></a>Справка сообщества разработчиков

Microsoft Teams разработчиков используют Stack Overflow и Microsoft Q&A для взаимодействия с другими разработчиками для идею, получения уточнений и отправки запросов. Кроме того, можно использовать другие контакты или сайты в зависимости от типа справки сообщества, необходимой для возникновения проблем, ограничений и общих вопросов.

### <a name="developer-community-forums"></a>Форумы сообщества разработчиков

Опубликуйте свои вопросы и помогите другим участникам сообщества, делиться Microsoft Teams и отвечать на них.

| **Community канала**|   **Контакт**  |
|:---------------------|:---------------|
|**Community справки** | <ul> <li>:::image type="icon" source="assets/icons/stack-overflow-icon.png" border="false":::[Stack Overflow](https://stackoverflow.com/questions/tagged/microsoft-teams): используйте тег для `microsoft-teams` публикации вопросов. Следуйте рекомендациям Stack Overflow, таким как указание описательного заголовка, краткое заявление о проблеме и подробные сведения для воспроизведения проблемы. Запросы на функции или общие вопросы не являются темами. Если вы новый пользователь, см. раздел ["Центр справки Stack Overflow"](https://stackoverflow.com/help).</li>  <li>:::image type="icon" source="assets/icons/microsoft-icon.png" border="false":::[Microsoft Q&A](/answers/topics/office-teams-app-dev.html). Мы ценим Stack Overflow и предоставляем расширенную поддержку вопросов, опубликованных на сайте Stack Overflow. Используйте тег `office-teams-app-dev` для отправки вопросов. Stack Overflow имеет критерии для соответствующих вопросов [, а Microsoft Q&A](/answers/topics/office-teams-app-dev.html) имеет открытую политику по этому поводу. </li><ul> |
| **Community чата** | Используйте неформальную комнату чата сообщества [для Microsoft Teams](https://gitter.im/OfficeDev/MicrosoftTeamsAppDev):::image type="icon" source="assets/icons/Teams-icon.png" border="false"::: приложений. Если **вход выполняется** в первый раз, выберите "ВОЙТИ, чтобы начать говорить". Выберите **JOIN ROOM**, если вы уже вошли в систему. |

### <a name="report-issues"></a>Сообщить о проблеме

Отправьте проблемы и задавайте общие вопросы, связанные с пакетом SDK и примерами, с помощью каналов справки сообщества.

| **Community канала** | **Контакт** |
|:----------------------|:------------|
| **Сбои платформы** | Проверьте, не вызвана ли проблема при [Teams](https://aka.ms/TeamsPlatform/Outages) сбоях платформы, прежде чем вызывать ее. |
| **Примеры** | Отправка проблем с примерами в соответствующий [репозиторий Microsoft Teams samples](https://github.com/OfficeDev/Microsoft-Teams-Samples).|
|  **Teams набор средств или проблемы с пакетом SDK для TeamsFx и интерфейсом командной строки** | <ul><li> :::image type="icon" source="assets/icons/GitHub-icon.png" border="false":::  [GitHub](https://github.com/OfficeDev/TeamsFx/issues) проблемы. Создайте новые проблемы в репозитории [TeamsFx (Microsoft Teams Framework)](https://github.com/OfficeDev/TeamsFx) GitHub, чтобы сообщить о проблемах или вызвать запрос на функцию. Мы рекомендуем использовать GitHub для запросов и получения помощи от сообщества. <li> :::image type="icon" source="assets/icons/stack-overflow-icon.png" border="false":::[Stack Overflow](https://stackoverflow.com/questions/tagged/teams-toolkit): используйте тег для `teams-toolkit` публикации вопросов. Следуйте рекомендациям Stack Overflow, таким как указание описательного заголовка, краткое заявление о проблеме и подробные сведения для воспроизведения проблемы. Запросы на функции или общие вопросы не являются темами. </li> </ul> |
| **Проблемы с пакетом SDK для расширения ботов и сообщений или предложения** | Отправка проблем и запросов функций в репозиторий пакета SDK для языка бота ([C#](https://github.com/Microsoft/botbuilder-dotnet/), [JavaScript](https://github.com/Microsoft/botbuilder-js) или [Python](https://github.com/Microsoft/botbuilder-python)). Опубликуйте инструкции на [сайте Stack Overflow](https://stackoverflow.com/questions/tagged/botframework%20microsoft-teams) с помощью тега `botframework` и тега `microsoft-teams` . |
| **Проблемы с пакетом SDK для вкладок или предложения** | Отправка проблем и запросов функций в репозиторий пакета SDK [для Microsoft Teams JavaScript](https://github.com/OfficeDev/microsoft-teams-library-js/issues). Используйте тег `microsoft-teams` для публикации вопросов о пакете SDK в [Stack Overflow](https://stackoverflow.com/questions/tagged/microsoft-teams). |
