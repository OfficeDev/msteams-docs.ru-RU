# <a name="build-your-first-messaging-extension-for-microsoft-teams"></a>Создайте свое первое расширение обмена сообщениями для Microsoft Teams

Существует два типа Teams *обмена сообщениями: команды* поиска [и](../messaging-extensions/how-to/search-commands/define-search-command.md) [команды действий.](../messaging-extensions/how-to/action-commands/define-action-command.md)

Этот учебник научит вас создавать команду поиска *(также известную* как расширение обмена сообщениями на *основе поиска),* которая является ярлыком для поиска внешнего контента и обмена им в Teams. Пользователи могут получить доступ к командам поиска из Teams составить или командовать окном.

## <a name="what-youll-learn"></a>Чему вы научитесь

* Создайте проект приложения и бот расширения обмена сообщениями, используя Microsoft Teams набор средств для Visual Studio Code.
* Поймите конфигурации приложений и леса, имеющие отношение к расширениям обмена сообщениями.
* Хост приложения локально.
* Настройте бота для расширения обмена сообщениями.
* Sideload и протестировать расширение обмена сообщениями в Teams.

## <a name="prerequisites"></a>Предварительные требования

Убедитесь, что вы понимаете, как настроить и построить простой Teams приложение. Для получения дополнительной [информации, см Microsoft Teams.](../build-your-first-app/build-and-run.md)

## <a name="1-create-your-app-project"></a>1. Создание проекта приложения

Система Microsoft Teams набор средств позволяет настроить следующие компоненты для расширения обмена сообщениями:

* **Конфигурации приложений и леса, имеющие** отношение к расширениям обмена сообщениями.
* **Бот** для расширения обмена сообщениями, который автоматически регистрируется в Microsoft Azure Bot Service.

**Создание проекта приложения**

1. В Visual Studio Code выберите **Microsoft Teams левой** :::image type="icon" source="../assets/icons/vsc-toolkit.png"::: стороне активности и **выберите Создать новое Teams приложение.**
1. По запросу вопишитесь на свой аккаунт Microsoft 365 разработки.
1. На **экране проекта Select,** в **поиске расширений**  >  **сообщений нажмите** **JS (JavaScript).** 
1. Введите имя для вашего Teams приложения. Это имя по умолчанию для вашего приложения, а также название каталога проекта приложения на локальной машине.
1. Выберите **Создать новый Бот затем нажмите** **Создать Bot Регистрация кнопку.** В случае успеха, ваш новый бот будет иметь статус *Зарегистрированный.* Теперь ваш бот автоматически регистрируется в Microsoft Azure Bot. 
1. **Выберите** отделку в нижней части экрана, чтобы настроить проект и сохранить проект на локальной машине.

## <a name="2-understand-your-app-project-components"></a>2. Понимание компонентов проекта приложения

Большая часть конфигураций приложений и строительных лесов настроены автоматически при создании проекта с помощью Teams набор средств.

* Конфигурации приложений: Для просмотра или обновления конфигураций расширения обмена сообщениями выберите **App Studio в** наборе инструментов и перейдите к **расширениям обмена сообщениями.**
* Строительные леса приложения: строительные леса приложения предоставляют `botActivityHandler.js` файл, расположенный в корневом каталоге вашего проекта, для обработки того, как расширение обмена сообщениями (или технически, [бот расширения обмена сообщениями)](#4-configure-the-bot-for-your-messaging-extension)реагирует на поисковые запросы в Teams.

## <a name="3-set-up-a-secure-tunnel-to-your-app"></a>3. Настройка безопасного туннеля для приложения

Для целей тестирования давайте разохозим расширение обмена сообщениями на локальном веб-сервере (порт 3978). Вы собираетесь использовать [ngrok для](https://ngrok.com/download) настройки безопасного туннеля для localhost. Для [получения подробной информации можно Microsoft Teams первый](../build-your-first-app/build-bot.md#3-securely-expose-your-localhost-to-the-internet) бот для получения подробной информации. 

1. Если вы еще не сделали этого, [установите ngrok](https://ngrok.com/download).
1. В терминале, запустить `ngrok http -host-header=rewrite 3978` .
1. Копирование URL-адреса HTTPS в выходе (например, `https://468b9ab725e9.ngrok.io` ) с тех пор, как Teams требует подключения HTTPS.

   С помощью этого URL Teams (который требует подключения HTTPS) сможет туннель, где вы хостинг вашего приложения `localhost` (на порт 3978).

## <a name="4-configure-the-bot-for-your-messaging-extension"></a>4. Настройте бота для расширения обмена сообщениями

Расширения обмена сообщениями полагаются на ботов для отправки и обработки запросов пользователей Teams в ваш хостинг-сервис. Бот должен быть зарегистрирован в службе Azure Bot, что было сделано при настройке приложения с помощью Teams набор средств.

Вы все еще должны указать URL-адрес конечной точки бота для получения и обработки поисковых запросов в расширении обмена сообщениями. Как правило, URL выглядит `https://HOST_URL/api/messages` как . Вы можете настроить это быстро в наборе инструментов.

1. В Visual Studio Code выберите **Microsoft Teams левой** :::image type="icon" source="../assets/icons/vsc-toolkit.png"::: стойке активности и выберите Open **Microsoft Teams набор средств.**
1. Перейти к **Bots > существующих регистраций ботов и** выбрать бота вы создали во время установки.
1. В поле **адреса конечной точки** Bot введите URL ngrok (например, `https://468b9ab725e9.ngrok.io` ), где вы принимаете бота и придаток `/api/messages` к нему.

   Ваш бот сможет обрабатывать запросы в вашем расширении обмена сообщениями.

## <a name="5-build-and-run-your-app"></a>5. Создание и запуск приложения

Вы настроили URL-адрес для размещения расширения обмена сообщениями и настроили его для обработки поиска. Пришло время, чтобы получить ваше приложение и работает.

1. Откройте терминал и перейдите в корневой каталог проекта приложения.
1. Запуск `npm install` .
1. Запуск `npm start` .

   В случае успеха вы видите следующее сообщение, указывающее на то, что служба расширения обмена сообщениями прослушивает действия в вашей `localhost` службе:

   `Bot/ME service listening at http://localhost:3978`

## <a name="6-sideload-your-messaging-extension-in-teams"></a>6. Sideload расширение обмена сообщениями в Teams

С запуском расширения обмена сообщениями вы можете установить его в Teams.

> [!TIP]
> Если вы еще не перегрузили приложение Teams и не затеяли проблемы, следуйте этим [инструкциям.](../build-your-first-app/build-and-run.md#4-sideload-your-app-in-teams)

1. В Visual Studio Code, выберите **F5** ключ для запуска Teams веб-клиента.
1. В диалоге установки приложения, **выберите Добавить для меня**.

## <a name="7-test-your-messaging-extension"></a>7. Проверьте расширение обмена сообщениями

Узнайте, как расширения обмена сообщениями работают в Teams чате.

1. Запустите новый чат. В поле compose выберите **More и выберите** :::image type="icon" source="../assets/icons/teams-client-more.png"::: приложение для расширения обмена сообщениями, которое вы только что перезагрузили.
1. Попробуйте найти что-то (например, **билеты).** Если ваше приложение работает, вы увидите результаты поиска образцов (вы можете добавить свой собственный позже).

   :::image type="content" source="../assets/images/build-your-first-app/me-teams-test.png" alt-text="Скриншот, показывающий, как в приложении используется расширение обмена сообщениями на основе поиска Teams составить поле.":::

## <a name="troubleshooting"></a>Устранение неполадок

Следующая информация может помочь, если у вас были проблемы с завершением этого учебника.

**Бот не подключен к Teams**

Если вы установили приложение, но оно не работает, убедитесь, что бот расширения обмена [сообщениями подключен к каналу службы Azure Bot *Teams.*](/azure/bot-service/channel-connect-teams?view=azure-bot-service-4.0&preserve-view=true)

Важно понимать, что это не то же самое, что канал в Teams. В этом случае канал — это способ подключения службы Azure Bot к Teams или другому [поддерживаемому приложению microsoft или сторонних коммуникаций.](/azure/bot-service/bot-service-channels-reference?view=azure-bot-service-4.0&preserve-view=true)
