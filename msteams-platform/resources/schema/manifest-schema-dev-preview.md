# <a name="developer-preview-manifest-schema-for-microsoft-teams"></a>Схема манифеста предварительной версии для разработчиков Microsoft Teams

> [!NOTE]
> Сведения о программе и способе присоединения можно найти в разделе [Preview Developer](~/resources/dev-preview/developer-preview-intro.md) .
> Если вы не используете предварительный просмотр разработчика, не следует использовать эту версию манифеста. Ознакомьтесь со статьей [reference: schema manifest для Microsoft Teams](~/resources/schema/manifest-schema.md) для общедоступной версии манифеста.

Манифест Microsoft Teams описывает, как приложение интегрируется в продукт Microsoft Teams. Манифест должен соответствовать схеме, размещенной в [`https://raw.githubusercontent.com/OfficeDev/microsoft-teams-app-schema/preview/DevPreview/MicrosoftTeams.schema.json`](https://raw.githubusercontent.com/OfficeDev/microsoft-teams-app-schema/preview/DevPreview/MicrosoftTeams.schema.json).

Дополнительные сведения о доступных функциях: [функции в общедоступной предварительной версии для разработчиков Microsoft Teams](~/resources/dev-preview/developer-preview-features.md).

## <a name="schema"></a>$schema

*Необязательная, но рекомендуемая* &ndash; строка

URL-адрес https://, ссылающийся на схему JSON для манифеста.

## <a name="manifestversion"></a>Версия манифеста

**Обязательная** &ndash; строка

Версия схемы манифеста, используемая манифестом. Он должен иметь слово "Девпревиев".

## <a name="version"></a>version

**Обязательная** &ndash; строка

Версия конкретного приложения. Если вы обновите какой-либо элемент в манифесте, необходимо также увеличить номер версии. Благодаря этому при установке нового манифеста будет переписан существующий, а пользователь получит новые функции. Если это приложение было отправлено в хранилище, новый манифест необходимо будет повторно отправить и проверить повторно. После этого пользователи этого приложения автоматически получат новый обновленный манифест в течение нескольких часов после его утверждения.

Если приложение запросило изменение разрешений, пользователям будет предложено выполнить обновление и повторное согласие на доступ к приложению.

Эта строка версии должна соответствовать стандарту [semver](http://semver.org/) (Major. Значительные. PATCH).

## <a name="id"></a>id

**Обязательный** &ndash; идентификатор приложения Microsoft

Уникальный идентификатор, созданный корпорацией Майкрософт для этого приложения. Если вы зарегистрировали Bot с помощью Microsoft Bot Framework, или веб-приложение вашей вкладки уже входит в Microsoft, у вас уже должен быть идентификатор, и его следует ввести здесь. В противном случае необходимо создать новый идентификатор на портале регистрации приложений ([Мои приложения](https://apps.dev.microsoft.com)) Майкрософт, ввести его здесь, а затем повторно использовать его при [добавлении ленты](~/bots/how-to/create-a-bot-for-teams.md).

## <a name="packagename"></a>паккаженаме

**Обязательная** &ndash; строка

Уникальный идентификатор для этого приложения в обратной нотации домена; Пример: com. example. MyApp.

## <a name="developer"></a>developer

**Required**

Указывает сведения о компании. Для приложений, отправляемых в AppSource (ранее — магазин Office), эти значения должны быть сопоставлены со сведениями в записи AppSource.

|Имя| Максимальный размер | Обязательный | Описание|
|---|---|---|---|
|`name`|32 символов|✔|Отображаемое имя разработчика.|
|`websiteUrl`|2048 символов|✔|URL-адрес https://на веб-сайт разработчика. Эта ссылка должна принять участие у пользователей в вашей компании или целевой странице для конкретного продукта.|
|`privacyUrl`|2048 символов|✔|URL-адрес https://для политики конфиденциальности разработчика.|
|`termsOfUseUrl`|2048 символов|✔|URL-адрес https://для условий использования разработчиком.|
|`mpnId`|10 символов|✔|**Необязательное требование** Идентификатор сети партнера Майкрософт, определяющий партнерскую организацию, которая создает приложение.|

## <a name="localizationinfo"></a>локализатионинфо

**Необязательное**

Разрешает спецификацию языка по умолчанию, а также указателей на дополнительные языковые файлы. См.: [Локализация](~/concepts/build-and-test/apps-localization.md).

|Имя| Максимальный размер | Обязательный | Описание|
|---|---|---|---|
|`defaultLanguageTag`|4 символа|✔|Тег языка строк в этом файле манифеста верхнего уровня.|

### <a name="localizationinfoadditionallanguages"></a>Локализатионинфо. Аддитионаллангуажес

Массив объектов, задающий дополнительные языковые переводы.

|Имя| Максимальный размер | Обязательный | Описание|
|---|---|---|---|
|`languageTag`|4 символа|✔|Тег языка строк в указанном файле.|
|`file`|4 символа|✔|Относительный путь к файлу. JSON, содержащему переведенные строки.|

## <a name="name"></a>name

**Required**

Имя приложения, отображаемое для пользователей в интерфейсе Teams. Для приложений, отправляемых в AppSource, эти значения должны быть соответствующими сведениям в записи AppSource. Значения `short` и `full` не должны совпадать.

|Имя| Максимальный размер | Обязательный | Описание|
|---|---|---|---|
|`short`|30 символов|✔|Краткое отображаемое имя приложения.|
|`full`|100 символов||Полное имя приложения, используемое, если имя полного приложения превышает 30 символов.|

## <a name="description"></a>description

**Required**

Описывает приложение для пользователей. Для приложений, отправляемых в AppSource, эти значения должны быть соответствующими сведениям в записи AppSource.

Убедитесь, что ваше описание точно описывает ваш интерфейс и предоставляет сведения, помогающие потенциальным клиентам определить, что делает ваш интерфейс. Кроме того, в полном описании следует отметить, если для использования требуется внешняя учетная запись. Значения `short` и `full` не должны совпадать.  Краткое описание не должно повторяться в длинном описании и не должно включать имя другого приложения.

|Имя| Максимальный размер | Обязательный | Описание|
|---|---|---|---|
|`short`|80 символов|✔|Краткое описание интерфейса приложения, используемое при ограниченном пространстве.|
|`full`|4000 символов|✔|Полное описание приложения.|

## <a name="icons"></a>icons

**Required**

Значки, используемые в приложении Teams. Файлы значков должны быть включены в состав пакета отправки.

|Имя| Максимальный размер | Обязательный | Описание|
|---|---|---|---|
|`outline`|2048 символов|✔|Относительный путь к файлу с прозрачным значком изображения в формате 32x32 PNG.|
|`color`|2048 символов|✔|Относительный путь к файлу полного цветового значка PNG 192x192.|

## <a name="accentcolor"></a>акцентколор

**Обязательная** &ndash; строка

Цвет, используемый в сочетании с фоном для значков структуры.

Значение должно быть допустимым цветом HTML-кода, начинающимся с "#", `#4464ee`например.

## <a name="configurabletabs"></a>конфигураблетабс

**Необязательное**

Используется, когда у вашего приложения есть вкладка "канал группы", требующая дополнительной настройки перед добавлением. Настраиваемые вкладки поддерживаются только в области Teams, и в настоящее время поддерживается только одна вкладка для каждого приложения.

Объект является массивом со всеми элементами типа `object`. Этот блок необходим только для решений, которые предоставляют настраиваемое решение для вкладки канала.

|Имя| Тип| Максимальный размер | Обязательный | Описание|
|---|---|---|---|---|
|`configurationUrl`|String|2048 символов|✔|URL-адрес https://, используемый при настройке вкладки.|
|`canUpdateConfiguration`|Boolean|||Значение, указывающее, может ли пользователь обновлять экземпляр конфигурации вкладки после создания. Умолчани`true`|
|`scopes`|Массив перечисления|1 |✔|В настоящее время настраиваемые вкладки поддерживают только области `team` и `groupchat` области. |
|`sharePointPreviewImage`|String|2048||Относительный путь к изображению вкладки для использования в SharePoint. Размер 1024x768. |
|`supportedSharePointHosts`|Массив перечисления|1 ||Определяет, как будет предоставляться вкладка в SharePoint. `sharePointFullPage` Параметры`sharePointWebPart` |

## <a name="statictabs"></a>статиктабс

**Необязательное**

Определяет набор вкладок, которые по умолчанию можно закреплять, не добавляя пользователей вручную. Статические вкладки, `personal` объявляемые в области, всегда фиксируются в личном интерфейсе приложения. Статические вкладки, объявленные в `team` области, в настоящее время не поддерживаются.

Объект является массивом (не более 16 элементов) со всеми элементами типа `object`. Этот блок необходим только для решений, предоставляющих статическое решение для вкладок.

|Имя| Тип| Максимальный размер | Обязательный | Описание|
|---|---|---|---|---|
|`entityId`|String|64 символов|✔|Уникальный идентификатор объекта, отображаемого на вкладке.|
|`name`|String|128 символов|✔|Отображаемое имя вкладки в интерфейсе канала.|
|`contentUrl`|String|2048 символов|✔|URL-адрес https://, указывающий на пользовательский интерфейс сущности, который будет отображаться в полотне Teams.|
|`websiteUrl`|String|2048 символов||URL-адрес https://, указывающий, когда пользователь попытается просмотреть его в браузере.|
|`scopes`|Массив перечисления|1 |✔|В настоящее время статические вкладки поддерживают `personal` только область, что означает, что ее можно подготовить только в рамках личного интерфейса.|

## <a name="bots"></a>Боты

**Необязательное**

Определяет одноэлементное решение, а также дополнительные сведения, такие как свойства команды по умолчанию.

Объект является массивом (в настоящее время для каждого приложения&mdash;допускается только один элемент "bot" для каждого приложения) со всеми элементами типа `object`. Этот блок необходим только для решений, предоставляющих интерфейс Bot.

|Имя| Тип| Максимальный размер | Обязательный | Описание|
|---|---|---|---|---|
|`botId`|String|64 символов|✔|Уникальный идентификатор приложения для ленты, зарегистрированный с помощью Bot Framework. Это может быть так же, как и общий [идентификатор приложения](#id).|
|`needsChannelSelector`|Boolean|||Описывает, использует ли пользователь почтовые подсказка для добавления ленты к определенному каналу. Умолчани`false`|
|`isNotificationOnly`|Boolean|||Указывает, является ли Bot односторонним, только для уведомления, а не с сообщением Bot. Умолчани`false`|
|`supportsFiles`|Boolean|||Указывает, поддерживает ли Bot возможность загрузки и скачивания файлов в личном чате. Умолчани`false`|
|`scopes`|Массив перечисления|3 |✔|Указывает `team`, является ли Bot интерфейсом пользователя в контексте канала в, в разделе "участие в группах" (`groupchat`) или взаимодействия с областью действия только для отдельного пользователя (`personal`). Эти параметры не являются монопольными.|

### <a name="botscommandlists"></a>Боты. Коммандлистс

Необязательный список команд, которые ваш робот может рекомендовать пользователям. Объект является массивом (не более 2 элементов) со всеми элементами типа `object`; необходимо определить отдельный список команд для каждой области, которую поддерживает Bot. Дополнительные сведения содержатся в [меню Bot](~/bots/how-to/create-a-bot-commands-menu.md) .

|Имя| Тип| Максимальный размер | Обязательный | Описание|
|---|---|---|---|---|
|`items.scopes`|массив перечисления|3 |✔|Задает область действия, для которой список команд является допустимым. Параметры: `team`, `personal`, и `groupchat`.|
|`items.commands`|Массив объектов|10 |✔|Массив команд, поддерживаемых в Bot:<br>`title`: имя команды Bot (строка, 32)<br>`description`: простое описание или пример синтаксиса команды и его аргумента (строка, 128).|

Объект является массивом (не более 1 элемента) со всеми элементами типа `object`. Этот блок необходим только для решений, предоставляющих соединитель.

|Имя| Тип| Максимальный размер | Обязательный | Описание|

|`scopes`|Массив перечисления|1 |✔|Указывает `team`, будет ли соединитель работать в контексте канала в или в интерфейсе для отдельного пользователя (`personal`). В настоящее время поддерживается `team` только область действия.|

## <a name="composeextensions"></a>компосикстенсионс

**Необязательное**

Определяет расширение системы обмена сообщениями для приложения.

> [!NOTE]
> Имя компонента изменилось с "расширение составления" на "расширение обмена сообщениями" в ноябре 2017, но имя манифеста остается без изменений, поэтому все существующие расширения продолжают работать.

Объект является массивом (не более 1 элемента) со всеми элементами типа `object`. Этот блок необходим только для решений, предоставляющих расширение системы обмена сообщениями.

|Имя| Тип | Максимальный размер | Обязательный | Описание|

## <a name="permissions"></a>permissions

**Необязательное**

Массив `string` , определяющий, какие разрешения запрашиваются приложением, что позволяет конечным пользователям знать, как будет выполняться расширение. Следующие параметры неисключительны:

* `identity`&emsp; Требуются сведения об удостоверении пользователя
* `messageTeamMembers`&emsp; Требуется разрешение на отправку прямых сообщений участникам группы

Изменение этих разрешений при обновлении приложения приводит к тому, что пользователи будут повторять процесс разрешения при первом запуске обновленного приложения.

## <a name="validdomains"></a>валиддомаинс

**Необязательно**, за исключением **обязательного** места, где отмечено

Список допустимых доменов, из которых приложение ожидает загрузки какого бы то ни было содержимого. Примеры доменов могут включать подстановочные знаки `*.example.com`. Это соответствует только одному сегменту домена; Если вам нужно использовать `a.b.example.com` `*.*.example.com`этот параметр. Если конфигурация вкладки или контентный пользовательский интерфейс должны перейти к любому другому домену, Кроме того, что используется для настройки вкладок, этот домен необходимо указать здесь.

Тем не менее, **не** нужно включать домены поставщиков удостоверений, которые требуется поддерживать в вашем приложении. Например, для проверки подлинности с помощью идентификатора Google необходимо перенаправить на accounts.google.com, но не следует включать accounts.google.com в `validDomains[]`.

> [!IMPORTANT]
> Не добавляйте домены за предельный элемент управления (напрямую или с помощью подстановочных знаков). Например, `yourapp.onmicrosoft.com` является допустимым, но `*.onmicrosoft.com` не является допустимым.

Объект является массивом со всеми элементами типа `string`.
