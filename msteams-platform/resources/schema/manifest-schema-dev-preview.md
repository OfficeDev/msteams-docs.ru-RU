# <a name="developer-preview-manifest-schema-for-microsoft-teams"></a>Схема манифеста предварительного просмотра разработчика для Microsoft Teams

> [!NOTE]
> Смотрите [Предварительный просмотр разработчика](~/resources/dev-preview/developer-preview-intro.md) для получения информации о программе и о том, как вы можете присоединиться.
> Если вы не используете предварительный просмотр разработчика, вы не должны использовать эту версию манифеста. [См. Справку: Microsoft Teams для](~/resources/schema/manifest-schema.md) публичной версии манифеста.

В Microsoft Teams описывается, как приложение интегрируется в Microsoft Teams продукт. Ваш манифест должен соответствовать схеме, размещенным на [`https://raw.githubusercontent.com/OfficeDev/microsoft-teams-app-schema/preview/DevPreview/MicrosoftTeams.schema.json`](https://raw.githubusercontent.com/OfficeDev/microsoft-teams-app-schema/preview/DevPreview/MicrosoftTeams.schema.json) .

Для получения дополнительной информации о доступных функциях [см.: Особенности в публичном предварительном просмотре разработчика для Microsoft Teams](~/resources/dev-preview/developer-preview-features.md).

## <a name="schema"></a>$schema

*Необязательно, но рекомендуется* &ndash; струна

В https:// URL-адрес, ссылаясь на схему JSON для манифеста.

## <a name="manifestversion"></a>манифестВерсия

**Требуется** &ndash; струна

Версия манифеста схема этого манифеста использует. Это должно быть "девПревью".

## <a name="version"></a>version

**Требуется** &ndash; струна

Версия конкретного приложения. Если вы обновите что-то в манифесте, версия также должна быть приращена. Благодаря этому при установке нового манифеста будет переписан существующий, а пользователь получит новые функции. Если это приложение было представлено в магазин, новый манифест должен быть повторно представлен и повторно проверен. Затем пользователи этого приложения получат новый обновленный манифест автоматически в течение нескольких часов, после его утверждения.

Если приложение запросило изменение разрешений, пользователям будет предложено обновить и повторно дать согласие на приложение.

Эта строка версии должна следовать [стандарту semver](http://semver.org/) (MAJOR. несовершеннолетний. ПАТЧ).

## <a name="id"></a>id

**Требуется** &ndash; Идентификатор приложения Майкрософт

Уникальный идентификатор, созданный корпорацией Майкрософт для этого приложения. Если вы зарегистрировали бота через Microsoft Bot Framework, или веб-приложение вашей вкладки уже входит в Microsoft, вы уже должны иметь идентификатор и должны ввести его здесь. В противном случае, вы должны создать новый идентификатор на портале регистрации приложений Майкрософт[(Мои приложения),](https://apps.dev.microsoft.com)введите его здесь, а затем повторно использовать его [при добавляют бота](~/bots/how-to/create-a-bot-for-teams.md).

## <a name="packagename"></a>пакетИмя

**Требуется** &ndash; струна

Уникальный идентификатор для этого приложения в обратных нотациях домена; например, com.example.myapp.

|`privacyUrl`|2048 символов|✔|Веб https:// адрес для политики конфиденциальности разработчика.|
|`termsOfUseUrl`|2048 символов|✔|Данный https:// URL-адресу к условиям использования разработчика.|
|`mpnId`|10 символов|✔|**Необязательно** Идентификатор партнерской сети Майкрософт, который идентифицирует организацию-партнера, строя приложение.|

## <a name="name"></a>name

**Required**

Название приложения, отображаемое пользователям в приложении, Teams опыт. Для приложений, представленных в AppSource, эти значения должны соответствовать информации в вашей записи AppSource. Ценности `short` и `full` не должны быть одинаковыми.

|Имя| Максимальный размер | Обязательный | Описание|
|---|---|---|---|
|`short`|30 символов|✔|Краткое название дисплея для приложения.|
|`full`|100 символов||Полное название приложения, используемое, если полное имя приложения превышает 30 символов.|

## <a name="description"></a>description

**Required**

Описывает ваше приложение для пользователей. Для приложений, представленных в AppSource, эти значения должны соответствовать информации в вашей записи AppSource.

Убедитесь, что ваше описание точно описывает ваш опыт и предоставляет информацию, чтобы помочь потенциальным клиентам понять, что ваш опыт делает. Вы также должны отметить, в полном описании, если внешний счет требуется для использования. Ценности `short` и `full` не должны быть одинаковыми.  Ваше краткое описание не должно повторяться в длинном описании и не должно включать любое другое имя приложения.

|Имя| Максимальный размер | Обязательный | Описание|
|---|---|---|---|
|`short`|80 символов|✔|Краткое описание вашего приложения, используемого при ограничении пространства.|
|`full`|4000 символов|✔|Полное описание вашего приложения.|

Объект является массивом со всеми элементами `object` типа. Этот блок необходим только для решений, которые обеспечивают настраиваемое решение вкладки канала.

|`scopes`|Массив перечислений|1|✔|В настоящее время настраиваемые вкладки поддерживают `team` только `groupchat` области и области. |
|`sharePointPreviewImage`|String|2048||Относительный путь файла к изображению предварительного просмотра вкладок для использования в SharePoint. Размер 1024x768. |
|`supportedSharePointHosts`|Массив перечислений|1||Определяет, как ваша вкладка будет доступна в SharePoint. Варианты `sharePointFullPage` и `sharePointWebPart` |

## <a name="statictabs"></a>статическиеTabs

**Необязательное**

Определяет набор вкладок, которые могут быть "закреплены" по умолчанию, без добавления пользователем их вручную. Статические вкладки, `personal` заявленные в области, всегда прикрепляются к личному опыту приложения. Статические вкладки, заявленные в `team` области, в настоящее время не поддерживаются.

Объект является массивом (максимум 16 элементов) со всеми элементами `object` типа. Этот блок необходим только для решений, которые обеспечивают статическое решение вкладки.

|Имя| Тип| Максимальный размер | Обязательный | Описание|
|---|---|---|---|---|
|`entityId`|String|64 символа|✔|Уникальный идентификатор сущности, отображаемой вкладкой.|
|`name`|String|128 символов|✔|Название отображения вкладки в интерфейсе канала.|
|`contentUrl`|String|2048 символов|✔|В https:// URL-адрес, который указывает на пользовательский интерфейс сущности, который будет отображаться в Teams холсте.|
|`websiteUrl`|String|2048 символов||Веб https:// адрес, чтобы указать на если пользователь выбирает для просмотра в браузере.|
|`scopes`|Массив перечислений|1|✔|В настоящее время статические вкладки поддерживают только `personal` область, что означает, что она может быть предусмотрена только как часть личного опыта.|

Объект массив (максимум только 1 элемент в настоящее &mdash; время только один бот допускается в приложении) со всеми элементами типа `object` . Этот блок необходим только для решений, которые обеспечивают бот опыт.

|Имя| Тип| Максимальный размер | Обязательный | Описание|
|---|---|---|---|---|
|`botId`|String|64 символа|✔|Уникальный идентификатор приложения Майкрософт для бота, зарегистрированный в Bot Framework. Это вполне может быть так же, как общий [идентификатор приложения](#id).|
|`needsChannelSelector`|Логический|||Описывает, использует ли бот пользовательское указание для добавления бота в определенный канал. по умолчанию: `false`|
|`isNotificationOnly`|Логический|||Указывает, является ли бот односторонним и только для уведомлений, в отличие от бота для беседы. по умолчанию: `false`|
|`supportsFiles`|Логический|||Указывает, поддерживает ли бот возможность отправки и скачивания файлов в личном чате. по умолчанию: `false`|
|`scopes`|Массив перечислений|3|✔|Указывает, предлагает ли бот функции в контексте канала в `team`, в групповом чате (`groupchat`) или функции, ограниченные только отдельным пользователем (`personal`). Эти параметры не являются исключающими.|

Для получения дополнительной информации [см.](~/bots/how-to/create-a-bot-commands-menu.md)

|`items.commands`|массив объектов|10 |✔|Массив команд, поддерживаемых ботом:<br>`title`: имя команды бота (строка, 32).<br>`description`: простое описание или пример синтаксиса команды и его аргумента (строка, 128).|

## <a name="composeextensions"></a>составьтеЭкстрасты

**Необязательное**

Определяет расширение обмена сообщениями для приложения.

## <a name="permissions"></a>permissions

**Необязательное**

Массив, в `string` котором указывается, какие разрешения запрашивает приложение, который позволяет конечной пользователям знать, как будет выполняться расширение. Следующие варианты не являются эксклюзивными:

* `identity`&emsp;Требуется идентификационная информация пользователя.
* `messageTeamMembers`&emsp;Требуется разрешение на отправку прямых сообщений членам команды.

Изменение этих разрешений при обновлении приложения приведет к тому, что пользователи впервые повторят процесс согласия при запуске обновленного приложения.

## <a name="validdomains"></a>действительныйДомайнс

**Дополнительно,** за исключением **Требуемые,** где отметил

Список действительных доменов, из которых приложение рассчитывает загрузить любой контент. Списки доменов могут включать, например, подстановочные `*.example.com` знаки. Это соответствует ровно одному сегменту домена; если вам нужно, чтобы `a.b.example.com` соответствовать затем использовать `*.*.example.com` . Если конфигурация вкладок или пользовательский интерфейс содержимого должны перемещаться в любой другой домен, кроме того, который используется для конфигурации вкладок, этот домен должен быть указан здесь.

Однако **нет** необходимости включать в приложение домены поставщиков идентификации, которые вы хотите поддержать. Например, для проверки подлинности с помощью Идентификатора Google необходимо перенаправить на accounts.google.com, но вы не должны включать accounts.google.com в `validDomains[]` .

> [!IMPORTANT]
> Не добавляйте домены, которые находятся вне вашего контроля, ни непосредственно, ни через подстановочные знаки. Например, `yourapp.onmicrosoft.com` действителен, `*.onmicrosoft.com` но не действителен.

Объект является массивом со всеми элементами `string` типа.

## <a name="configurableproperties"></a>настраиваемыеПредложения

**Дополнительно** - массив

Блок `configurableProperties` определяет свойства приложения, которые Teams администратор может настроить. Для получения дополнительной [информации, смотрите настроить приложения в Microsoft Teams](/MicrosoftTeams/customize-apps).

> [!NOTE]
> Необходимо определить минимум одно свойство. Вы можете определить максимум девять свойств в этом блоке.
> В качестве наилучшей практики необходимо предоставить рекомендации по настройке для пользователей и клиентов приложений, чтобы следовать при настройке приложения. 

Вы можете определить любое из следующих свойств:
* `name`: Позволяет администратору изменить имя дисплея приложения.
* `shortDescription`: Позволяет администратору изменить краткое описание приложения.
* `longDescription`: Позволяет администратору изменить подробное описание приложения.
* `smallImageUrl`: Это `outline` свойство в `icons` блоке манифеста.
* `largeImageUrl`: Это `color` свойство в `icons` блоке манифеста.
* `accentColor`: Это цвет для использования в сочетании с и в качестве фона для вашего контура иконки.
* `websiteUrl`: Это https:// URL на веб-сайте разработчика.
* `privacyUrl`: Это https:// URL-адрес политики конфиденциальности разработчика.
* `termsOfUseUrl`: Это https:// URL-адрес к условиям использования разработчика.
