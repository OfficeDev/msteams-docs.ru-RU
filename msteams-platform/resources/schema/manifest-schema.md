# <a name="reference-manifest-schema-for-microsoft-teams"></a>Ссылка: схема манифеста для Microsoft Teams

Манифест Microsoft Teams описывает, как приложение интегрируется в продукт Microsoft Teams. Манифест должен соответствовать схеме, размещенной в [`https://developer.microsoft.com/json-schemas/teams/v1.7/MicrosoftTeams.schema.json`]( https://developer.microsoft.com/json-schemas/teams/v1.7/MicrosoftTeams.schema.json) . Также поддерживаются предыдущие версии 1.0-1.6 (в URL-адресе используется "v1. x").

В приведенном ниже примере схемы показаны все варианты расширения.

Схема определяет следующие свойства:

## <a name="schema"></a>$schema

*Необязательно, но рекомендуется* — строка

URL-адрес https://, ссылающийся на схему JSON для манифеста.

## <a name="manifestversion"></a>Версия манифеста

**Обязательный** — строка

Версия схемы манифеста, используемая манифестом. Он должен иметь значения "1,7".

## <a name="version"></a>version

**Обязательный** — строка

Версия конкретного приложения. Если вы обновите какой-либо элемент в манифесте, необходимо также увеличить номер версии. Благодаря этому при установке нового манифеста будет переписан существующий, а пользователь получит новые функции. Если это приложение было отправлено в хранилище, новый манифест необходимо будет повторно отправить и проверить повторно. После этого пользователи этого приложения автоматически получат новый обновленный манифест в течение нескольких часов после его утверждения.

Если приложение запросило изменение разрешений, пользователям будет предложено выполнить обновление и повторное согласие на доступ к приложению.

Эта строка версии должна соответствовать стандарту [semver](http://semver.org/) (Major. Значительные. PATCH).

## <a name="id"></a>id

**Обязательный** — идентификатор приложения Майкрософт

Уникальный идентификатор, созданный корпорацией Майкрософт для этого приложения. Если вы зарегистрировали Bot с помощью Microsoft Bot Framework, или веб-приложение вашей вкладки уже входит в Microsoft, у вас уже должен быть идентификатор, и его следует ввести здесь. В противном случае необходимо создать новый идентификатор на портале регистрации приложений ([Мои приложения](https://apps.dev.microsoft.com)) Майкрософт, ввести его здесь, а затем повторно использовать его при добавлении ленты. Note: Если вы отправляете обновление существующему приложению в AppSource, идентификатор в манифесте не должен изменяться.

## <a name="developer"></a>developer

**Обязательный** — объект

Указывает сведения о компании. Для приложений, отправляемых в AppSource (ранее — магазин Office), эти значения должны быть сопоставлены со сведениями в записи AppSource. Ознакомьтесь с нашими [рекомендациями по публикации](~/concepts/deploy-and-publish/appsource/prepare/frequently-failed-cases.md) для получения дополнительной информации.

|Имя| Максимальный размер | Обязательный | Описание|
|---|---|---|---|
|`name`|32 символов|✔|Отображаемое имя разработчика.|
|`websiteUrl`|2048 символов|✔|URL-адрес https://на веб-сайт разработчика. Эта ссылка должна принять участие у пользователей в вашей компании или целевой странице для конкретного продукта.|
|`privacyUrl`|2048 символов|✔|URL-адрес https://для политики конфиденциальности разработчика.|
|`termsOfUseUrl`|2048 символов|✔|URL-адрес https://для условий использования разработчиком.|
|`mpnId`|10 символов| |**Необязательное требование** Идентификатор сети партнера Майкрософт, определяющий партнерскую организацию, которая создает приложение.|

## <a name="name"></a>name

**Обязательный** — объект

Имя приложения, отображаемое для пользователей в интерфейсе Teams. Для приложений, отправляемых в AppSource, эти значения должны быть соответствующими сведениям в записи AppSource. Значения `short` и `full` не должны совпадать.

|Имя| Максимальный размер | Обязательный | Описание|
|---|---|---|---|
|`short`|30 символов|✔|Краткое отображаемое имя приложения.|
|`full`|100 символов||Полное имя приложения, используемое, если имя полного приложения превышает 30 символов.|

## <a name="description"></a>description

**Обязательный** — объект

Описывает приложение для пользователей. Для приложений, отправляемых в AppSource, эти значения должны быть соответствующими сведениям в записи AppSource.

Убедитесь, что ваше описание точно описывает ваш интерфейс и предоставляет сведения, помогающие потенциальным клиентам определить, что делает ваш интерфейс. Кроме того, в полном описании следует отметить, если для использования требуется внешняя учетная запись. Значения `short` и `full` не должны совпадать.  Краткое описание не должно повторяться в длинном описании и не должно включать имя другого приложения.

|Имя| Максимальный размер | Обязательный | Описание|
|---|---|---|---|
|`short`|80 символов|✔|Краткое описание интерфейса приложения, используемое при ограниченном пространстве.|
|`full`|4000 символов|✔|Полное описание приложения.|

Значки, используемые в приложении Teams. Файлы значков должны быть включены в состав пакета отправки. Дополнительные сведения можно найти в разделе [значки](~/concepts/build-and-test/apps-package.md#icons) .

## <a name="accentcolor"></a>акцентколор

**Необязательное** — шестнадцатеричный код цвета HTML

Цвет, используемый в сочетании с фоном для значков структуры.

Значение должно быть допустимым цветом HTML-кода, начинающимся с "#", например `#4464ee` .

## <a name="configurabletabs"></a>конфигураблетабс

|Имя| Тип| Максимальный размер | Обязательный | Описание|

|`canUpdateConfiguration`|boolean|||Значение, указывающее, может ли пользователь обновлять экземпляр конфигурации вкладки после создания. Значение по умолчанию: **true**.|
|`sharePointPreviewImage`|Строка|2048||Относительный путь к изображению вкладки для использования в SharePoint. Размер 1024x768. |

## <a name="statictabs"></a>статиктабс

**Необязательный** — массив

Определяет набор вкладок, которые по умолчанию можно закреплять, не добавляя пользователей вручную. Статические вкладки, объявляемые в `personal` области, всегда фиксируются в личном интерфейсе приложения. Статические вкладки, объявленные в `team` области, в настоящее время не поддерживаются.

Этот элемент является массивом (не более 16 элементов) со всеми элементами типа `object` . Этот блок необходим только для решений, предоставляющих статическое решение для вкладок.

|Имя| Тип| Максимальный размер | Обязательный | Описание|
|---|---|---|---|---|
|`entityId`|string|64 символа|✔|Уникальный идентификатор объекта, отображаемого на вкладке.|
|`name`|Строка|128 символов|✔|Отображаемое имя вкладки в интерфейсе канала.|
|`contentUrl`|Строка|2048 символов|✔|URL-адрес https://, указывающий на пользовательский интерфейс сущности, который будет отображаться в полотне Teams.|
|`websiteUrl`|Строка|2048 символов||URL-адрес https://, указывающий, когда пользователь попытается просмотреть его в браузере.|
|`scopes`|массив перечислений|1,1|✔|В настоящее время статические вкладки поддерживают только `personal` область, что означает, что ее можно подготовить только в рамках личного интерфейса.|

> [!NOTE]
> Если на вкладках требуются контекстно зависимые сведения для отображения релевантного контента или для инициализации процесса проверки подлинности, *обратитесь к разделу* [Получение контекста для вкладки Microsoft Teams](../../tabs/how-to/access-teams-context.md).

**Необязательный** — массив

Определяет одноэлементное решение, а также дополнительные сведения, такие как свойства команды по умолчанию.

Элемент является массивом ( &mdash; в настоящее время для каждого приложения допускается только один элемент "bot" для каждого приложения) со всеми элементами типа `object` . Этот блок необходим только для решений, предоставляющих интерфейс Bot.

|Имя| Тип| Максимальный размер | Обязательный | Описание|
|---|---|---|---|---|
|`botId`|string|64 символа|✔|Уникальный идентификатор приложения Майкрософт для бота, зарегистрированный в Bot Framework. Это может быть так же, как и общий [идентификатор приложения](#id).|
|`scopes`|массив перечислений|4|✔|Указывает, предлагает ли бот функции в контексте канала в `team`, в групповом чате (`groupchat`) или функции, ограниченные только отдельным пользователем (`personal`). Эти параметры не являются исключающими.|
|`needsChannelSelector`|boolean|||Описывает, использует ли бот пользовательское указание для добавления бота в определенный канал. Умолчани **`false`**|
|`isNotificationOnly`|boolean|||Указывает, является ли бот односторонним и только для уведомлений, в отличие от бота для беседы. Умолчани `**false**`|
|`supportsFiles`|boolean|||Указывает, поддерживает ли бот возможность отправки и скачивания файлов в личном чате. Умолчани **`false`**|

Каждый элемент команды представляет собой объект со следующей структурой:

|Имя| Тип| Максимальный размер | Обязательный | Описание|

|`title`|Строка|32 символов|✔|Понятное имя команды.|
|`type`|Строка|64 символа||Тип команды. Один из `query` или `action` . Значение по умолчанию: **Query**.|
|`description`|Строка|128 символов||Описание, которое отображается для пользователей, чтобы указать назначение этой команды.|

|`fetchTask`|boolean|||Логическое значение, которое указывает, следует ли выполнять динамическое извлечение модуля задачи. По умолчанию: **false**.|

|`taskInfo.width`|Строка|||Ширина диалогового окна — число в пикселях или макет по умолчанию, например "крупный", "средний" или "Малый".|

|`parameters.description`|Строка|128 символов||Понятная пользователю строка, описывающая назначение этого параметра.|

## <a name="permissions"></a>разрешения

**Необязательное** — массив строк

Массив, `string` определяющий, какие разрешения запрашиваются приложением, что позволяет конечным пользователям знать, как будет выполняться расширение. Следующие параметры неисключительны:

* `identity`&emsp;Требуются сведения об удостоверении пользователя
* `messageTeamMembers`&emsp;Требуется разрешение на отправку прямых сообщений участникам группы

Изменение этих разрешений при обновлении приложения приводит к тому, что пользователи будут повторять процесс разрешения при первом запуске обновленного приложения. Дополнительные сведения см. [в разделе Обновление приложения](~/concepts/deploy-and-publish/appsource/post-publish/overview.md) .

## <a name="validdomains"></a>валиддомаинс

**Необязательно**, за исключением **обязательного** места, где отмечено

Список допустимых доменов для веб-сайтов, которые приложение ожидает загружать в клиенте Teams. Примеры доменов могут включать подстановочные знаки `*.example.com` . Это соответствует только одному сегменту домена; Если вам нужно использовать этот параметр `a.b.example.com` `*.*.example.com` . Если конфигурация вкладки или контентный пользовательский интерфейс должны перейти к любому другому домену, Кроме того, что используется для настройки вкладок, этот домен необходимо указать здесь.

Тем не менее, **не** нужно включать домены поставщиков удостоверений, которые требуется поддерживать в вашем приложении. Например, для проверки подлинности с помощью идентификатора Google необходимо перенаправить на accounts.google.com, но не следует включать accounts.google.com в `validDomains[]` .

Приложения Teams, которым для правильной работы требуются собственные URL-адреса SharePoint, могут содержать "{теамситедомаин}" в допустимом списке доменов.

> [!IMPORTANT]
> Не добавляйте домены за предельный элемент управления (напрямую или с помощью подстановочных знаков). Например, `yourapp.onmicrosoft.com` является допустимым, но `*.onmicrosoft.com` не является допустимым.

Объект является массивом со всеми элементами типа `string` .
