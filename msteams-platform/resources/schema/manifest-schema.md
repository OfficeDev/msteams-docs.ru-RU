# <a name="reference-manifest-schema-for-microsoft-teams"></a>Справка: Схема манифеста для Microsoft Teams

В Teams манифесте описывается, как приложение интегрируется в Microsoft Teams продукт. Ваш манифест должен соответствовать схеме, размещенным на [`https://developer.microsoft.com/json-schemas/teams/v1.10/MicrosoftTeams.schema.json`]( https://developer.microsoft.com/json-schemas/teams/v1.10/MicrosoftTeams.schema.json) . Предыдущие версии 1.0, 1.1,..., 1.6 и так далее также поддерживаются (с помощью "v1.x" в URL).

В следующем образце схемы показаны все параметры разгибаемости:

## <a name="schema"></a>$schema

Необязательно, но рекомендуется - строка

В https:// URL-адрес, ссылаясь на схему JSON для манифеста.

## <a name="manifestversion"></a>манифестВерсия

**Требуется** - строка

Версия манифеста схема этого манифеста использует. Это должно быть 1.10.

## <a name="version"></a>version

**Требуется** - строка

Версия конкретного приложения. Если вы обновите что-то в манифесте, версия должна быть приращена тоже. Таким образом, при установке нового манифеста он перезахохорит существующий и пользователь получит новую функциональность. Если это приложение было представлено в магазин, новый манифест должен быть повторно представлен и повторно проверен. Пользователи приложения получают новый обновленный манифест автоматически в течение нескольких часов после утверждения манифеста.

Если приложение запрашивает изменения разрешений, пользователям предлагается обновить и повторно дать согласие на приложение.

Эта строка версии должна следовать [стандарту semver](http://semver.org/) (MAJOR. несовершеннолетний. ПАТЧ).

## <a name="id"></a>id

**Требуется** - идентификатор приложения Майкрософт

Идентификатор является уникальным идентификатором Microsoft для приложения. У вас есть идентификатор, если ваш бот зарегистрирован через Microsoft Bot Framework или веб-приложение вашей вкладки уже входят в систему Майкрософт. Вы должны ввести id здесь. В противном случае необходимо создать новый идентификатор на портале [регистрации приложений Майкрософт.](https://aka.ms/appregistrations) Используйте тот же идентификатор, если вы добавляете бота.

> [!NOTE]
> Если вы отправляете обновление существующего приложения в AppSource, идентификатор в вашем манифесте не должен быть изменен.

|---|---|---|---|

## <a name="description"></a>description

**Требуется** - объект

Описывает ваше приложение для пользователей. Для приложений, представленных в AppSource, эти значения должны соответствовать информации в вашей записи AppSource.

Убедитесь, что ваше описание точно описывает ваш опыт и предоставляет информацию, чтобы помочь потенциальным клиентам понять, что ваш опыт делает. Вы должны отметить в полном описании, если внешний счет требуется для использования. Ценности и `short` должны `full` быть разными. Ваше краткое описание не должно повторяться в длинном описании и не должно включать любое другое имя приложения.

|Имя| Максимальный размер | Обязательный | Описание|
|---|---|---|---|
|`short`|80 символов|✔|Краткое описание вашего приложения, используемого при ограничении пространства.|
|`full`|4000 символов|✔|Полное описание вашего приложения.|

## <a name="validdomains"></a>действительныйДомайнс

**Дополнительно,** за исключением **Требуется,** где отметил.

Список действительных доменов для веб-сайтов, которые приложение планирует загрузить в Teams клиента. Списки доменов могут включать подстановочные знаки, `*.example.com` например. Это соответствует ровно одному сегменту домена; если вам нужно, чтобы `a.b.example.com` соответствовать затем использовать `*.*.example.com` . Если конфигурация вкладок или пользовательский интерфейс содержимого должны перемещаться в любой другой домен, кроме того, который используется для конфигурации вкладок, этот домен должен быть указан здесь.

Нет **необходимости включать** в приложение домены поставщиков идентификации, которые вы хотите поддержать. Например, для проверки подлинности с помощью Идентификатора Google необходимо перенаправить на accounts.google.com, однако, вы не должны включать accounts.google.com в `validDomains[]` .

Teams приложения, которые требуют, чтобы их собственные URL-адреса с точкой обмена функционировали хорошо, включают в свой действительный список доменов «teamsitedomain».

> [!IMPORTANT]
> Не добавляйте домены, которые находятся вне вашего контроля, ни непосредственно, ни через подстановочные знаки. Например, `yourapp.onmicrosoft.com` является действительным, однако, `*.onmicrosoft.com` не является действительным.

Объект является массивом со всеми элементами `string` типа.
