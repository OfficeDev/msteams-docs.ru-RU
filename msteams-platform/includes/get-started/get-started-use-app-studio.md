### <a name="use-app-studio-to-update-the-app-package"></a>Используйте App Studio для обновления пакета приложений

App Studio это Teams приложение, которое вы можете установить Teams магазине. Упрощает создание и регистрацию приложения.

Выполните следующие шаги по обновлению пакета приложений:

1. Чтобы установить App Studio Teams, выберите **значок Apps** в нижней части левой бара и ищите **App Studio:**

    <img  width="450px" alt="Finding App Studio in the Store View" src="~/assets/images/get-started/searchforAppStudio.png"/>

1. Выберите **плитку App Studio** и выберите **Установку.** App Studio установлен:

    <img  width="450px" alt="Installing App Studio" src="~/assets/images/get-started/InstallingAppStudio.png"/>

1. Чтобы создать пакет приложений для вашего Teams приложения, выберите **вкладку редактора Manifest** в **App Studio:**

    <img  width="450px" alt="App Studio" src="~/assets/images/get-started/AppStudio.png"/>

    Образец поставляется со своим манифестом и предназначен для создания пакета приложений при построении проекта. На .NET это делается Visual Studio и Node.js это делается путем `gulp` ввода на командной строке в корневом каталоге проекта.

    ```bash
    $ gulp
    [13:39:27] Using gulpfile ~\documents\github\msteams-samples-hello-world-nodejs\gulpfile.js
    [13:39:27] Starting 'clean'...
    [13:39:27] Starting 'generate-manifest'...
    [13:39:27] Finished 'generate-manifest' after 11 ms
    [13:39:27] Finished 'clean' after 21 ms
    [13:39:27] Starting 'default'...
    Build completed. Output in manifest folder
    [13:39:27] Finished 'default' after 62 μs
    ```

    Имя сгенерированного пакета приложений **helloworldapp.zip.** Вы можете искать этот файл, если местоположение не ясно в инструменте, который вы используете.

1. Теперь, чтобы изменить этот пакет приложений, **выберите Импорт существующего** приложения в **редакторе Manifest:**

    <img  width="450px" alt="Importing an existing app" src="~/assets/images/get-started/Importinganapp.png"/>

1. Выберите **плитку Hello World** для недавно импортированного приложения:

    <img  width="450px" alt="Newly imported app view" src="~/assets/images/get-started/HelloWorldappdetails.png"/>

На следующем изображении показан импортный пакет приложений в App Studio:

<img  width="450px" alt="Importing the app package" src="~/assets/images/get-started/Importinganapp2.png"/>

На левой стороне редактора Manifest есть список шагов. С правой стороны есть список свойств, которые должны быть заполнены для каждого шага. Когда вы начали с образца приложения, большая часть информации уже завершена. Следующие шаги позволяют обновить свойства приложения Hello World.

#### <a name="app-details"></a>Подробная информация о приложении

Выберите **детали приложения в подробном** **состоянии**. Выберите **кнопку** Generate для создания нового идентификатора App ID.

Ваш новый идентификатор App похож на `2322041b-72bf-459d-b107-f4f335bc35bd` .

Пройдитесь по деталям приложения в правом стекле, включая информацию **разработчика и сведения** **о брендинге.** Эти детали важны, если вы пишете новое приложение для распространения.

#### <a name="tabs"></a>Вкладки

Просто добавить вкладки в приложение Teams приложении. Пример приложения уже поддерживает несколько вкладок, и вы можете включить их.

##### <a name="team-tab"></a>Вкладка команды

Ваше приложение может иметь только одну вкладку Team:

<img  width="450px" alt="Adding a Teams tab" src="~/assets/images/get-started/TeamTab.png"/>

В этом примере отображается вкладка Team. Выберите **символ** URL **конфигурации Tab и** выберите **Редактировать** из выпадают меню. Измените URL-адрес `https://yourteamsapp.ngrok.io/configure` на `yourteamsapp.ngrok.io` URL-адрес, где он должен быть заменен URL-адресом, который [использовался при размещении приложения.](#host-the-sample-app)

##### <a name="personal-tabs"></a>Личные вкладки

Ваше приложение может иметь до 16 вкладок, включая вкладку Team.

Личные вкладки отличаются от вкладки **Team. Hello Tab** уже указан в личном списке вкладок со значением заполнителя. `com.contoso.helloworld.hellotab` Выберите **символ** URL **конфигурации Tab и** выберите **Редактировать** из выпадают меню. Появляется следующее диалоговое окно:

<img  width="450px" alt="Adding a personal tab dialog" src="~/assets/images/get-started/PersonalTab.png"/>

Обновление следующих коробок с URL-адресом приложения:

- Изменение **url-адреса содержимого** на `https://yourteamsapp.ngrok.io/hello`
- Изменение **URL-адреса веб-сайта** на `https://yourteamsapp.ngrok.io/hello`

Замените `yourteamsapp.ngrok.io` URL-адрес, который вы использовали при размещении приложения.

#### <a name="bots"></a>Боты

Легко добавить функциональность ботов в ваше приложение. В **примере** приложения Hello World уже есть бот как часть образца, но вы должны зарегистрировать его в корпорации Майкрософт:

<img  width="450px" alt="Adding a bot" src="~/assets/images/get-started/Bots.png"/>

Бот, который был импортирован из образца, не имеет связанного идентификатора App ID. Вы должны создать нового бота, чтобы App Studio можно было создать новый идентификатор App ID и зарегистрировать его в корпорации Майкрософт.

> [!NOTE]
> Идентификатор App, созданный App Studio для бота, отличается от идентификатора App ID, созданного для приложения. Каждый бот в приложении требует своего идентификатора App ID.

Выполните следующие шаги по настройке бота:

1. Выберите **Удалить** рядом с импортированным ботом в списке ботов. Теперь нет ботов осталось показать. 
1. Выберите **настройку** для **отображения диалогового окна** Для бота.

    <img  width="450px" alt="Adding a bot dialog" src="~/assets/images/get-started/Setupbot.png"/>

1. Добавьте бот имя **Contoso бот и** выберите все три чек-боксы под **областью.**
1. Выберите **Сохранить,** чтобы выйти из диалогового окна. App Studio регистрирует вашего бота в корпорации Майкрософт и отображает ваш новый бот в списке ботов. 
1. Теперь откройте текстовый файл в блокноте и скопировать и вставить свой новый идентификатор бота в него.
1. Нажмите **Generate New Password** и обратите внимание на пароль в том же текстовом файле, который вы отметили в идентификаторе приложения бота.
1. **Обновите адрес конечной точки** `https://yourteamsapp.ngrok.io/api/messages` Bot и `yourteamsapp.ngrok.io` замените URL-адрес, который вы использовали при размещении приложения.
1. Теперь сохраните текстовый файл, поскольку вы должны добавить информацию из файла в ваше приложение для хостинга, чтобы обеспечить безопасную связь с ботом.

#### <a name="messaging-extensions"></a>Расширения для система обмена сообщениями

Расширения сообщений позволяют пользователям запрашивать информацию из вашего сервиса и размещать эту информацию. Информация размещена в виде карт в разговоре канала. Расширения сообщений отображаются в нижней части окна compose.

Выполните следующие шаги по настройке расширения обмена сообщениями:

1. Выберите **расширения обмена** **сообщениями под** возможностями в левом стекле App Studio для настройки расширения обмена сообщениями:

    <img  width="450px" alt="Adding a messaging extension" src="~/assets/images/get-started/Messagingextensions.png"/>

    Пример расширения обмена сообщениями указан в **панели расширения сообщений.**

1. Выберите **Удалить,** чтобы удалить расширение обмена сообщениями, **выберите Настройка,** и следовать тем же шагам, используемым [для ботов.](#bots) Отображается диалоговое окно Messaging **Extension.**
1. Выберите **существующую вкладку бота** и **выберите одного из моих существующих ботов.**
1. Выберите бота, созданного из выпадают из меню. Добавьте **имя Bot и** выберите **Сохранить,** чтобы закрыть диалоговое окно.
1. В разделе **Командование** выберите **Добавить**. Чтобы добавить команду на основе поиска, выберите **Разрешить пользователям запрашивать вашу службу для получения информации и вставить ее в опцию сообщения.**
1. В новом **диалоговом** поле команды введите следующие значения:

    Под **новым командованием**:

    - **Идентификатор команды**: Введите случайный текст
    - **Название**: Введите случайное название
    - **Описание /** описание : Введите случайное описание

    В **соответствии с параметром**:

    - **Имя**: Введите имя параметра
    - **Название**: Введите название карты
    - **Описание**/ см. Описание : Введите описание карты

1. После вставки информации выберите **Сохранить,** чтобы закрыть диалоговое окно.

#### <a name="register-your-app-in-teams"></a>Зарегистрируйте приложение в Teams

После ввода деталей приложения выполните следующие шаги по регистрации приложения в Teams:

1. Используйте **тест и распространение** App Studio для установки приложения в Teams. 
1. Обновите размещенную заявку с помощью идентификатора приложения и пароля для бота. Для образца приложения используйте тот же идентификатор App ID и пароль как для бота, так и для расширения обмена сообщениями. 
1. Выберите **тест и** **распределите** под отделкой в левом стекле App Studio:

    <img  width="450px" alt="Testing your app" src="~/assets/images/get-started/Testanddistribute.png"/>

1. Чтобы загрузить приложение в Teams, выберите кнопку **«Установить»** в рамках **тестирования и распространения:**

    <img  width="450px" alt="Adding a messaging extension dialog" src="~/assets/images/get-started/InstallingHelloWorld.png"/>

1. Выберите **поле** поиска в **разделе Добавить в раздел команды** и выберите команду для добавления образца приложения. Вы можете создать специальную группу для тестирования.
1. Выберите **кнопку** «Установить» в нижней части диалогового окна.

Ваше приложение теперь доступно в Teams. Тем не менее, бот и расширение обмена сообщениями не будут работать до тех пор, пока вы не обновите среду размещенных приложений с помощью ID и паролей App.
