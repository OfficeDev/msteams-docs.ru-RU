### <a name="use-app-studio-to-update-the-app-package"></a>Использование App Studio для обновления пакета приложения

App Studio — это приложение Teams, которое можно установить из магазина Teams. Это упрощает создание и регистрацию приложения.

Чтобы установить App Studio в Teams, выберите значок **"Приложения"** в нижней части левой панели и нащите **App Studio.**

<img  width="450px" alt="Finding App Studio in the Store View" src="~/assets/images/get-started/searchforAppStudio.png"/>

Выберите **плитку App Studio** и выберите **"Установить".** Установлена App Studio.

<img  width="450px" alt="Installing App Studio" src="~/assets/images/get-started/InstallingAppStudio.png"/>

Выберите **вкладку редактора манифеста,** чтобы создать пакет приложения для приложения Teams.

<img  width="450px" alt="App Studio" src="~/assets/images/get-started/AppStudio.png"/>

Пример поставляется с собственным предварительным манифестом и разработан для создания пакета приложения при сборке проекта. В .NET это делается в Visual Studio, а в Node JS это делается путем ввода в командной строке корневого `gulp` каталога проекта.

```bash
$ gulp
[13:39:27] Using gulpfile ~\documents\github\msteams-samples-hello-world-nodejs\gulpfile.js
[13:39:27] Starting 'clean'...
[13:39:27] Starting 'generate-manifest'...
[13:39:27] Finished 'generate-manifest' after 11 ms
[13:39:27] Finished 'clean' after 21 ms
[13:39:27] Starting 'default'...
Build completed. Output in manifest folder
[13:39:27] Finished 'default' after 62 μs
```

Имя сгенерированного пакета приложения **helloworldapp.zip.** Вы можете найти этот файл, если в используемом средстве не ясное расположение.

Теперь, чтобы изменить этот пакет приложения, выберите **"Импорт существующей** плитки приложения" в **редакторе манифеста.**

<img  width="450px" alt="Importing an existing app" src="~/assets/images/get-started/Importinganapp.png"/>

Щелкните **плитку Hello World** для недавно импортируемого приложения.

<img  width="450px" alt="Newly imported app view" src="~/assets/images/get-started/HelloWorldappdetails.png"/>

На следующем рисунке показан импортируемый пакет приложения в App Studio:

<img  width="450px" alt="Importing the app package" src="~/assets/images/get-started/Importinganapp2.png"/>

В левой части редактора манифеста есть список действий, а справа — список свойств, которые необходимо заполнить для каждого из этих действий. Так как вы начали работу с образцом приложения, большая часть информации уже заполнена. Далее вы сможете изменить части, которые все еще нужно обновить.

#### <a name="app-details"></a>Сведения о приложении

Щелкните запись *сведений о приложении* в области *"Сведения".* Нажмите *кнопку* "Создать", чтобы создать новый ид приложения.

Ваш новый ид приложения должен выглядеть так: `2322041b-72bf-459d-b107-f4f335bc35bd` .

Ознакомьтесь с остальными сведениями о приложении на правой области и ознакомьтесь с  некоторыми записями, такими как сведения о разработчике и *фирменный брендинг.* Эти разделы важны при написании нового приложения для распространения.

#### <a name="capabilities-tabs"></a>Возможности: вкладки

Вкладки — это один из самых простых элементов, добавляемой в приложение Teams. Пример приложения уже поддерживает несколько вкладок, и их можно включить следующим образом.

##### <a name="team-tab"></a>Вкладка "Группа"

В вашем приложении может быть только одна вкладка "Группа".

<img  width="450px" alt="Adding a Teams tab" src="~/assets/images/get-started/TeamTab.png"/>

В этом примере на вкладке "Группа" находится страница конфигурации. Щелкните *символ ...в* конце записи  и выберите "Изменить" в выпадаемом. Измените URL-адрес на то, где следует заменить URL-адрес, используемый выше `https://yourteamsapp.ngrok.io/configure` `yourteamsapp.ngrok.io` при размещении приложения.

##### <a name="personal-tabs"></a>Личные вкладки

Ваше приложение может иметь до 16 вкладок, включая вкладку команды.

Личные вкладки представлены иначе, чем на вкладке команды. В списке личных *вкладок должна* быть уже указана вкладка "Hello". В настоящее время он имеет значение-замес.. `com.contoso.helloworld.hellotab` Щелкните *символ ...в* конце записи  и выберите "Изменить" в выпадаемом. Появится следующее диалоговое окно.

<img  width="450px" alt="Adding a personal tab dialog" src="~/assets/images/get-started/PersonalTab.png"/>

Существует два поля, которые необходимо обновить с помощью URL-адреса приложения.

- Изменение URL-адреса контента на `https://yourteamsapp.ngrok.io/hello`
- Изменение URL-адреса веб-сайта на `https://yourteamsapp.ngrok.io/hello`

Где `yourteamsapp.ngrok.io` следует заменить URL-адрес, используемый выше при размещении приложения.

#### <a name="bots"></a>Боты

Боты — это наиболее распространенный способ добавления функций в приложение. В примере hello world уже есть бот в составе примера, но он еще не зарегистрирован в Корпорации Майкрософт.

<img  width="450px" alt="Adding a bot" src="~/assets/images/get-started/Bots.png"/>

У бота, импортируемого из примера, еще нет связанного с ним ИД приложения. Вам придется создать нового бота, чтобы App Studio мог создать новый ИД приложения и зарегистрировать его в Корпорации Майкрософт. Обратите внимание, что это ИД приложения для бота, который отличается от ИД приложения, созданного для приложения. Каждому боту в приложении требуется собственный ИД приложения.

Нажмите *кнопку удаления* рядом с импортируемым *ботом* в списке ботов.

Теперь не осталось ботов для показа. Щелкните *"Установка"*. Отобразит *диалоговое окно "Настройка* бота".

<img  width="450px" alt="Adding a bot dialog" src="~/assets/images/get-started/Setupbot.png"/>

Добавьте имя бота, `Contoso bot` например, и выберите все три кнопки в области.

Choose *Create bot* to exit the dialog. App Studio регистрирует бота в корпорации Майкрософт и отображает нового бота в списке ботов. Пришло бы время открыть текстовый файл в блокноте, скопировать и ввести в него новый бот. Этот ид потребуется позже.

Нажмите *кнопку "Создать* новый пароль" и заметьте пароль в том же текстовом файле, в который вы указали код приложения-бота. Это единственный раз, когда будет показан пароль, поэтому не забудьте сделать это сейчас.

*Обновите адрес конечной точки бота,* где его следует заменить URL-адресом, который вы использовали при размещении `https://yourteamsapp.ngrok.io/api/messages` `yourteamsapp.ngrok.io` приложения.

Пришло бы время сохранить текстовый файл, если вы еще этого не сделали. Эти сведения будут добавлены в ваше приложение, которое будет добавлено далее в этом поговом поговом, что обеспечит безопасное взаимодействие с ботом.

#### <a name="messaging-extensions"></a>Расширения для система обмена сообщениями

Расширения обмена сообщениями позволяет пользователям запросить сведения из вашей службы и опубликовать эти сведения в виде карточек прямо в беседе канала. Расширения сообщений отображаются в нижней части окна составить.

Выберите **расширения сообщений в** области **"Возможности"** в левом столбце App Studio, чтобы настроить расширение обмена сообщениями.

<img  width="450px" alt="Adding a messaging extension" src="~/assets/images/get-started/Messagingextensions.png"/>

Пример расширения обмена сообщениями находится в правой области в **списке расширений обмена сообщениями.** Еще **раз нажмите** кнопку "Удалить", чтобы удалить эту запись, а затем нажмите кнопку "Настроить", выполив те же действия, что и для ботов.  Отобразит *диалоговое окно расширения* обмена сообщениями.

Выберите *вкладку "Использовать существующий бот",* а затем выберите один *из существующих ботов.* В выпадаемом меню выберите бота, созданного в разделе выше. Добавьте имя *бота и* нажмите *кнопку* "Сохранить", чтобы закрыть диалоговое окно.

В разделе *"Команда"* нажмите кнопку *"Добавить".* Мы добавляем команду на основе поиска, поэтому выберите параметр *"Разрешить пользователям запрашивать вашу службу...".*

В **диалоговом окте** "Новая команда" введите следующие значения.

В *области "Новая команда" :*

- *ИД команды*  = getRandomText
- *Title*       = Get some random text for fun
- *Description* = Получает произвольный текст и изображения

Under *Parameter*:

- *Name*        = cardTitle
- *Title*       = Card title
- *Description* = Card title to use

После того как вы ввели данные, нажмите кнопку *"Сохранить",* чтобы закрыть диалоговое окно.

#### <a name="register-your-app-in-teams"></a>Регистрация приложения в Teams

Теперь вы завершили ввод сведений о своем приложении, но осталось два следующих этапа:
1. Используйте раздел "Тестирование и распространение" в App Studio, чтобы установить приложение в Teams.
1. Обновите свое приложение с помощью ИД приложения и пароля для бота. Помните, что в примере предполагается использовать один и тот же код приложения и пароль для бота и расширения обмена сообщениями.

Выберите элемент **"Тестирование и распространение"** в области **"Готово"** в левом столбце App Studio.

<img  width="450px" alt="Testing your app" src="~/assets/images/get-started/Testanddistribute.png"/>

Чтобы отправить приложение в Teams, нажмите кнопку *"Установить"* в области *"Тестирование и распространение".*

<img  width="450px" alt="Adding a messaging extension dialog" src="~/assets/images/get-started/InstallingHelloWorld.png"/>

Выберите поле **поиска** в разделе **"Добавить в** команду" и выберите команду для добавления примера приложения. Обычно можно настроить специальную команду для тестирования.

Выберите **кнопку "Установить"** в нижней части диалога.

Это завершает часть Этого по walkthrough в App Studio. Теперь приложение должно работать в Teams, но бот и расширение обмена сообщениями не будут работать, пока вы не обновите среду приложений, чтобы узнать, что такое коды и пароли приложений.

<img  width="450px" alt="The finished app" src="~/assets/images/get-started/Finishedhelloworld.png"/>
