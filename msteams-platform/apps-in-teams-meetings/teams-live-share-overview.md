# <a name="live-share-sdk"></a>Пакет SDK Live Share

> [!VIDEO https://www.youtube.com/embed/971YIvosuUk]

Live Share — это пакет SDK, предназначенный для преобразования приложений Teams в совместные многопользовательские приложения без написания выделенного внутреннего кода. С помощью Live Share пользователи могут совместно просматривать, совместно создавать и редактировать их во время собраний.

Иногда демонстрации экрана недостаточно, поэтому корпорация Майкрософт создает такие инструменты, как PowerPoint Live и доску непосредственно в Teams. При переводе веб-приложения непосредственно в центр интерфейса собрания пользователи могут без проблем совместно работать во время собраний и звонков.

> [!div class="nextstepaction"]
> [Начало работы](teams-live-share-quick-start.md)

## <a name="feature-overview"></a>Обзор функций

Live Share содержит три пакета, поддерживающих бесграничные сценарии совместной работы. Эти пакеты предоставляют набор распределенных структур данных (DDS), включая примитивные стандартные блоки и сценарии с пошаговой настройкой.

Live Share легко интегрирует собрания с [Fluid Framework](https://fluidframework.com/). Fluid Framework — это набор клиентских библиотек для распространения и синхронизации общего состояния. Live Share предоставляет бесплатную, полностью управляемую и готовую к использованию службу [Azure Fluid Relay](/azure/azure-fluid-relay/), поддерживаемую безопасностью и глобальным масштабом Teams.

### <a name="live-share-core"></a>Live Share Core

Live Share позволяет подключаться к специальному гибкому контейнеру, связанному с каждым собранием, в нескольких строках кода. Помимо структур данных, предоставляемых Fluid Framework, Live Share также поддерживает новый набор классов DDS для упрощения синхронизации состояния приложения на собраниях.

Ниже компоненты, поддерживаемые пакетом Live Share Core:

- Присоединяйтесь к сеансу Live Share собрания `LiveShareClient`с помощью .
- Отслеживание присутствия на собрании и синхронизация метаданных пользователя с `LivePresence`.
- Отправка событий в режиме реального времени другим клиентам в сеансе с помощью `LiveEvent`.
- Координирует состояние приложения, которое исчезает, когда пользователи покидают сеанс с `LiveState`помощью .
- Синхронизация таймера обратного отсчета с `LiveTimer`.
- Используйте любую функцию Fluid Framework, например `SharedMap` и `SharedString`.

Дополнительные сведения об этом пакете можно найти на странице [основных возможностей](./teams-live-share-capabilities.md).

### <a name="live-share-media"></a>Динамический общий доступ к мультимедиа

:::image type="content" source="../assets/images/teams-live-share/teams-live-share-contoso-video.gif" alt-text="Снимок экрана: пример функции общего доступа к видео в Live Share.":::

Видео и аудио являются важной частью современного мира и рабочего места. Мультимедиа Live Share обеспечивает синхронизацию **мультимедиа** для любого проигрывателя мультимедиа с несколькими строками кода. Синхронизация мультимедиа на уровне состояния проигрывателя и элементов управления транспортировкой позволяет по отдельности атрибутизировать представления, обеспечивая при этом наивысшее качество, доступное в приложении. Так как корпорация Майкрософт не выполняет повторную рассылку содержимого мультимедиа, требования к лицензированию и доступу не изменяются.

К функциям, поддерживаемым мультимедиа Live Share, относятся:

- Синхронизация состояния проигрывателя мультимедиа и отслеживание с помощью `MediaPlayerSynchronizer`.
- Интеллектуальные корректировки громкости мультимедиа по мере того, как пользователи общались во время собрания.
- Ограничение того, какие пользователи могут изменять состояние проигрывателя.
- Приостановка и возобновление синхронизации мультимедиа во время выполнения или в запланированных точках ожидания.

Дополнительные сведения об этом пакете можно найти на странице [мультимедиа Live Share](./teams-live-share-media-capabilities.md).

> [!NOTE]
> Live Share не выполняет повторную рассылку содержимого мультимедиа. Он предназначен для использования со встроенными веб-проигрывателем, такими как HTML5 `<video>` или Проигрыватель мультимедиа Azure.

### <a name="live-share-canvas"></a>Холст Live Share

:::image type="content" source="../assets/images/teams-live-share/Teams-live-share-schematics.png" alt-text="На снимках экрана показан пример нескольких пользователей, нарисованных на холсте во время собрания.":::

При совместной работе на собраниях важно, чтобы пользователи могли указывать на содержимое на экране и подчеркивать его. Холст Live Share упрощает добавление рукописного ввода, лазерных указателей и курсоров в приложение для удобной совместной работы.

Функции, поддерживаемые холстом Live Share:

- Добавьте совместную работу `<canvas>` в приложение с помощью `LiveCanvas`.
- Делиться идеями можно с помощью пера, выделения, линии и инструментов со стрелками.
- Эффективное использование лазерного указателя.
- Следуйте указаниям курсоров мыши в режиме реального времени.
- Настройте параметры для переменных устройств и состояний просмотра.
- Используйте полностью поддерживаемые входные данные мыши, касания и пера.

Дополнительные сведения об этом пакете можно найти на странице [холста Live Share](./teams-live-share-canvas.md).

## <a name="why-build-apps-with-live-share"></a>Зачем создавать приложения с помощью Live Share?

Создание приложений для совместной работы может быть сложным, трудоемким, дорогостоящим и включает в себя сложные требования соответствия в масштабе. Пользователи Teams тратят значительное количество времени на анализ работы с товарищами по команде, совместный просмотр видео и мозговой штурм новых идей посредством демонстрации экрана. Пакет SDK Live Share позволяет преобразовать ваше приложение в нечто более полезное для совместной работы с минимальными вложениями.

Ниже приведены некоторые основные преимущества пакета SDK Live Share.

- Простое управление сессиями и безопасность.
- Распределенные структуры данных с состоянием и без состояния.
- Расширения мультимедиа для удобной синхронизации видео и звука.
- Рукописный ввод с помощью ключа, лазерная указка и курсоры.
- Соблюдение привилегий собрания с помощью проверки роли.
- Бесплатная и полностью управляемая служба с низкой задержкой.

Чтобы понять, является ли Live Share правильным для вашего сценария совместной работы, полезно понимать различия между Live Share и другими платформами для совместной работы, в том числе:

- [Веб-сокеты](#web-sockets)
- [Гибкий ретранслятор Azure](#azure-fluid-relay)
- [Live Share](#live-share-hosted-service)

### <a name="web-sockets"></a>Веб-сокеты

Веб-сокеты — это распространенная технология для обмена данными в режиме реального времени в Интернете, и некоторые приложения могут использовать собственную собственную серверную часть веб-сокета. В отличие от REST API, веб-сокеты используют открытое соединение между сервером и клиентами в сеансе.

Как и другие пользовательские службы API, требования обычно включают проверку подлинности сеансов, региональное сопоставление, обслуживание и масштабирование. Многие сценарии совместной работы также требуют поддержания состояния сеанса на сервере, что требует инфраструктуры хранилища, разрешения конфликтов и т. д.

С помощью Live Share вы получаете все функции веб-сокетов без дополнительных затрат.

### <a name="azure-fluid-relay"></a>Гибкий ретранслятор Azure

[Azure Fluid Relay](/azure/azure-fluid-relay/) — это управляемое предложение для Fluid Framework, которое помогает разработчикам создавать возможности совместной работы в режиме реального времени и реплицировать состояние между подключенными клиентами JavaScript. Microsoft Whiteboard, Loop и OneNote — это все примеры приложений, созданных с помощью Fluid Framework.

Как и другие службы Azure, Azure Fluid Relay предназначен для адаптации к потребностям отдельных проектов с минимальной сложностью. Требования включают разработку истории проверки подлинности для контейнеров Fluid и соответствие региональным требованиям. После настройки разработчики могут сосредоточиться на высококачественной совместной работе.

### <a name="live-share-hosted-service"></a>Размещенная служба Live Share

Live Share предоставляет пошаговую службу Azure Fluid Relay, поддерживаемую безопасностью собраний Microsoft Teams. Контейнеры Live Share ограничены участниками собрания, поддерживают требования к месту расположения клиента и могут быть доступны в нескольких строках клиентского кода.

# <a name="javascript"></a>[JavaScript](#tab/javascript)

```javascript
import { LiveShareClient, LivePresence } from "@microsoft/live-share";

// Join the Fluid container
const liveShare = new LiveShareClient();
const schema = {
  initialObjects: { presence: LivePresence },
};
const { container } = await liveShare.joinContainer(schema);

// ... ready to start app sync logic
```

# <a name="typescript"></a>[TypeScript](#tab/typescript)

```TypeScript
import { LiveShareClient, LivePresence } from "@microsoft/live-share";
import { ContainerSchema } from "fluid-framework";

// Join the Fluid container
const liveShare = new LiveShareClient();
const schema: ContainerSchema = {
  initialObjects: { presence: LivePresence },
};
const { container } = await liveShare.joinContainer(schema);

// ... ready to start app sync logic
```

---

> [!IMPORTANT]
> Все данные, отправляемые или хранящиеся в размещенной службе Azure Fluid Relay пакета SDK Live Share, доступны в течение 24 часов. Дополнительные сведения см. в разделе [Вопросы и ответы о Live Share](teams-live-share-faq.md).

#### <a name="using-a-custom-azure-fluid-relay-service"></a>Использование пользовательской службы Azure Fluid Relay

Хотя большинству из вас предпочтительнее использовать бесплатную размещенную службу, по-прежнему есть ситуации, в которых полезно использовать собственную службу Azure Fluid Relay для приложения Live Share.

Рассмотрите возможность использования пользовательской службы, если:

- Требовать хранение данных в контейнерах с плавными данными за пределами времени существования собрания.
- Передайте конфиденциальные данные через службу, которая требует настраиваемой политики безопасности.
- Разработка функций с помощью Fluid Framework, например, `SharedMap`для приложения за пределами Teams.

Дополнительные сведения см. в руководстве по пользовательской службе Azure Fluid [Relay.](./teams-live-share-how-to/how-to-custom-azure-fluid-relay.md)
