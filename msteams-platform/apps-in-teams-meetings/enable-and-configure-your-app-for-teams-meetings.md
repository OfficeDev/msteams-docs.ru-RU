# <a name="enable-and-configure-your-apps-for-teams-meetings"></a>Включение и настройка приложений для Teams собраний

У каждой команды есть разные способы взаимодействия и совместной работы. Для выполнения этих различных задач настройте Teams приложениями для собраний. Включите приложения для Teams собраний и настройте их доступность в области собраний в манифесте приложения.

## <a name="prerequisites"></a>Предварительные требования

Приложения для Teams собраний расширяют возможности своих приложений на протяжении всего жизненного цикла собрания. Прежде чем работать с приложениями для Teams собраний, необходимо выполнить следующие предварительные требования:

* Узнайте, как разрабатывать Teams приложения. Дополнительные сведения о разработке Teams см. в Teams[.](../overview.md)

* Используйте приложение, которое поддерживает настраиваемые вкладки в области groupchat. Дополнительные сведения см. в [области группового чата](../resources/schema/manifest-schema.md#configurabletabs) [и создании вкладки группы](../build-your-first-app/build-channel-tab.md).

* Соблюдайте общие [Teams вкладки](../tabs/design/tabs.md) для сценариев до и после собрания. Сведения о взаимодействиях во время собраний см. [](../apps-in-teams-meetings/design/designing-apps-in-meetings.md#use-an-in-meeting-tab) в рекомендациях по проектированию вкладок в собрании и в диалоговом [окне собрания](../apps-in-teams-meetings/design/designing-apps-in-meetings.md#use-an-in-meeting-dialog).

* Чтобы ваше приложение обновлялись в режиме реального времени, оно должно быть актуальным на основе действий, связанных с событиями в собрании. Эти события могут быть в диалоговом окне собрания и на других этапах жизненного цикла собрания. Сведения о диалоговом окне собрания см `completionBotId` . в описании параметра в полезных данных уведомления о [собрании](API-references.md#send-an-in-meeting-notification).

## <a name="enable-your-app-for-teams-meetings"></a>Включение приложения для Teams собраний

Чтобы включить приложение для Teams собраний, обновите манифест приложения и используйте свойства контекста, чтобы определить, где должно отображаться приложение.

### <a name="update-your-app-manifest"></a>Обновление манифеста приложения

Возможности приложения для собраний объявляются в манифесте приложения с помощью массивов `configurableTabs``context` и `scopes`массивов. Область определяет, кто может получить доступ, а контекст определяет, где доступно ваше приложение.

> [!NOTE]
>
> * Необходимо обновить манифест приложения, используя [схему манифеста](../resources/schema/manifest-schema-dev-preview.md).
> * Приложения на собраниях требуют области `groupchat` действия. Область `team` работает только для вкладок в каналах.

Манифест приложения должен содержать следующий фрагмент кода:

```json

"configurableTabs": [
    {
      "configurationUrl": "https://contoso.com/teamstab/configure",
      "canUpdateConfiguration": true,
      "scopes": [
        "team",
        "groupchat"
      ],
      "context":[
        "channelTab",
        "privateChatTab",
        "meetingChatTab",
        "meetingDetailsTab",
        "meetingSidePanel",
        "meetingStage"
     ]
    }
  ]
```

### <a name="context-property"></a>Свойство Context

Свойство `context` определяет, что должно отображаться, когда пользователь вызывает приложение на собрании в зависимости от того, где пользователь вызывает приложение. Вкладка `context` и `scopes` свойства позволяют определить, где должно отображаться приложение. Вкладки в области `team` или области `groupchat` могут иметь более одного контекста.

Поддержка области `groupchat` , чтобы включить приложение в чатах перед собранием и после собрания. С помощью приложения перед собранием вы можете находить и добавлять приложения для собраний и выполнять задачи перед собранием. С помощью приложения после собрания вы можете просмотреть результаты собрания, например результаты опроса или плату.

 Ниже приведены значения для свойства `context` , из которого можно использовать все или некоторые значения:

|Значение|Описание|
|---|---|
| **channelTab** | Вкладка в заголовке канала команды. |
| **privateChatTab** | Вкладка в заголовке группового чата между набором пользователей, а не в контексте команды или собрания. |
| **meetingChatTab** | Вкладка в заголовке группового чата между набором пользователей для запланированного собрания. Вы можете указать **meetingChatTab** **или meetingDetailsTab** , чтобы обеспечить работу приложений на мобильных устройствах. |
| **meetingDetailsTab** | Вкладка в заголовке представления сведений о собрании календаря. Вы можете указать **meetingChatTab** **или meetingDetailsTab** , чтобы обеспечить работу приложений на мобильных устройствах. |
| **meetingSidePanel** | Панель собрания, открытая через единую панель (U-bar). |
| **meetingStage** | Приложение из папки может `meetingSidePanel` быть предоставлено на этап собрания. Это приложение нельзя использовать в клиентах на мобильных устройствах или клиентах комнат Teams. |

После включения приложения для Teams собраний необходимо настроить приложение перед собранием, во время собрания и после собрания.

## <a name="configure-your-app-for-meeting-scenarios"></a>Настройка приложения для сценариев собраний

Teams собрания обеспечивают совместную работу для вашей организации. Настройте приложение для различных сценариев собраний и улучшите взаимодействие с собранием. Теперь вы можете определить, какие действия можно предпринять в следующих сценариях собраний:

* [Перед собранием](#before-a-meeting)
* [Во время собрания](#during-a-meeting)
* [После собрания](#after-a-meeting)

### <a name="before-a-meeting"></a>Перед собранием

Перед собранием пользователи могут добавлять вкладки, боты и расширения сообщений. Пользователи с ролями организатора и докладчика могут добавлять вкладки к собранию.

Чтобы добавить вкладку к собранию:

1. В календаре выберите собрание, к которому вы хотите добавить вкладку.
1. Выберите **вкладку "Сведения** " и нажмите кнопку <img src="~/assets/images/apps-in-meetings/plusbutton.png" alt="Plus button" width="30"/>.

    <img src="../assets/images/apps-in-meetings/PreMeeting.png" alt="Pre-meeting experience" width="900"/>

1. В отображемой коллекции вкладок выберите приложение, которое вы хотите добавить, и выполните необходимые действия. Приложение устанавливается как вкладка.

Чтобы добавить расширение сообщения на собрание:

1. Щелкните многоточие &#x25CF;&#x25CF;&#x25CF; в области сообщения создания в чате.
1. Выберите приложение, которое вы хотите добавить, и выполните необходимые действия. Приложение устанавливается как расширение сообщения.

Чтобы добавить бота на собрание:

В чате собрания введите ключ **@** и выберите " **Получить ботов"**.

> [!NOTE]
> * В диалоговом окне собрания отображается диалоговое окно собрания и одновременно публикуется адаптивная карточка в чате собрания, к которой пользователи могут получить доступ. Адаптивная карточка в чате собрания помогает пользователям посетить собрание или если Teams приложение свернуто.
> * Удостоверение пользователя должно быть подтверждено с помощью единого входа [табуляции](../tabs/how-to/authentication/auth-aad-sso.md). После проверки подлинности приложение может получить роль пользователя с помощью `GetParticipant` API.
> * В зависимости от роли пользователя приложение может предоставлять возможности для работы с определенной ролью. Например, приложение для опроса позволяет создавать опрос только организаторам и выступающим.
> * Назначения ролей можно изменить во время собрания. Дополнительные сведения см. [в разделе ролей Teams собрания](https://support.microsoft.com/office/roles-in-a-teams-meeting-c16fa7d0-1666-4dde-8686-0a0bfe16e019).

### <a name="during-a-meeting"></a>Во время собрания

Во время собрания вы можете `meetingSidePanel` использовать уведомление или уведомление на собрании для создания уникальных интерфейсов для приложений.

#### <a name="meeting-sidepanel"></a>SidePanel собрания

Это `meetingSidePanel` позволяет настраивать интерфейсы собрания, позволяющие организаторам и выступающим иметь разные представления и действия. В манифесте приложения необходимо добавить в `meetingSidePanel` массив контекста. Во время собрания и во всех сценариях приложение отображается на вкладке в собрании шириной 320 пикселей. Дополнительные сведения см. в [интерфейсе FrameContext](/javascript/api/@microsoft/teams-js/microsoftteams.framecontext?view=msteams-client-js-latest&preserve-view=true).

Сведения об использовании `userContext` API для маршрутизации запросов см. Teams [SDK](../tabs/how-to/access-teams-context.md#user-context). Дополнительные сведения см[. в Teams проверки подлинности для вкладок](../tabs/how-to/authentication/auth-flow-tab.md). Поток проверки подлинности для вкладок аналогичен потоку проверки подлинности для веб-сайтов. Поэтому вкладки могут использовать OAuth 2.0 напрямую. Дополнительные сведения см[. в платформа удостоверений Майкрософт кода авторизации OAuth 2.0](/azure/active-directory/develop/v2-oauth2-auth-code-flow).

Расширение сообщений работает должным образом, когда пользователь находится в представлении собрания. Пользователь может публиковать карточки расширения сообщения создания. AppName в собрании — это подсказка, которая указывает имя приложения на панели U-bar собрания.

> [!NOTE]
> Используйте версию 1.7.0 или более позднюю версию Teams [SDK](/javascript/api/overview/msteams-client?view=msteams-client-js-latest&preserve-view=true), так как предыдущие версии не поддерживают боковую панель.

#### <a name="in-meeting-notification"></a>Уведомление на собрании

Уведомление о собрании используется для привлечения участников во время собрания и сбора сведений или отзывов во время собрания. Используйте [полезные данные уведомления в](API-references.md#send-an-in-meeting-notification) собрании, чтобы активировать уведомление на собрании. В качестве полезных данных запроса на уведомление укажите URL-адрес, в котором размещается отображаемое содержимое.

Уведомление на собрании не должно использовать модуль задачи. Модуль задач не вызывается в чате собрания. URL-адрес внешнего ресурса используется для отображения уведомления о собрании. Этот метод можно использовать `submitTask` для отправки данных в чате собрания.

:::image type="content" source="../assets/images/apps-in-meetings/in-meeting-dialogbox.png" alt-text="В примере показано, как можно использовать диалоговое окно собрания." border="true":::

#### <a name="shared-meeting-stage"></a>Общая трибуна собраний

Общая стадия собрания позволяет участникам собрания взаимодействовать с содержимым приложения и работать над ним в режиме реального времени. Вы можете поделиться приложениями на этапе совместной работы следующими способами:

* [Предоставление общего доступа ко всему приложению для промежуточной](#share-entire-app-to-stage) подготовки с помощью кнопки "Общий доступ к этапу" в Teams клиенте.
* [Совместное использование определенных частей приложения для промежуточной подготовки](#share-specific-parts-of-the-app-to-stage) с помощью API в Teams клиентском пакете SDK.

##### <a name="share-entire-app-to-stage"></a>Предоставление общего доступа ко всему приложению на этапе

Участники могут предоставлять общий доступ ко всему приложению на этапе совместной работы, используя кнопку "Общий доступ к этапу" на боковой панели приложения.

<img src="../assets/images/apps-in-meetings/share_to_stage_during_meeting.png" alt="Share full app" width = "900"/>

Чтобы поделиться всем приложением на этапе, в манифесте приложения необходимо настроить `meetingStage` `meetingSidePanel` и в качестве контекстов кадров. Например:

```json
"configurableTabs": [
    {
      "configurationUrl": "https://contoso.com/teamstab/configure",
      "canUpdateConfiguration": true,
      "scopes": [
        "groupchat"
      ],
      "context":[
        "meetingSidePanel",
        "meetingStage"
     ]
    }
  ]
```

Дополнительные сведения см. в [манифесте приложения](../resources/schema/manifest-schema-dev-preview.md#configurabletabs).

##### <a name="share-specific-parts-of-the-app-to-stage"></a>Предоставление общего доступа к определенным частям приложения для промежуточной подготовки

Участники могут совместно использовать определенные части приложения на этапе совместной работы, используя общую папку для подготовки API. API-интерфейсы доступны в Teams клиентском пакете SDK и вызываются на боковой панели приложения.

<img src="../assets/images/apps-in-meetings/share-specific-content-to-stage.png" alt="Share specific parts of the app" width = "900"/>

Чтобы совместно использовать определенные части приложения для промежуточного выполнения, необходимо вызвать связанные API в Teams клиентской библиотеки SDK. Дополнительные сведения см. в [справочнике по API](API-references.md).

> [!NOTE]
> * Чтобы поделиться определенными частями приложения для промежуточной подготовки, используйте Teams версии манифеста 1.12 или более поздней.
> * Совместное использование определенных частей приложения на этапе поддерживается только для Teams настольных компьютеров.
