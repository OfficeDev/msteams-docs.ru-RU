# <a name="teams-toolkit-overview"></a>Teams набор средств Обзор

> [!NOTE]
> В настоящее время эта функция доступна только **для предварительного просмотра общедоступных** разработчиков.

Teams набор средств позволяет создавать, отлаговка и развертывать Teams приложение прямо Visual Studio Code. Разработка приложения с помощью инструментария имеет преимущества:

- Интегрированная идентификация
- Доступ к облачному хранилище
- Данные из Microsoft Graph
- Службы Azure и Microsoft 365 с подходом нулевой конфигурации

Teams набор средств все средства, необходимые для создания Teams приложения в одном месте.

Для Teams разработки приложения, аналогично Teams набор средств для Visual Studio Code, можно использовать [CLI-инструмент](https://github.com/OfficeDev/TeamsFx/blob/dev/docs/cli/user-manual.md), состоящий из набор средств`teamsfx`.

## <a name="user-journey-of-teams-toolkit"></a>Пользовательский путь Teams набор средств

Teams набор средств автоматизирует ручную работу и обеспечивает большую интеграцию ресурсов Teams Azure. На следующем изображении Teams набор средств путь пользователя:

:::image type="content" source="../assets/images/teams-toolkit-v2/teams toolkit fundamentals/teams-toolkit-user-journey.png" alt-text="Teams набор средств путешествие пользователя" border="true":::

Основные вехи этого путешествия:

1. Начните с создания нового проекта или примера Teams приложения.
1. Добавление возможностей или редактирование файла манифеста по мере необходимости.
1. Используйте Microsoft 365 учетную запись для создания и отлаговки Teams приложения.
1. Используйте учетную запись Azure для обеспечения и развертывания приложения в облаке.
1. Опубликуйте приложение Teams.

## <a name="accounts"></a>Учетные записи

Чтобы разработать приложение Teams, вам потребуется по крайней Microsoft 365 учетная запись с действительной подпиской. Если вы хотите, чтобы в Azure были свои дополнительные ресурсы, также необходима учетная запись Azure. Teams набор средств поддерживает интегрированный опыт для регистрации, обеспечения и развертывания ресурсов Azure. Перед [началом можно создать бесплатную учетную запись Azure](https://azure.microsoft.com/free/) .

## <a name="deployment"></a>Развертывание

Во время или после разработки убедитесь в том, что приложение Teams и опубликовать его до того, как оно будет доступно пользователям.

### <a name="provision-in-the-cloud"></a>Подготовка в облаке

Он интегрируется с менеджером ресурсов Azure, который позволяет обеспечить ресурсы Azure, которые нужны вашему приложению для подхода к коду.

### <a name="deploy-to-the-cloud"></a>Развертывание в облаке

 Это поможет вам развернуть исходный код в Azure.

### <a name="publish-to-teams"></a>Публикация Teams

После создания приложения вы можете распространить приложение в различные области, такие как отдельные, группы, организации или любой другой. Публикация Teams поможет вам опубликовать разработанную приложение.

### <a name="cicd-guide"></a>Руководство по CI/CD

Это помогает автоматизировать рабочий процесс разработки при создании Teams приложения. Руководство по ci/CD предоставляет инструменты и шаблоны для начала работы при настройке конвейеров ci или CD.

#### <a name="teamsfx-cli"></a>TeamsFx CLI

Это интерфейс командной строки на основе текста, который ускоряет Teams разработку приложений, а также включает сценарий CI/CD, в котором можно интегрировать CLI в скрипты для автоматизации.

#### <a name="teamsfx-sdk"></a>Пакет SDK TeamsFx

Это позволяет сократить задачи реализации удостоверений и доступа к облачным ресурсам к однострочному заявлению с нулевой конфигурацией.
