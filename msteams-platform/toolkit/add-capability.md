# <a name="add-capabilities-to-teams-apps"></a>Добавление возможностей в приложения Teams

Добавление возможностей с помощью набора средств Teams позволяет включить дополнительные функции в существующее приложение Microsoft Teams. Преимущество добавления дополнительных возможностей заключается в том, что вы можете добавить в приложение дополнительные функции, автоматически добавляя исходные коды с помощью набора средств Teams. Вы также можете выбрать различные возможности в зависимости от проекта, созданного в приложении Teams. В следующей таблице перечислены возможности приложений Teams:

|Возможность|Описание|Другие поддерживаемые возможности|
|--------|-------------|-----------------|
|**Базовое приложение Teams**|              |
| Tab |  Вкладки — это простые HTML-теги, которые относятся к доменам, объявленным в манифесте приложения. Вкладки можно добавлять как часть канала в команде, групповом чате или личном приложении для отдельного пользователя.|Вкладка, бот уведомлений, командный бот, бот, бот, расширение сообщений|
|Вкладка SPFx| Приложения для вкладок SPFx размещаются в Microsoft 365 и поддерживают разработку и размещение клиентского решения SPFx.|Нет|
|Вкладка с поддержкой единого входа|Вы можете создать приложение-вкладку с поддержкой единого входа, которое позволяет пользователю с функцией единого входа.|Вкладка с поддержкой единого входа, бот уведомлений, бот команд, бот, расширение сообщений|
| Bot |  Боты помогают взаимодействовать с веб-службой с помощью текстовых, интерактивных карточек и модулей задач.|Расширение сообщений, вкладка с поддержкой единого входа, вкладка|
| Расширение для сообщений | Расширения сообщений помогают взаимодействовать с веб-службой с помощью кнопок и форм в клиенте Microsoft Teams.|Бот, вкладка с поддержкой единого входа, вкладка|
|**Приложение Teams на основе сценариев**|             |
| Бот уведомлений | Бот уведомлений заблаговременно отправляет сообщения в канале Teams, групповом чате или личном чате. Вы можете активировать бот уведомлений с помощью HTTP-запроса, например карточек или текстовых сообщений. |Вкладка с поддержкой единого входа, вкладка|
| Командный бот | Командный бот позволяет автоматизировать повторяющиеся задачи с помощью бота команд. Он реагирует на простые команды, отправленные в чатах с помощью адаптивных карточек. |Вкладка с поддержкой единого входа, вкладка|
| Бот рабочего процесса| Бот рабочего процесса позволяет пользователям взаимодействовать с адаптивной карточкой, включенной функцией обработчика действий адаптивной карточки в приложении бота рабочего процесса.|Вкладка с поддержкой единого входа, вкладка|

> [!NOTE]
> Вкладки можно добавить до 16 экземпляров. Что касается бота и расширения сообщений, вы можете добавить по одному для каждого экземпляра за раз.

## <a name="add-capabilities"></a>Добавление возможностей

Вы можете добавить возможности следующими методами:

* [Использование набора средств Teams в Microsoft Visual Studio Code](#using-teams-toolkit-in-microsoft-visual-studio-code)
* [Использование палитры команд](#using-the-command-palette)
* [Использование Интерфейса командной строки TeamsFx](#using-teamsfx-cli)

### <a name="using-teams-toolkit-in-microsoft-visual-studio-code"></a>Использование набора средств Teams в Microsoft Visual Studio Code

   1. Откройте **Visual Studio Code**.
   1. Выберите **Набор средств Teams** на панели действий.
   1. Выберите **Добавить компоненты** в разделе **РАЗРАБОТКА**.

       :::image type="content" source="~/assets/images/teams-toolkit-v2/manual/select-feature123.png" alt-text="Добавление возможностей из набора средств Teams":::

      > [!NOTE]
      > После успешного добавления возможностей в приложение Teams необходимо подготовить для каждой среды.

### <a name="using-the-command-palette"></a>Использование палитры команд

   1. Выберите **Просмотр** > **палитры команд...** или **CTRL+SHIFT+P**.

      :::image type="content" source="../assets/images/teams-toolkit-v2/manual/add-capabilities-command-palette.png" alt-text="Добавление возможностей из команды palatte":::

   1. Введите **Teams: добавление компонентов**.
   1. Нажмите клавишу ВВОД.

      :::image type="content" source="~/assets/images/teams-toolkit-v2/manual/teams-add-features.png" alt-text="Добавление возможностей с помощью палитры команд.":::

   1. Во всплывающем окне выберите возможности, которые необходимо добавить в проект.

       :::image type="content" source="~/assets/images/teams-toolkit-v2/manual/notification-add-capabilities.png" alt-text="Уведомления":::

### <a name="using-teamsfx-cli"></a>Использование Интерфейса командной строки TeamsFx

* Измените каталог на **папку проекта**.
* В следующей таблице перечислены возможности и необходимые команды.

  |Возможности и сценарий| Команда|
  |-----------------------|----------|
  |Добавление бота уведомлений |`teamsfx add notification`|
  |Добавление бота команд |`teamsfx add command-and-response`|
  |Добавление вкладки с поддержкой единого входа |`teamsfx add sso-tab`|
  |Добавление вкладки |`teamsfx add tab`|
  |Добавление бота |`teamsfx add bot`|
  |Добавление расширения сообщений |`teamsfx add message extension`|

## <a name="changes-after-adding-capabilities"></a>Изменения после добавления возможностей

В следующей таблице показаны изменения, которые можно увидеть в файлах приложения при добавлении возможностей:

|Добавление возможности|Описание| Изменения|
|------------|------------------------|---------|
|Бот, расширение сообщений и вкладка|Включает в проект шаблон бота hello **world**&nbsp;или приложения вкладки.|Код шаблона внешнего бота или вкладки добавляется во вложенную папку с путем `yourProjectFolder/bot` или `yourProjectFolder/tab` соответственно.|
| Бот, расширение сообщений и вкладка |Включает необходимые скрипты для Visual Studio Code и выполняется при локальной отладке приложения. |Файлы `launch.json` и `task.json` в папке `.vscode` обновляются.|
| Бот и расширение сообщений|Включает сведения, связанные с ботом или вкладками, в файл манифеста, который представляет приложение на платформе Teams.|Файл`manifest.template.json` в `templates/appPackage` папке обновляется, который содержит сведения, связанные с вкладками, в файле манифеста, представляющего приложение на платформе Teams. Изменения отображаются в идентификаторе бота, областях бота и командах, на которые может отвечать бот hello world или приложение tab.|
|Tab|Включает сведения, связанные с ботом или вкладками, в файл манифеста, который представляет приложение на платформе Teams.|Файл `manifest.template.json` в `templates/appPackage` папке обновляется, который содержит сведения, связанные с вкладками, в файле манифеста, представляющего приложение на платформе Teams. Изменения отображаются на настраиваемых и статических вкладках, а также в областях вкладок.|
|Бот, расширение сообщений и вкладка|Включает сведения, связанные с&nbsp;ботом или вкладками, в teamsfx и подготавливает файлы, которые предназначены для интеграции функций Azure.|Файлы в разделе `templates/azure/teamsfx` обновляются, а `templates/azure/provision/xxx`BICEP-файлы создаются повторно.|
|Бот, расширение сообщений и вкладка|Гарантирует, что в проекте заданы правильные конфигурации для новых добавленных возможностей.|Файлы в разделе `.fx/config` повторно создаются|
