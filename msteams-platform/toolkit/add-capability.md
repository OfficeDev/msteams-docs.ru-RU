# <a name="add-capabilities-to-teams-apps"></a>Добавление возможностей в приложения Teams

Добавление возможностей с помощью Набора средств Teams позволяет добавить дополнительные функции в существующее приложение Microsoft Teams. Преимущество добавления дополнительных возможностей заключается в том, что вы можете добавить дополнительные функции в приложение, автоматически добавляя исходные коды с помощью Набора средств Teams. Вы также можете выбрать различные возможности в зависимости от проекта, созданного в приложении Teams. В следующей таблице перечислены возможности приложения Teams:

|Возможность|Описание|Другие поддерживаемые возможности|
|--------|-------------|-----------------|
|**Базовое приложение Teams**|              |
| Tab |  Вкладки — это простые HTML-теги, которые ссылаются на домены, объявленные в манифесте приложения. Вкладки можно добавлять в составе канала внутри команды, группового чата или личного приложения для отдельного пользователя.|Tab, notification bot, command bot, bot, message extension|
|Вкладка SPFx| Приложения на вкладке SPFx размещаются в Microsoft 365 и поддерживают разработку и размещение клиентского решения SPFx.|Нет|
|Вкладка с поддержкой единого входа|Вы можете создать приложение вкладки с поддержкой единого входа, которое позволяет пользователю использовать функцию единого входа.|Вкладка с поддержкой единого входа, бот уведомлений, командный бот, бот, расширение сообщения|
| Bot |  Боты помогают взаимодействовать с веб-службой с помощью текстовых, интерактивных карточек и модулей задач.|Расширение сообщения, вкладка с поддержкой единого входа, вкладка|
| Расширение для сообщений | Расширения сообщений помогают взаимодействовать с веб-службой с помощью кнопок и форм в клиенте Microsoft Teams.|Бот, вкладка с поддержкой единого входа, вкладка|
|**Приложение Teams на основе сценариев**|             |
| Бот уведомлений | Бот уведомлений заблаговременно отправляет сообщения в канале Teams, групповом чате или личном чате. Бот уведомлений можно активировать с помощью HTTP-запроса, например карточек или текстовых сообщений. |Вкладка с поддержкой единого входа, вкладка|
| Командный бот | Командный бот позволяет автоматизировать повторяющиеся задачи с помощью командного бота. Он отвечает на простые команды, отправляемые в чатах с помощью адаптивных карточек. |Вкладка с поддержкой единого входа, вкладка|

> [!NOTE]
> Вы можете добавить вкладки до 16 экземпляров. Что же до бота и расширения сообщений, вы можете добавить по одному для каждого экземпляра за раз.

## <a name="add-capabilities"></a>Добавление возможностей

Возможности можно добавить следующими способами:

* [Использование набора средств Teams в Microsoft Visual Studio Code](#using-teams-toolkit-in-microsoft-visual-studio-code)
* [Использование палитры команд](#using-the-command-palette)
* [Использование Командной строки TeamsFx](#using-teamsfx-cli)

### <a name="using-teams-toolkit-in-microsoft-visual-studio-code"></a>Использование набора средств Teams в Microsoft Visual Studio Code

   1. Откройте **Visual Studio Code**.
   1. Выберите **Набор средств Teams на** панели действий.
   1. Выберите **"Добавить компоненты"** в разделе **"РАЗРАБОТКА"**.

       :::image type="content" source="~/assets/images/teams-toolkit-v2/manual/select-feature123.png" alt-text="Добавление возможностей из Набора средств Teams":::

      > [!NOTE]
      > После успешного добавления возможностей в приложение Teams необходимо подготовить каждую среду.

### <a name="using-the-command-palette"></a>Использование палитры команд

   1. Выберите **"Просмотреть** > **палитру команд"...** или **CTRL+SHIFT+P**.

      :::image type="content" source="../assets/images/teams-toolkit-v2/manual/add-capabilities-command-palette.png" alt-text="Добавление возможностей из командной таблицы":::

   1. **Введите Teams: добавление функций**.
   1. Нажмите клавишу ВВОД.

      :::image type="content" source="~/assets/images/teams-toolkit-v2/manual/teams-add-features.png" alt-text="Добавление возможностей с помощью палитры команд.":::

   1. Во всплывающем окне выберите возможность, которую необходимо добавить в проект.

       :::image type="content" source="~/assets/images/teams-toolkit-v2/manual/notification-add-capabilities.png" alt-text="Уведомления":::

### <a name="using-teamsfx-cli"></a>Использование Командной строки TeamsFx

* Измените каталог на **папку проекта**.
* В следующей таблице перечислены возможности и необходимые команды:

  |Возможности и сценарии| Команда|
  |-----------------------|----------|
  |Добавление бота уведомлений |`teamsfx add notification`|
  |Добавление бота команды |`teamsfx add command-and-response`|
  |Добавление вкладки с поддержкой единого входа |`teamsfx add sso-tab`|
  |Добавление вкладки |`teamsfx add tab`|
  |Добавление бота |`teamsfx add bot`|
  |Добавление расширения сообщения |`teamsfx add message extension`|

## <a name="changes-after-adding-capabilities"></a>Изменения после добавления возможностей

В следующей таблице показаны изменения, которые можно увидеть в файлах приложения при добавлении возможностей:

|Добавление возможности|Описание| Изменения|
|------------|------------------------|---------|
|Бот, расширение сообщения и вкладка|Включает в проект **шаблон приложения hello world**&nbsp;или приложения табуляции.|Интерфейсный бот или код шаблона вкладки добавляется во вложенную папку с путем `yourProjectFolder/bot` или `yourProjectFolder/tab` соответственно.|
| Бот, расширение сообщения и вкладка |Включает необходимые скрипты для Visual Studio Code и выполняется, когда требуется выполнить локальную отладку приложения. |Файлы `launch.json` и `task.json` папка `.vscode` обновляются.|
| Расширение бота и сообщения|Включает сведения о боте или вкладке в файл манифеста, представляющий приложение на платформе Teams.|Файл`manifest.template.json` в `templates/appPackage` папке обновляется, что включает сведения, связанные с вкладками, в файл манифеста, представляющий приложение на платформе Teams. Изменения отображаются в идентификаторе бота, областях бота и командах, на которые может отвечать бот hello world или приложение табуляции.|
|Tab|Включает сведения о боте или вкладке в файл манифеста, представляющий приложение на платформе Teams.|Файл `manifest.template.json` в `templates/appPackage` папке обновляется, что включает сведения, связанные с вкладками, в файл манифеста, представляющий приложение на платформе Teams. Изменения отображаются на настраиваемых и статических вкладах, а также в областях вкладок.|
|Бот, расширение сообщения и вкладка|Включает сведения о боте или вкладке&nbsp;в teamsfx и файлы подготовки, которые предназначены для интеграции функций Azure.|Файлы в `templates/azure/teamsfx` разделе обновляются, а `templates/azure/provision/xxx`BICEP-файлы создаются повторно.|
|Бот, расширение сообщения и вкладка|Гарантирует, что в проекте настроены правильные конфигурации для вновь добавленной возможности.|Файлы в разделе `.fx/config` повторно создаются|
