# <a name="glossary"></a>Глоссарий

Общие термины и определения, используемые в документации для разработчиков Teams.

## <a name="a"></a>A

| Термин | Определение |
| --- | --- |
| [Команда действия](../messaging-extensions/how-to/action-commands/define-action-command.md) | Тип приложения расширения для обмена сообщениями, использующий всплывающее окно для сбора или отображения информации. <br>**См. также**: [Расширение для обмена сообщениями](#m); [Команды поиска](#s) |
| [Адаптивные карточки](../task-modules-and-cards/what-are-cards.md) | Фрагмент содержимого с действиями, добавленный в беседу ботом или расширением для обмена сообщениями. Используйте текст, графику и кнопки с помощью этих карточек для разнообразия коммуникаций. |
| [Анонимный пользователь](../apps-in-teams-meetings/meeting-app-extensibility.md#user-types-in-a-meeting) | Тип участника собрания Teams, у которого нет удостоверения Azure AD и федерации с клиентом. Такие пользователи аналогичны внешним пользователям на собрании.<br>**См. также**: [федеративный пользователь](#f) |
| [Каталог приложений](../toolkit/publish.md). | Сайт, на котором хранятся приложения SharePoint и Office для внутреннего использования в организации. <br>**См. также**: [SPFx](#s) |
| [Манифест приложения](../resources/schema/manifest-schema.md) | Манифест приложения Teams описывает, каким образом приложение интегрируется с продуктом Microsoft Teams. Манифест должен соответствовать [схеме манифеста](https://developer.microsoft.com/json-schemas/teams/v1.11/MicrosoftTeams.schema.json). |
| [Пакет приложения](../concepts/build-and-test/apps-package.md) | Пакет приложения Teams — это ZIP-файл, содержащий файл манифеста приложения, цветной значок и контурный значок. |
| [Разрешение приложения](../concepts/device-capabilities/browser-device-permissions.md#enable-apps-device-permissions) | Параметр в приложении Teams, чтобы включить разрешения устройства. Он доступен только в том случае, если в файле манифеста приложения объявлено, что приложению нужны разрешения устройства. <br> **См. также**: разрешения устройства |
| [Область приложения](../concepts/design/app-structure.md) | Область в Teams, где люди могут использовать ваше приложение. Приложения могут иметь одну или несколько областей, включая личную область, каналы, чаты и собрания. Приложение Teams может существовать в разных областях. |
| [App Studio](../concepts/build-and-test/app-studio-overview.md) | Приложение для создания или интеграции ваших собственных приложений Microsoft Teams. Теперь оно превратилось в портал разработчика. <br> Если до сих пор вы использовали App Studio, рекомендуем попробовать Портал разработчика для настройки приложений Teams, распространения их и управления ими. App Studio станет нерекомендуемой к употреблению не позднее 30 июня 2022 г.<br> **См. также**: [портал разработчика](#d) |
| Область приложений | Область приложений расположена на нижней панели мобильного приложения Teams. В ней собраны все приложения, которые открыты, но в настоящее время не используются или не активны. <br>**См. также**: [мобильная версия Teams](#t) |
| [Ресурс Azure](../toolkit/provision.md) | Служба, доступная через Azure, которую ваше приложение Teams может использовать для развертывания Azure. Это могут быть учетные записи хранения, веб-приложения, базы данных и т. д. |
| [Azure Active Directory](../tabs/how-to/authentication/auth-tab-aad.md) | Облачная служба управления идентификацией и доступом Майкрософт. Это помогает прошедшим проверку подлинности пользователям получать доступ к внутренним и внешним ресурсам Azure. |
| [Проверка подлинности](../concepts/authentication/authentication.md) | Процесс проверки доступа пользователей для использования вашего приложения. Это можно сделать с помощью API Microsoft Graph или веб-проверки подлинности. <br> **См. также**: [поставщики удостоверений](#i); [единый вход](#s) |
| [Поток проверки подлинности](../concepts/authentication/authentication.md#web-based-authentication-flow) | В Teams существует два потока проверки подлинности для пользователей с целью использования приложения: веб-проверка подлинности и поток OAuthPrompt. |

## <a name="b"></a>Б

| Термин | Определение |
| --- | --- |
| [Blazor](../get-started/get-started-overview.md) | Бесплатная веб-платформа с открытым кодом, которая позволяет разработчикам создавать веб-приложения с использованием C# и HTML. Разрабатывается корпорацией Майкрософт. |
| [Bicep](../toolkit/provision.md) | Декларативный язык, который означает, что элементы могут отображаться в любом порядке. В отличие от императивных языков, порядок элементов не влияет на обработку развертывания. |
| [Бот](../bots/what-are-bots.md) | Бот — это приложение, которое выполняет запрограммированные повторяющиеся задачи. <br> **См. также**: [бот для общения](#c); [чат-бот](#c) |
| [Эмулятор бота](../bots/how-to/debug/locally-with-an-ide.md#use-the-bot-emulator) | Классическое приложение, позволяющее тестировать и отлаживать боты локально или удаленно. |
| [Bot Framework](../bots/bot-features.md) | Многофункциональный пакет SDK для создания ботов с использованием C#, Java, Python и JavaScript. Если у вас есть бот, основанный на Bot Framework, вы можете изменить его для работы в Teams. |

Возможность приложения используется для расширения Teams под потребности вашего приложения. Приложение может иметь одну или несколько основных возможностей, таких как вкладка, бот и расширение для обмена сообщениями. <br>**См. также**: [возможность устройства](#d); [возможность мультимедиа](#m) |
| [Чат-бот](../bots/how-to/conversations/conversation-basics.md) | Бот также называется чат-ботом или ботом для общения. Это приложение, которое выполняет простые и повторяющиеся задачи пользователей, например в службах обслуживания или поддержки клиентов. <br> **См. также**: [бот для общения](#c) |
| Канал | Единое место для команды с целью обмена сообщениями, инструментами и файлами. Вы можете использовать канал для командной работы и общения. <br>**См. также**: [беседа](#c) |
| [Секрет клиента](../bots/how-to/authentication/add-authentication.md) | Секрет клиента/пароль или пара открытого и закрытого ключей, которая является сертификатом. Это необязательно для собственных приложений. <br> **См. также**: [бот](#b) |
| [Облачные ресурсы](../toolkit/add-resource.md) | Служба, доступная в облаке через Интернет, которую может использовать приложение Teams. Это могут быть учетные записи хранения, веб-приложения, базы данных и т. д. |
| [Приложение для совместной работы](../concepts/extensibility-points.md) | Приложение с возможностями для работы пользователя в рабочей области взаимодействия с другими пользователями. <br> **См. также**: [отдельное приложение](#s) |
| [Расширение создания](../resources/schema/manifest-schema.md#composeextensions) | Свойство в манифесте приложения (`composeExtensions`), ссылающееся на возможность расширения для обмена сообщениями. Оно используется, когда вашему расширению необходима либо проверка подлинности, либо настройка для продолжения. <br>**См. также**: [Манифест приложения](#a); [Расширение для обмена сообщениями](#m) |
| [Командное поле](../resources/schema/manifest-schema.md) | Тип контекста в манифесте приложения (`commandBox`), который можно настроить для вызова расширения для обмена сообщениями из командного поля Teams. |
| [Соединитель](../webhooks-and-connectors/what-are-webhooks-and-connectors.md) | Он позволяет пользователям подписаться на получение уведомлений и сообщений от веб-служб. Соединители предоставляют службе конечную точку HTTPS для публикации сообщений в каналах Teams, обычно в виде карточек. <br> **См. также**: [веб-перехватчик](#w) |
| Беседа | Серия сообщений, отправляемых между вашим приложением Microsoft Teams (вкладка или бот) и одним или несколькими пользователями. Беседа может иметь три области: канал, личный и групповой чат. <br>**См. также**: [приватный чат](#o); [групповой чат](#g); [канал](#c) |
| [Бот для общения](../bots/how-to/conversations/conversation-messages.md) |  Он позволяет пользователю взаимодействовать с веб-службой с помощью текста, интерактивных карточек и модулей задач. <br>**См. также:** [чат-бот](#c) |
