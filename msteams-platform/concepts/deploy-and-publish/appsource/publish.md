# <a name="submit-your-app-to-appsource"></a>Отправка приложения в AppSource

## <a name="teams-app-submission"></a>Отправка приложений teams

Сделайте приложение доступным в каталоге приложений Microsoft Teams и в Интернете, опубликовав его [в AppSource.](https://appsource.microsoft.com) На высоком уровне процесс отправки приложения в AppSource следующим образом:

1. Разработайте приложение, следуя [рекомендациям по проектированию.](~/concepts/design/understand-use-cases.md) Вкладки должны следовать рекомендациям по разработке вкладок как для настольных, [так и для веб-и мобильных](~/tabs/design/tabs.md) [устройств.](~/tabs/design/tabs-mobile.md) Боты должны следовать рекомендациям [по разработке ботов.](~/bots/design/bots.md)
1. Убедитесь, что ваше приложение соответствует политикам проверки [приложений](/legal/marketplace/certification-policies) для Microsoft Teams. 
1. Проверьте приложение с помощью средства [проверки манифеста.](prepare/submission-checklist.md#teams-app-validation-tool)
1. Настройка учетной [записи разработчика](/office/dev/store/open-a-developer-account) в [Центре партнеров.](https://support.microsoft.com/help/4499930/partner-center-overview) *См.* [также, как создать учетную запись Центра партнеров](#how-do-i-create-a-partner-center-account) в разделе FAQ.
1. Подготовка приложения к отправке, следуя [контрольным списком отправки.](prepare/submission-checklist.md)
1. Просмотрите [наиболее неудачные тестовые случаи для более быстрого](prepare/frequently-failed-cases.md)утверждения качества приложения.
1. Отправка пакета [в AppSource через Центр партнеров.](/office/dev/store/use-partner-center-to-submit-to-appsource)
1. Отслеживание процесса утверждения на панели мониторинга Центра партнеров. *См.* [обзор центра партнеров.](https://support.microsoft.com/help/4499930/partner-center-overview)
1. После отправки просмотрите руководство по [обслуживанию и поддержке опубликованного приложения.](post-publish/overview.md)

>[!NOTE]
>
>- Приложение Teams должно быть мобильным и не соответствовать требованиям к [upsell](~/concepts/deploy-and-publish/appsource/prepare/frequently-failed-cases.md#-mobile-responsiveness-no-direct-upsell-or-payment) на мобильной ОС (iOS и Android). 
>- Если приложение Teams содержит бот, необходимо соблюдать Кодекс поведения разработчика [ботов.](https://aka.ms/bf-conduct)
>- Если ваше приложение содержит соединители Office 365, могут применяться дополнительные условия. См. [панель мониторинга разработчиков соединители](https://aka.ms/connectorsdashboard) [и соглашение разработчика приложений.](https://sellerdashboard.microsoft.com/Assets/Content/Agreements/Office_Store_Seller_Agreement_20120927.htm)
>- Чтобы сделать приложение доступным для пользователей government Community Cloud (GCC) и избежать дублирования списков приложений в магазине, процесс и поток auth должны идентифицировать пользователя и маршрутировать его в указанный или ожидаемый URL-адрес контента для пользователей GCC.

## <a name="how-do-i-create-a-partner-center-account"></a>Как создать учетную запись Центра партнеров?

Существует два способа создания учетной записи Центра партнеров:

- Если вы не в Центре партнеров и не имеете учетную запись в сети Майкрософт, создайте учетную запись на странице регистрации [Центра партнеров.](/office/dev/store/open-a-developer-account#create-an-account-using-an-existing-partner-center-enrollment)
- Если вы уже зарегистрированы в партнерской сети, создайте учетную запись непосредственно в Центре партнеров с помощью существующей [страницы регистрации.](/office/dev/store/)

## <a name="what-if-i-cannot-find-my-office-store-account-in-partner-center"></a>Что делать, если я не могу найти свою учетную запись Office Store в Центре партнеров?

Откройте билет [поддержки Центра партнеров](https://partner.microsoft.com/support/v2/?stage=1) и выберите следующее из выпадаемого меню:

| Меню | Вариант |
| -------   | -------  |
|Категория| Коммерческий маркетплейс|
| Тема | Общая справка по рынку и вопросы о том, как их задать |
| Subtopic| Надстройка Office |

Посетите [страницу поддержки издателей для](https://aka.ms/marketplacepublishersupport) поиска темы проблемы и поиска руководства. Если предоставленные рекомендации не помогают, подними билет [поддержки Центра партнеров.](/azure/marketplace/partner-center-portal/support#how-to-open-a-support-ticket)

Номер телефона имеет три части, код страны, код области и номер телефона. Если номер телефона не содержит код области, оставьте второе поле пустым и заполните третье поле.

## <a name="why-do-i-receive-the-message-this-account-is-not-publish-eligible-when-i-try-to-submit-my-add-in-through-partner-center"></a>Почему я получаю сообщение "Эта учетная запись не может публиковаться", когда я пытаюсь отправить надстройки через Центр партнеров?

Проверьте состояние проверки учетной записи на панели мониторинга Центра [партнеров.](https://partner.microsoft.com/dashboard) Выберите **Параметры**, значок передач в верхнем правом углу оболочки заголовки страницы. Выберите **параметры**  =>  **учетной**   =>  **записи учетной записи разработчика.**

Состояние каждого необходимого шага, например владения электронной почтой, проверки занятости и проверки **бизнеса,** отображается в процессе проверки учетной записи. После завершения процесса проверки состояние проверки вашей регистрации на странице профиля изменяется с ожидающих *до* *утверждения.* Этапы процесса больше не отображаются.

## <a name="what-is-verified-in-the-partner-center-account-verification-process-and-how-to-respond"></a>Что проверяется в процессе проверки учетной записи Центра партнеров и как реагировать?
Существует три области проверки: **владение электронной почтой,** **занятость** и **бизнес.** Дополнительные сведения о процессе проверки см. в сайте [What is verified and how to respond.](/partner-center/verification-responses#what-is-verified-and-how-to-respond)
Если вы основной контакт, глобальный администратор или администратор учетной записи, перейдите в профиль партнера, чтобы отслеживать состояние проверки и отслеживать ход выполнения.

После завершения процесса проверки состояние проверки вашей регистрации на странице профиля изменяется с ожидающих *до* *утверждения.* После авторизации этапы процесса и их состояние больше не доступны на странице. Основной контакт получает сообщение электронной почты от Корпорации Майкрософт в течение нескольких дней после завершения проверки.

## <a name="my-account-verification-status-has-not-advanced-beyond-email-ownership-in-the-partner-center-how-should-i-proceed"></a>Мой статус проверки учетной записи не вышел за рамки владения электронной почтой в Центре партнеров. Как мне действовать?

В процессе **проверки владения** электронной почтой электронной почты на основной контактный адрес электронной почты отправляется сообщение проверки. Проверьте ваш основной почтовый ящик контакта для электронной почты maccount@ **<span>microsoft.com</span>** с необходимой строкой *темы Действие: Проверка* учетной записи электронной почты в Корпорации Майкрософт . Завершите процесс проверки электронной почты. Сообщение проверки отправляется на адрес электронной почты, указанный на странице параметры учетной записи в Центре партнеров.

> [!NOTE]
> * Ссылка проверки электронной почты действительна только в течение семи дней. 
> * Вы можете попросить нас повторно отправить сообщение электронной почты, посетив страницу профиля партнера и выбрав ссылку электронной почты проверки **Resend.**
> * Чтобы убедиться, что электронная почта получена, microsoft.com список электронной почты из безопасного домена и проверьте нежелательные папки электронной почты.

## <a name="how-do-i-get-further-support-for-my-account-related-issues"></a>Как получить дополнительную поддержку по связанным с учетной записью вопросам?

## <a name="ive-checked-my-mail-folders-and-havent-received-the-verification-email-what-must-i-do-next"></a>Я проверил почтовые папки и не получил электронной почты проверки. Что мне делать дальше?

Попробуйте следующее:
* Проверьте папку нежелательной или нежелательной почты.
* Очистить кэш браузера, перейдите на панель мониторинга учетной записи Центра партнеров и выберите ссылку электронной почты проверки **Resend,** чтобы отправить сообщение проверки на адрес электронной почты.
* Попробуйте получить доступ к ссылке **электронной почты проверки Resend** из другого браузера.
* Работайте с ИТ-отделом, чтобы убедиться, что электронные письма проверки не заблокированы сервером электронной почты.
* Настройте фильтр нежелательной почты сервера, чтобы разрешить или безопасно перечислять все электронные сообщения из **maccount@microsoft. <span></span> com**.

## <a name="ive-created-a-support-ticket-it-has-been-7-business-days-and-i-havent-received-an-update-where-can-i-get-additional-help"></a>Я создал билет поддержки, это было 7 дней работы, и я не получил обновления. Где можно получить дополнительную помощь?

Отправьте сообщение **<teamsubm@microsoft.com>** электронной почты со следующими сведениями:

* **Subject Line.** Проблема учетной записи центра <App_Name> для <App_Name> (укажите имя приложения).
* **Тело электронной почты:**
    * Номер билета поддержки
    * Ваш ИД продавца
    * Снимок экрана проблемы (по возможности)
