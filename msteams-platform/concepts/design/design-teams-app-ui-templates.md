# <a name="designing-your-microsoft-teams-app-with-ui-templates"></a>Проектирование приложения Microsoft Teams с помощью шаблонов пользовательского интерфейса

Быстрее проектировать приложение Microsoft Teams с помощью шаблонов пользовательского интерфейса. Шаблоны — это коллекция компонентов на основе пользовательского интерфейса Fluent, которые работают в распространенных случаях использования Teams, что дает вам больше времени, чтобы найти оптимальный интерфейс для пользователей.

## <a name="getting-started-with-tools-and-samples"></a>Начало работы со средствами и примерами

Следующие ресурсы помогут вам разработать и разработать приложение с помощью шаблонов пользовательского интерфейса.

Получите шаблоны пользовательского интерфейса для разработки приложения из комплекта пользовательского интерфейса Microsoft Teams, который также содержит подробную информацию об использовании, анатомии, возможностях и практических методиках.

> [!div class="nextstepaction"]
> [Get the UI kit (Figma)](https://www.figma.com/community/file/916836509871353159)

### <a name="microsoft-teams-ui-library"></a>Библиотека пользовательского интерфейса Microsoft Teams

Просмотр и тестирование отдельных шаблонов пользовательского интерфейса Teams и связанных компонентов в браузере.

> [!div class="nextstepaction"]
> [Попробуйте библиотеку пользовательского интерфейса (игровая библиотека)](https://dev-int.teams.microsoft.com/storybook/main/index.html)

Импортировать эти шаблоны и связанные компоненты непосредственно в проект приложения Teams.

Установите пример приложения, чтобы увидеть, как шаблоны пользовательского интерфейса выглядят и работают в контекстах Teams.

## <a name="dashboard"></a>Информационная панель

Панель мониторинга отображает различные типы содержимого в центральном расположении (личное приложение или вкладка Teams). Пользователи должны иметь возможность настраивать по крайней мере некоторые из того, что они видят на панели мониторинга.

:::image type="content" source="../../assets/images/ui-templates/dashboard.png" alt-text="В примере показан шаблон пользовательского интерфейса панели мониторинга." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Анализ данных
* Показатели отчета
* Организация разных сведений в одном месте

## <a name="form"></a>Form

* Профили пользователей
* Параметры
* Коллекция пользовательских входных данных

Вы можете разработать потоки входов в приложения для различных контекстов Teams и поставщиков удостоверений. В следующем примере включается единый вход ( SSO), который рекомендуется для простейшей проверки подлинности.

## <a name="task-board"></a>Доска задач

Доска задач, иногда называемая доской канан или плаваком, — это коллекция карточек, часто используемых для отслеживания состояния элементов или билетов. Его также можно использовать для сортировки любого типа контента по категориям. Карточки можно редактировать и перемещать между столбцами.

:::image type="content" source="../../assets/images/ui-templates/task-board.png" alt-text="В примере показан шаблон пользовательского интерфейса доски задач." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* управление проектами; Назначение задач и состояние отслеживания
* Многая мягок. Добавление идей в различных категориях
* Сортировка упражнений. Организация любых сведений по сегментам

## <a name="data-visualization"></a>Визуализация данных

Для стека и организации визуализации данных на одной странице можно использовать разные размеры карт (один, двойной и полный). Карточки масштабются в форме макета столбца и заполняются пустыми пробелами.

:::image type="content" source="../../assets/images/ui-templates/data-viz.png" alt-text="В примере показан шаблон пользовательского интерфейса визуализации данных." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Отображение сложной информации
* Создание панели мониторинга

Мастер поможет пользователю выполнить задачу (например, выполнить настройку) на нескольких экранах.

## <a name="empty-state"></a>Пустое состояние

Пустой шаблон состояния можно использовать для многих сценариев, включая вход, первый запуск, сообщения об ошибках и многое другое. Он очень гибкий — адаптируется к использованию одного, нескольких или всех компонентов в следующем проекте.

:::image type="content" source="../../assets/images/ui-templates/empty-state.png" alt-text="В примере показан пустой шаблон пользовательского интерфейса состояния." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Вход
* Приветствия и первый запуск
* Сообщения об успешном успешном собраии
* Сообщения об ошибках

Область уведомлений — это выделенная область для отображения кратких важных сообщений, которые не требуют немедленного действия пользователя. Определенные цвета фона и значки связаны с определенными типами сообщений (см. ниже).

:::image type="content" source="../../assets/images/ui-templates/notification-bar.png" alt-text="В примере показаны шаблоны панели уведомлений." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Критические сообщения, ошибки и предупреждения
* Сообщения об успешном успешном собраии
* Информационные или рекламные сообщения

* Навигация

## <a name="stage"></a>Стадия

Этап позволяет пользователям открывать объект ( например, изображение, файл или веб-сайт) в Teams вместо открытия в другом приложении или браузере. Основной пример использования стадии — просмотр; поверхность не должна использоваться для сложных взаимодействий.

(Примечание по реализации. Создание стадии с помощью модуля больших [задач.)](../../task-modules-and-cards/task-modules/design-teams-task-modules.md)

:::image type="content" source="../../assets/images/ui-templates/stage.png" alt-text="В примере показан шаблон стадии." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Открытие сущности в Teams вместо другого приложения или браузера
* Мультимедиа в центре внимания или другое содержимое
