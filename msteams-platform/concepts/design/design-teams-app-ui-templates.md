# <a name="designing-your-microsoft-teams-app-with-ui-templates"></a>Проектирование приложения Microsoft Teams с помощью шаблонов пользовательского интерфейса

Быстрее проектировать приложение Microsoft Teams с помощью шаблонов пользовательского интерфейса. Шаблоны — это коллекция компонентов на основе пользовательского интерфейса Fluent, которые работают в распространенных вариантах использования Teams, что дает вам больше времени на определение оптимального интерфейса для пользователей.

## <a name="getting-started-with-tools-and-samples"></a>Начало работы со средствами и примерами

Следующие ресурсы помогут вам спроектировать и разработать приложение с помощью шаблонов пользовательского интерфейса.

### <a name="microsoft-teams-ui-kit"></a>Комплект разработчика для пользовательского интерфейса Microsoft Teams

Получите шаблоны пользовательского интерфейса для разработки приложения из пакета пользовательского интерфейса Microsoft Teams, который также содержит подробную информацию об использовании, структуре, специальных возможностях и рекомендациях.

> [!div class="nextstepaction"]
> [Получить комплект пользовательского интерфейса (Figma)](https://www.figma.com/community/file/916836509871353159)

### <a name="microsoft-teams-ui-library"></a>Библиотека пользовательского интерфейса Microsoft Teams

Просматривайте и тестируйте отдельные шаблоны пользовательского интерфейса Teams и связанные компоненты в браузере.

> [!div class="nextstepaction"]
> [Попробуйте библиотеку пользовательских интерфейсов (песочница)](https://dev-int.teams.microsoft.com/storybook/main/index.html)

Импортируйте эти шаблоны и связанные компоненты непосредственно в проект приложения Teams.

> [!div class="nextstepaction"]

## <a name="calendar"></a>Календарь

В Teams календарь — это место, в котором пользователь просматривает, планирует и управляет предстоящими и прошедшими событиями для себя или группы.

### <a name="top-use-cases"></a>Основные варианты использования

* Планирование собраний и событий
* Получение напоминаний о предстоящих собраниях и событиях.
* Просмотр расписаний

## <a name="dashboard"></a>Панель мониторинга

Панель мониторинга отображает различные типы содержимого в центральном расположении (например, в личном приложении или вкладке Teams). Пользователи должны иметь возможность настраивать по крайней мере некоторые из элементов, которые они видят на панели мониторинга.

### <a name="top-use-cases"></a>Основные варианты использования

* Анализ данных
* Метрики отчета
* Упорядочение разных сведений в одном месте.

## <a name="empty-state"></a>Пустое состояние

Шаблон пустого состояния можно использовать во многих сценариях, включая вход, интерфейсы первого запуска, сообщения об ошибках и многое другое. Он очень гибкий — адаптируется к использованию одного, нескольких или всех компонентов в следующем проекте.

### <a name="top-use-cases"></a>Основные варианты использования

* Вход
* Приветственные сообщения и интерфейсы первого запуска.
* Сообщения об успешном выполнении
* Сообщения об ошибках

## <a name="filter"></a>Filter

Фильтр позволяет уменьшить объем информации, которую вы видите, в зависимости от выбранных критериев. Вы можете включить фильтры с таблицами, списками, карточками и другими компонентами, которые упорядочивают содержимое.

### <a name="top-use-cases"></a>Основные варианты использования

Упорядочение содержимого в:

* Списки
* Таблицы
* Панели мониторинга
* Визуализация данных

:::image type="content" source="../../assets/images/ui-templates/filter.png" alt-text="В примере показан шаблон фильтра." border="false":::

## <a name="form"></a>Форма

Формы используются для сбора, проверки и отправки входных данных пользователя структурированным способом. Четкие метки и логические группировки полей ввода критически важны для удобного взаимодействия с пользователем.

### <a name="top-use-cases"></a>Основные варианты использования

* Вход
* Профили пользователей
* Settings
* Коллекция входных данных пользователя

## <a name="task-board"></a>Доска задач

Доска задач, иногда называемая канбан-доской или дорожками, — это коллекция карточек, часто используемых для отслеживания состояния рабочих элементов или билетов. Его также можно использовать для сортировки содержимого любого типа по категориям. Карточки можно изменять и перемещать между столбцами.

### <a name="top-use-cases"></a>Основные варианты использования

* управление проектами; Назначение задач и состояние отслеживания.
* Мозгового штурма. Добавление идей в разные категории.
* Упражнения по сортировке. Упорядочив любую информацию в контейнеры.
