# <a name="designing-your-microsoft-teams-app-with-ui-templates"></a>Проектирование приложения Microsoft Teams с помощью шаблонов пользовательского интерфейса

Быстрее создайте Microsoft Teams с помощью шаблонов пользовательского интерфейса. Шаблоны представляет собой набор компонентов на основе пользовательского интерфейса Fluent, которые работают в обычных случаях Teams использования, что дает вам больше времени, чтобы выяснить лучший опыт для ваших пользователей.

## <a name="getting-started-with-tools-and-samples"></a>Начало работы с инструментами и образцами

Следующие ресурсы помогут вам спроектировать и разработать приложение с помощью шаблонов пользовательского интерфейса.

### <a name="microsoft-teams-ui-kit"></a>Комплект разработчика для пользовательского интерфейса Microsoft Teams

Захватите шаблоны пользовательского интерфейса для дизайна приложения из набора пользовательского интерфейса Microsoft Teams, который также включает обширную информацию об использовании, анатомии, доступности и передовом опыте.

> [!div class="nextstepaction"]
> [Получить комплект пользовательского интерфейса (Фигма)](https://www.figma.com/community/file/916836509871353159)

### <a name="microsoft-teams-ui-library"></a>Microsoft Teams Библиотека пользовательского интерфейса

Просмотр и тестирование отдельных Teams пользовательского интерфейса и связанных с ними компонентов в браузере.

> [!div class="nextstepaction"]
> [Попробуйте библиотеку пользовательского интерфейса (площадка)](https://dev-int.teams.microsoft.com/storybook/main/index.html)

Импорт этих шаблонов и связанных с ними компонентов непосредственно в ваш Teams проекта приложения.

> [!div class="nextstepaction"]
> [Получить библиотеку пользовательского интерфейса (GitHub)](https://github.com/OfficeDev/microsoft-teams-ui-component-library)

### <a name="sample-app"></a>Пример приложения

Установите пример приложения, чтобы увидеть, как выглядят и ведут себя шаблоны пользовательского интерфейса Teams контексте.

> [!div class="nextstepaction"]
> [Получить образец приложения (GitHub)](https://github.com/OfficeDev/Microsoft-Teams-Samples/tree/main/samples/tab-ui-templates/ts)

Вы можете использовать список для отображения связанных элементов в сканируемом формате и позволить пользователям принимать меры по всему списку или отдельным элементам.

## <a name="dashboard"></a>Информационная панель

Панель мониторинга отображает различные типы содержимого в центральном месте (Teams приложение или вкладка). Пользователи должны иметь возможность настроить по крайней мере некоторые из того, что они видят на панели мониторинга.

:::image type="content" source="../../assets/images/ui-templates/dashboard.png" alt-text="Пример показывает шаблон пользовательского интерфейса панели мониторинга." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Анализ данных
* Отчетные метрики
* Организация различной информации в одном месте

## <a name="form"></a>Form

Формы используются для сбора, проверки и отправки пользовательского ввода структурированным способом. Четкая маркировка и логические группировки входных полей имеют решающее значение для хорошего пользовательского опыта.

:::image type="content" source="../../assets/images/ui-templates/form.png" alt-text="Пример показывает шаблон пользовательского интерфейса формы." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Вход
* Профили пользователей
* Параметры
* Сбор пользовательских входных данных

Вы можете проектировать потоки вовсяки приложений для различных Teams контекстов и поставщиков идентификации. Следующий пример включает в себя один знак -на (SSO), который мы рекомендуем для простейшего опыта проверки подлинности.

:::image type="content" source="../../assets/images/ui-templates/sign-in.png" alt-text="Пример показывает знак в шаблоне пользовательского интерфейса." border="false":::

### <a name="top-use-case"></a>Верхний случай использования

* Аутентичность пользователей

## <a name="task-board"></a>Целевая доска

Доска задач, иногда называемая доска kanban или swim lanes, представляет собой набор карт, часто используемых для отслеживания состояния рабочих предметов или билетов. Он также может быть использован для сортировки любого типа контента по категориям. Вы можете редактировать и перемещать карты между столбцами.

:::image type="content" source="../../assets/images/ui-templates/task-board.png" alt-text="Пример показывает шаблон пользовательского интерфейса доски задач." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Project: Назначение задач и отслеживание статуса.
* Мозговой штурм: Добавление идей в разных категориях.
* Сортировка упражнения: Организация любой информации в ведра.

## <a name="data-visualization"></a>Визуализация данных

Вы можете использовать различные размеры карт (одиночные, двойные и полные) для укладки и организации визуализации данных на одной странице. Масштаб карт соответствует макету столбца и заполняет пробелы.

:::image type="content" source="../../assets/images/ui-templates/data-viz.png" alt-text="Пример показывает шаблон визуализации данных пользовательского интерфейса." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Отображение сложной информации
* Создание панели мониторинга

## <a name="wizard"></a>Мастер

Мастер направляет пользователя через несколько экранов для выполнения задачи (например, процесса настройки).

:::image type="content" source="../../assets/images/ui-templates/wizard.png" alt-text="Пример показывает шаблон пользовательского интерфейса мастера." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Настройка
* Адаптация
* Первый опыт работы

## <a name="empty-state"></a>Пустое состояние

Пустой шаблон состояния может быть использован для многих сценариев, включая войти в первый запуск, сообщения об ошибках и многое другое. Он очень гибкий – адаптируем его к использованию одного, нескольких или всех компонентов в следующем дизайне.

:::image type="content" source="../../assets/images/ui-templates/empty-state.png" alt-text="Пример показывает пустой шаблон пользовательского интерфейса состояния." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Вход
* Приветственные сообщения и первый запуск
* Сообщения об успехе
* Сообщения об ошибках

## <a name="notification-bar"></a>Панель уведомлений

Бар уведомлений является выделенной областью для отображения кратких, важных сообщений, которые не требуют от пользователя принятия незамедлительных мер. Конкретные цвета фона и значки связаны с определенными типами сообщений (см. ниже).

:::image type="content" source="../../assets/images/ui-templates/notification-bar.png" alt-text="Пример показывает шаблоны баров уведомлений." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Критические сообщения, ошибки и предупреждения
* Сообщения об успехе
* Информационные или рекламные сообщения

## <a name="left-nav"></a>Левый навигатор

Используйте левый навигатор для просмотра нескольких страниц в вашей Teams вкладке. В следующем примере левый навигатор находится между списком каналов и содержанием вкладок.

:::image type="content" source="../../assets/images/ui-templates/left-nav.png" alt-text="Пример показывает левый шаблон навигации." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Просматривайте несколько страниц в Teams вкладке.
* Разбей сложные приложения на несколько страниц.

## <a name="breadcrumb"></a>Breadcrumb

Breadcrumbs являются навигационным помощником, который передает иерархию вашего приложения. Они помогают пользователям понять, как страница, которую они просматривают, вписывается в общий опыт и позволить себе доступ одним щелчком мыши к более высоким уровням в этой иерархии.

:::image type="content" source="../../assets/images/ui-templates/breadcrumb.png" alt-text="Пример показывает шаблон панировочных сухарей." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Иерархия общения
* Навигация

## <a name="toolbar"></a>Панель инструментов

Панель инструментов является контейнером для группировки набора элементов управления.

:::image type="content" source="../../assets/images/ui-templates/toolbar.png" alt-text="Пример показывает шаблон панели инструментов." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Контекстные действия на содержимом приложения
* Контекстный фильтр и найти
* Навигация и панировочные сухари

## <a name="stage"></a>Этап

Stage предлагает пользователям способ открыть объект, например изображение, файл или веб-сайт, в Teams вместо того, чтобы открывать его в другом приложении или браузере. Основным примером использования этапа является просмотр; поверхность не должна использоваться для сложных взаимодействий.

(Примечание к реализации: Создайте свой этап с помощью [большого модуля задач](../../task-modules-and-cards/task-modules/design-teams-task-modules.md).)

:::image type="content" source="../../assets/images/ui-templates/stage.png" alt-text="Пример показывает шаблон сцены." border="false":::

### <a name="top-use-cases"></a>Лучшие случаи использования

* Откройте объект в Teams вместо другого приложения или браузера.
* Средства массовой информации или другой контент.
