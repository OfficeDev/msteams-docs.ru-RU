# <a name="designing-your-microsoft-teams-app-with-ui-templates"></a>Разработка приложения Microsoft Teams с помощью шаблонов пользовательского интерфейса

Быстрое проектирование приложения Microsoft Teams с помощью шаблонов пользовательского интерфейса. Шаблоны представляют собой набор компонентов на основе пользовательского интерфейса Fluent, которые работают в стандартных случаях использования Teams, что позволяет вам больше времени для того, чтобы определить оптимальное взаимодействие с пользователями.

## <a name="microsoft-teams-ui-kit"></a>Набор элементов пользовательского интерфейса Microsoft Teams

Шаблоны пользовательского интерфейса можно взять из набора UI Microsoft Teams, который также включает в себя подробные сведения об использовании, анатомии, специальных возможностях и рекомендациях.

> [!div class="nextstepaction"]
> [Получение набора элементов пользовательского интерфейса Microsoft Teams (фигма)](https://www.figma.com/community/file/916836509871353159)

## <a name="list"></a>Список

Вы можете использовать список, чтобы отображать связанные элементы в формате с возможностью записи и разрешить пользователям выполнять действия для всего списка или отдельных элементов.

:::image type="content" source="../../assets/images/ui-templates/list.png" alt-text="Пример показывает шаблон пользовательского интерфейса списка." border="false":::

### <a name="top-use-cases"></a>Основные варианты использования

* Отображение данных
* Контекстные действия для контента приложения

## <a name="dashboard"></a>Информационная панель

Панель мониторинга отображает различные типы контента в центральном расположении (личное приложение Teams или вкладка). Пользователи должны иметь возможность настраивать по крайней мере некоторые из них на панели мониторинга.

:::image type="content" source="../../assets/images/ui-templates/dashboard.png" alt-text="Пример показывает шаблон пользовательского интерфейса панели мониторинга." border="false":::

### <a name="top-use-cases"></a>Основные варианты использования

* Анализ данных
* Показатели отчетов
* Организация разных сведений в одном месте

## <a name="form"></a>Form

Формы используются для сбора, проверки и отсылки данных, вводимых пользователем, в структурированном виде. Удаление меток и логических групп полей ввода является критически важным для хорошего взаимодействия с пользователем.

:::image type="content" source="../../assets/images/ui-templates/form.png" alt-text="Пример показывает шаблон пользовательского интерфейса формы." border="false":::

### <a name="top-use-cases"></a>Основные варианты использования

* Вход
* Профили пользователей
* Параметры
* Коллекция вводимых пользователем данных

## <a name="sign-in"></a>Вход

Вы можете проектировать потоки входа в приложение для различных контекстов команд и поставщиков удостоверений. В приведенном ниже примере используется единый вход (SSO), который мы рекомендуем использовать простейший способ проверки подлинности.

:::image type="content" source="../../assets/images/ui-templates/sign-in.png" alt-text="Пример показывает шаблон пользовательского интерфейса входа." border="false":::

### <a name="top-use-case"></a>Верхний вариант использования

* Проверка подлинности пользователей

## <a name="task-board"></a>Доска задач

Доска задач, иногда называемая доской Канбан или свим желобами, представляет собой коллекцию карточек, часто используемых для отслеживания состояния рабочих элементов или билетов. Он также может использоваться для сортировки любого типа контента по категориям. Вы можете редактировать и перемещать карты между столбцами.

:::image type="content" source="../../assets/images/ui-templates/task-board.png" alt-text="В примере показан шаблон пользовательского интерфейса доски задач." border="false":::

### <a name="top-use-cases"></a>Основные варианты использования

* управление проектами; Назначение задач и состояния отслеживания
* Мозговой штурм. Добавление идей в разные категории
* Упражнения по сортировке. Организация любого вида информации в сегменты

## <a name="data-visualization"></a>Визуализация данных

Вы можете использовать различные размеры карт (Single, Double и Full) для стеков и систематизации визуализаций данных на одной странице. Карты масштабируются в соответствии с разметкой столбцов и заполнение пробелами.

:::image type="content" source="../../assets/images/ui-templates/data-viz.png" alt-text="Пример показывает шаблон пользовательского интерфейса визуализации данных." border="false":::

### <a name="top-use-cases"></a>Основные варианты использования

* Отображение сложной информации
* Создание панели мониторинга

## <a name="wizard"></a>Мастер

Мастер помогает пользователю выполнить задачу (например, процесс установки) с помощью нескольких экранов.

:::image type="content" source="../../assets/images/ui-templates/wizard.png" alt-text="В примере показан шаблон пользовательского интерфейса мастера." border="false":::

### <a name="top-use-cases"></a>Основные варианты использования

* Настройка
* Адаптация
* Интерфейс первого запуска

## <a name="empty-state"></a>Пустое состояние

Шаблон пустое состояние можно использовать для многих сценариев, в том числе входа, первого запуска, сообщений об ошибках и многого другого. Он обладает широкими возможностями, адаптируя его к использованию одного, нескольких или всех компонентов в приведенном ниже проекте.

:::image type="content" source="../../assets/images/ui-templates/empty-state.png" alt-text="В примере показан шаблон пользовательского интерфейса мастера." border="false":::

### <a name="top-use-cases"></a>Основные варианты использования

* Вход
* Приветственные сообщения и интерфейс первого запуска
* Сообщения об успешном выполнении
* Сообщения об ошибках

## <a name="notification-bar"></a>Панель уведомлений

Панель уведомлений — это выделенная область для отображения коротких, важных сообщений, не требующих от пользователя немедленного выполнения действия. Определенные цвета фона и значки связаны с определенными типами сообщений (см. ниже).

:::image type="content" source="../../assets/images/ui-templates/notification-bar.png" alt-text="Пример показывает шаблоны панели уведомлений." border="false":::

### <a name="top-use-cases"></a>Основные варианты использования

* Критические сообщения, ошибки и предупреждения
* Сообщения об успешном выполнении
* Информационные или рекламные сообщения

## <a name="left-nav"></a>Левая панель навигации

Используйте левую панель навигации для просмотра нескольких страниц на вкладке Teams (команды). В следующем примере левая панель навигации находится между списком канал и содержимым Tab.

:::image type="content" source="../../assets/images/ui-templates/left-nav.png" alt-text="Пример показывает левый шаблон навигации." border="false":::

### <a name="top-use-cases"></a>Основные варианты использования

* Обзор нескольких страниц на вкладке "команды"
* Разделение сложных приложений на несколько страниц

## <a name="breadcrumb"></a>Строка навигации

Иерархические пути — это помощь в навигации, которая передает иерархию вашего приложения. Они помогают пользователям разослаться на просматриваемую страницу в общем интерфейсе и позволить одноэлементному доступу к более высоким уровням в этой иерархии.

:::image type="content" source="../../assets/images/ui-templates/breadcrumb.png" alt-text="В примере показан пример иерархического шаблона." border="false":::

### <a name="top-use-cases"></a>Основные варианты использования

* Иерархия "обмен сообщениями"
* Навигация

## <a name="toolbar"></a>Панель инструментов

Панель инструментов — это контейнер для группировки набора элементов управления.

:::image type="content" source="../../assets/images/ui-templates/toolbar.png" alt-text="Пример показывает шаблон панели инструментов." border="false":::

### <a name="top-use-cases"></a>Основные варианты использования

* Контекстные действия для контента приложения
* Контекстный фильтр и поиск
* Навигация и иерархические строки

## <a name="stage"></a>Этап

Stage позволяет пользователям открывать сущности (например, изображение, файл или веб-сайт) в Teams, не открывая их в другом приложении или браузере. Основным вариантом использования стадии является просмотр; поверхность не должна использоваться для сложных взаимодействий.

(Примечание о реализации. Создание стадии с помощью большого [модуля задач](../../task-modules-and-cards/task-modules/design-teams-task-modules.md).)

:::image type="content" source="../../assets/images/ui-templates/stage.png" alt-text="В примере показан шаблон Stage." border="false":::

### <a name="top-use-cases"></a>Основные варианты использования

* Открытие объекта в Teams вместо другого приложения или браузера
* Важный носитель или другое содержимое
