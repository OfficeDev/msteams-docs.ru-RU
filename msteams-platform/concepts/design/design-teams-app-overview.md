# <a name="designing-your-microsoft-teams-app"></a>Проектирование приложения Microsoft Teams

:::image type="content" source="../../assets/images/design-guidelines-overview.png" alt-text="Концептуальный образ с рекомендациями по проектированию Microsoft Teams.":::

Независимо от того, являетесь ли вы дизайнером, менеджером по продуктам, разработчиком или создателем с помощью средств с низким кодом, эти рекомендации помогут вам быстро принять правильные решения по проектированию для вашего приложения Microsoft Teams.

## <a name="teams-app-design-principles"></a>Принципы проектирования приложений Teams

Приложения Teams помогают людям добиться большего вместе. Используйте эти принципы для разработки.

:::row:::
   :::column span="":::

### <a name="collaborative"></a>Совместная работа

Приложения Teams помогают людям добиться большего вместе. Используйте эти принципы для разработки.

   :::column-end:::
   :::column span="":::

### <a name="trustworthy"></a>Надежность

Приложение защищено и соответствует требованиям. Пользователи могут легко находить сведения о конфиденциальности.

   :::column-end:::
:::row-end:::

:::row:::
   :::column span="":::

### <a name="globally-inclusive"></a>Глобальный инклюзивный

Приложение могут использовать люди всех фонов, навыков и навыков. Это культурная, расовая и социальная интеграция.

   :::column-end:::
   :::column span="":::

### <a name="light"></a>Легкие

Приложение фокусируется на основных сценариях, которые смешиваются с рабочего процесса Teams.

   :::column-end:::
:::row-end:::

:::row:::
   :::column span="":::

### <a name="native-or-distinct"></a>Native или distinct

Приложение использует собственные компоненты проектирования Teams или ваши собственные. Не существует смешения цветовых схем, элементов управления и т. д.

   :::column-end:::
   :::column span="":::

### <a name="useful"></a>Полезно

Приложение основано на сценарии, который необходимо сделать в Teams.

   :::column-end:::
:::row-end:::

:::row:::
   :::column span="":::

### <a name="easy-to-use"></a>Простой в использовании

Пользовательский интерфейс прост в понятии, прекрасно выглядит и тон, а также повышает продуктивность работы людей.

   :::column-end:::
   :::column span="":::

### <a name="responsive"></a>Оперативно

Приложение не зависит от устройства и экрана.

   :::column-end:::
:::row-end:::

:::row:::
   :::column span="":::

### <a name="accessible"></a>Доступные

Приложение соответствует требованиям к специальным требованиям Teams с точки зрения цветовой контрастности, альтернативных вариантов навигации и других возможностей.

   :::column-end:::
   :::column span="":::

### <a name="well-described"></a>Хорошо описано

Текст, значки и изображения четко понять, для чего нужно приложение и как его использовать.

   :::column-end:::
:::row-end:::

## <a name="creating-a-cohesive-experience"></a>Создание единого и согласованного опыта

Проектирование приложения Teams похоже на разработку обычного веб-приложения, но также немного отличается. Эффективная разработка выделяет уникальные атрибуты вашего приложения, при этом естественным образом вписываясь в функции и контексты Teams.

Эти рекомендации и ресурсы помогут вам найти баланс. Вы узнаете, что делать и чего избегать при разработке приложения Teams (например, многоуровневая навигация на вкладке).
