# <a name="manage-your-apps-with-app-studio-for-microsoft-teams"></a>Управление приложениями с помощью App Studio для Microsoft Teams

> [!WARNING]
> **Попробуйте портал разработчика**: App Studio развивается. Настройте, распространите и управляйте Teams приложениями с помощью нового [портала разработчика](https://dev.teams.microsoft.com/). <br> App Studio станет не рекомендованным к использованию не позднее 30 июня 2022 г.

С помощью App Studio можно удобно создавать и интегрировать ваши собственные приложения для Microsoft Teams, если вы разрабатываете пользовательские приложения для своего предприятия или приложения SaaS для команд во всем мире. App Studio упрощает создание манифеста и пакета для вашего приложения и предоставляет вам полезные инструменты, такие как редактор карточек и библиотека элементов управления React.

> [!IMPORTANT]
> В настоящее время App Studio недоступен в следующих типах Teams организаций:
>
> * Облако сообщества для государственных организаций (GCC)
> * GCC High
> * DoD

## <a name="installing-app-studio"></a>Установка App Studio

App Studio — это Teams, которое можно найти в Teams магазине. Перейдите по этой ссылке для прямого [скачивания App Studio](https://aka.ms/InstallTeamsAppStudio). Приложение также можно найти в магазине приложений.

Найдите App Studio в магазине приложений.

:::image type="content" source="../../assets/images/get-started/StoreTeamsAppStudio.png" alt-text="Запись App Studio в магазине приложений":::

Выберите плитку App Studio, чтобы открыть страницу установки этого приложения:

:::image type="content" source="../../assets/images/get-started/teamsAppStudioConfiguration.png" alt-text="Настройка app studio":::

Выберите **Установить**.

:::image type="content" source="../../assets/images/get-started/TeamsAppStudio.png" alt-text="app studio":::

После входа в App Studio выберите на вкладке редактора манифеста, где можно импортировать существующее приложение или создать новое.

## <a name="app-studio-features"></a>Функции App Studio

В этом разделе описываются такие функции, как беседа, редактор манифестов, сведения и возможности. Вы можете настроить свои возможности с помощью настройки приложения.

### <a name="conversation"></a>Беседа

Здесь можно просмотреть, как будут выглядеть в Teams [карточки, созданные вами в App Studio](#card-editor), когда вы будете их тестировать, отправляя их себе.

### <a name="manifest-editor"></a>Редактор манифестов

Как упоминалось ранее, наиболее важной частью пакета Teams является его файл manifest.json. Этот файл, который должен [соответствовать](~/resources/schema/manifest-schema.md) схеме Teams приложения, содержит метаданные, которые Teams правильно представлять приложение пользователям.

Вкладка редактора манифеста в App Studio упрощает создание манифеста, позволяя описывать приложение, отправлять значки, добавлять возможности приложения и создавать файл .zip, который можно легко отправить в Teams для тестирования или распространять для других пользователей. App Studio не создает функциональный код для приложения или не размещает его. Ваше приложение уже должно быть размещено и запущено по URL-адресу, указанному в манифесте: только в этом случае результатом процесса отправки приложения станет действующее приложение.

#### <a name="details"></a>Сведения

В разделе сведений редактора манифестов определяется высокоуровневая информация о создаемом приложении. В частности, здесь содержится имя приложения, его описание и фирменное визуальное оформление. Можно автоматически сформировать идентификатор GUID для приложения и указать URL-адреса заявления о конфиденциальности и условий использования.

#### <a name="capabilities"></a>Возможности

В разделе возможностей в редакторе манифестов определяются возможности приложения и указываются сведения о каждой возможности.

> [!NOTE]
> Рекомендуется предоставить пользователям и клиентам приложения рекомендации по настройке, которым следует следовать при настройке приложения. Дополнительные сведения см. [в статье о настройке приложений в Microsoft Teams](/MicrosoftTeams/customize-apps).

##### <a name="tabs"></a>Вкладки

* **Вкладки команд.** Вкладка команды становится частью канала и обеспечивает быстрый доступ к сведениям и ресурсам группы. Например, вкладка "Планировщик" для канала содержит один план, а вкладка "Power BI" связана с определенным отчетом. Пользователи могут выполнять детализацию до соответствующего контекста, но они не должны иметь возможности переходить за пределы вкладки. Например, Power BI не включает навигацию по другим отчетам Power BI, но включает кнопку "Перейти к веб-сайту", которая запускает  отчет на основном веб-сайте Power BI.

  Для вкладок команд необходимо предоставить *URL-адрес конфигурации* для предоставления вариантов и сбора информации, чтобы пользователи могли настраивать содержимое и внешний вид вкладки. Эта HTML-страница отображается в элементе iframe, когда пользователь впервые добавляет эту вкладку в канал.

  Также необходимо указать все дополнительные домены, из которых эта вкладка будет загружать информацию или с которыми она будет связана.

* **Личные вкладки** Вы можете определить набор вкладок, которые по умолчанию представлены в личном интерфейсе приложения (взаимодействие пользователя с приложением за пределами контекста команды или канала). В этом разделе укажите имя вкладки, уникальный идентификатор и URL-адрес, указывающий на пользовательский интерфейс, который нужно отобразить в Teams. При желании можно указать URL-адрес на случай, если пользователь предпочтет просмотреть эту вкладку в браузере. Используя Teams вкладки, укажите дополнительные домены, из которых ожидается загрузка вкладки или ссылка на нее.

##### <a name="bots"></a>Боты

В этом разделе можно добавить в приложение [бот для интерактивных бесед](~/bots/what-are-bots.md). Если у вас уже есть бот, зарегистрированный в Bot Framework, его можно добавить, нажав  кнопку "Настроить" и указав имя бота, идентификатор Bot Framework и определив области, в которых работает бот.

Если вы еще не зарегистрировали бот в Bot Framework, **выберите "Зарегистрировать** ", чтобы создать новый. После регистрации бота вернитесь в этот раздел редактора манифестов и введите имя бота и его идентификатор Bot Framework.

После того как вы введите сведения о боте, вы можете при необходимости определить список команд, которые бот может предложить пользователям. Добавьте имя команды, описание команды, указывающее ее синтаксис и аргументы, а также области, к которым должна применяться эта команда.

> [!NOTE]
> Если вы определили, что бот поддерживает только одну область, команды, указанные для неподдерживаемой области, игнорируются. Можно в любое время изменить области, поддерживаемые вашим ботом.

##### <a name="connectors"></a>Соединители

В этом разделе можно добавить соединитель в приложение. Если вы уже зарегистрировали соединитель Office 365, выберите **Настройка** и введите имя и идентификатор соединителя. Если вы хотите, чтобы новый соединитель перенаправлен **на** панель мониторинга разработчика соединителя в браузере, выберите "Зарегистрировать".

##### <a name="message-extensions"></a>Расширения сообщений

[Расширения сообщений —](~/messaging-extensions/what-are-messaging-extensions.md) это эффективный способ взаимодействия пользователей с приложением в Teams. Пользователи могут запрашивать сведения в вашей службе и публиковать их в виде карточек непосредственно в канале или в чате, где идет беседа.

Расширения сообщений работают на основе ботов Bot Framework, поэтому для их работы требуется настроенный бот. Если у вас есть имя и идентификатор Bot Framework бота, введите его. В противном случае **выберите "** Зарегистрировать", чтобы создать его, и введите сведения позже. Укажите, может ли пользователь обновить конфигурацию расширения сообщений.

После настройки базового бота определите команды и параметры, которые может принять расширение сообщения.

Для каждой команды требуется название и идентификатор. Также команда может содержать описание для пользователя. Каждая команда может поддерживать до пяти из параметров, для каждого из которых требуется:

* Имя параметра в том виде, в Teams и включено в запрос пользователя.
* Понятное название.
* Дополнительное описание.

> [!NOTE]
> Сведения о создании расширения сообщений с помощью App Studio см. в статье ["Создание расширения сообщений с помощью App Studio"](~/resources/create-messaging-extension-using-appstudio.md).

#### <a name="test-and-distribute"></a>Тестирование и распространение

После определения приложения раздел "Тестирование и распространение" позволяет экспортировать определение приложения в виде ZIP-файла, который затем можно отправить в клиент Teams для тестирования. Если щелкнуть "Экспорт", ZIP-файл *appname.zip* будет скачан в папку по умолчанию для скачанных файлов.

##### <a name="publish-your-app-to-teams"></a>Опубликуйте свое приложение в Teams

На домашней странице проекта можно передать приложение в команду, отправить приложение в настраиваемый магазин приложений вашей компании, чтобы приложением смогли воспользоваться пользователи вашей организации, или отправить приложение в AppSource для всех пользователей Teams. ИТ-администратор проверяет эти отправки. Можно вернуться на страницу *Публикация*, чтобы проверить состояние отправки и узнать, было ли ваше предложение утверждено или отклонено ИТ-администратором. На этой странице вы также можете отправлять обновления для вашего приложения и отменять все действующие отправки.

### <a name="card-editor"></a>Редактор карточек

Карточка — это контейнер для коротких или связанных фрагментов информации. Teams поддерживает карточки, которые могут иметь несколько свойств и вложений. Карточки — основной способ, при помощи которого боты и соединители передают пользователям полезную информацию, требующую действий.

Чтобы упростить этот процесс и уменьшить вероятность ошибок, на вкладке "Редактор карточек" можно создавать карточки имиджевого баннера или эскизы с помощью формы, а также проверять и тестировать полученные карточки (точно так же, как пользователь будет видеть) с помощью бота. Также предоставляется соответствующий код карточки (JSON, C# или Node.js), который можно скопировать и вставить в исходный код вашего приложения.

Если у вас уже есть карточка, которую вы ходите проверить в Teams, вставьте код JSON этой карточки на вкладке "JSON" в разделе *Добавление сведений о карточках* и отправьте эту карточку себе, чтобы увидеть, как она выглядит в чате.

### <a name="react-control-library"></a>Библиотека элементов управления React

>[!Note]
> Эта React нерекомендуема в будущем. Рассмотрите возможность использования [Fluent-UI react в качестве альтернативы](https://microsoft.github.io/fluent-ui-react/) ранее созданному пользовательскому интерфейсу Stardust.

Создание приложения, соответствующего лучшим методикам Teams, — лучший способ придать приложению внешний вид и порядок использования, соответствующие пользовательскому интерфейсу клиента Teams. Для достижения этой цели крайне важны используемые вами элементы управления в пользовательском интерфейсе. Чтобы упростить создание согласованного пользовательского интерфейса, App Studio предоставляет несколько категорий элементов управления пользовательского интерфейса, которые соответствуют Teams проектирования.

Предоставляются примеры элементов управления и соответствующих компонентов React, готовых к использованию при создании вашего приложения.

#### <a name="controls"></a>Элементы управления

Доступны следующие элементы управления:

* Кнопки
* Раскрывающиеся списки
* Флажки
* Переключатели
* Выключатели
* Текстовые области
* Ссылки
* Вкладки
* Таблицы
* Значки
