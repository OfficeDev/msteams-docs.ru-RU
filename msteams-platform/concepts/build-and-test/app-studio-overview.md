# <a name="quickly-develop-apps-with-app-studio-for-microsoft-teams"></a>Быстрая разработка приложений с помощью App Studio для Microsoft Teams

Приложение App Studio упрощает создание или интеграцию собственных приложений Microsoft Teams, независимо от того, разрабатываете ли вы настраиваемые приложения для Teams или SaaS приложений для Teams по всему миру, что упрощает создание манифеста и пакета для приложения и предоставляет такие средства, как редактор карточки и библиотека элемента управления "реагируя".

## <a name="installing-app-studio"></a>Установка App Studio

App Studio — это приложение Teams, которое можно найти в магазине Teams. Используйте эту ссылку для непосредственной загрузки: [app Studio](https://aka.ms/InstallTeamsAppStudio) (вы также можете найти приложение в магазине App Store).

В магазине выполните поиск в App Studio.

![Запись хранилища для App Studio](~/assets/images/get-started/storeteamsappstudio.png)

Выберите плитку App Studio, чтобы открыть страницу установки приложения:

![Настройка App Studio](~/assets/images/get-started/teamsappstudioconfiguration.png)

Нажмите кнопку *установить*.

![App Studio](~/assets/images/get-started/teamsappstudio.png)

Когда вы используете приложение App Studio, перейдите на вкладку *редактор манифестов* , где можно импортировать существующее или создать новое приложение.

## <a name="app-studio-features"></a>Компоненты App Studio

### <a name="conversation"></a>Беседа

Здесь вы можете узнать, какие [карточки в приложении App Studio будут](#card-editor) выглядеть так, как в Teams, когда вы протестируйте их, отправив их себе.

### <a name="manifest-editor"></a>Редактор манифеста

Как упоминалось ранее, наиболее важной частью пакета приложений Microsoft Teams является его manifest.jsфайла. Этот файл, который должен соответствовать [схеме приложения Teams](~/resources/schema/manifest-schema.md), содержит метаданные, которые позволяют Teams правильно отображать приложение для пользователей.

Вкладка редактор манифестов в App Studio упрощает создание манифеста, позволяет вам описать приложение, отправить значки, добавить возможности приложений и создать ZIP-файл, который можно легко отправить в Teams для тестирования или распространения для использования другими пользователями. Обратите внимание, что приложение App Studio не создает функциональный код для приложения или не размещает свое приложение. Ваше приложение должно быть уже размещено и запущено на URL-адресе, указанном в манифесте для процесса отправки приложения, чтобы обеспечить работу приложения.

#### <a name="details"></a>Сведения

В разделе сведения редактора манифеста определяется высокоуровневое описание создаваемого приложения. Сюда входят такие элементы, как имя, описание и визуальная фирменная символика приложения. Можно автоматически создать GUID для вашего приложения и указать URL-адреса для заявления о конфиденциальности и условий использования.

#### <a name="capabilities"></a>Возможности

В разделе возможности редактора манифеста определяются возможности приложения и отображаются сведения о каждой из этих возможностей.

##### <a name="tabs"></a>Вкладки

* **Вкладки групп.** Вкладка "Группа" становится частью канала и обеспечивает быстрый доступ к сведениям о группе и ресурсам. Например, вкладка планировщика для канала содержит один план; Вкладка Power BI соответствует определенному отчету. Пользователи могут перейти к соответствующему контексту, но они не должны иметь возможности переходить за границы вкладки. На вкладке Power BI, например, не разрешите переход к другим отчетам Power BI, но при этом будет включена кнопка *Перейти на веб-сайт* , запускающий отчет на основном веб-сайте Power BI.

  Для командных вкладок необходимо указать *URL-адрес конфигурации* , чтобы представить параметры и собрать сведения, чтобы пользователи могли настраивать контент и работу вкладки. Эта HTML-страница с IFRAME отображается, когда пользователь впервые добавляет вкладку в канал.

  Кроме того, необходимо предоставить дополнительные домены, на которых должна загружаться вкладка, или связать ее.

* **Личные вкладки.** В этом разделе вы можете определить набор вкладок, которые отображаются по умолчанию в личном приложении (например, в работе пользователя с приложением вне контекста команды или канала). В этом разделе Укажите имя вкладки, уникальный идентификатор, URL-адрес, указывающий на пользовательский интерфейс, который будет отображаться в Teams, и (при необходимости) URL-адрес, который будет использоваться, если пользователь попытается просмотреть вкладку в браузере. Как и в случае с вкладками Teams, укажите дополнительные домены, с которых должна загружаться вкладка (или с которой ожидается связь).

##### <a name="bots"></a>Боты

В этом разделе вы можете добавить в свое приложение объект " [Bot](~/bots/what-are-bots.md) ". Если у вас уже есть Bot, зарегистрированный с помощью Bot Framework, вы можете добавить его, щелкнув элемент *Настройка* и указав имя пользователя и идентификатор ленты, а также определить области, в которых будет работать Bot.

Если вы еще не зарегистрировали Bot с помощью Bot Framework, нажмите кнопку *Регистрация* , чтобы создать новую. После завершения регистрации Bot вернитесь к этому разделу редактора манифеста, чтобы ввести имя и идентификатор платформы Bot.

После того как вы задали сведения о почтовом роботе, вы можете дополнительно определить список команд, предлагаемых для пользователя с помощью ленты. Добавьте имя команды, описание команды, которое указывает ее синтаксис и аргументы, а также области, к которым должна применяться эта команда.

Обратите внимание на то, что если вы определили, что для ленты определена только одна область, команды, указанные для неподдерживаемой области, будут игнорироваться. Вы можете изменить области, которые поддерживает ваш Bot, в любое время.

##### <a name="connectors"></a>Соединители

Этот раздел позволяет добавить соединитель в приложение. Если вы уже зарегистрировали соединитель Office 365, нажмите кнопку *Настройка* и введите имя и идентификатор соединителя. Если вы хотите добавить новый соединитель, нажмите кнопку Register (регистрация), чтобы *отписаться* на информационную панель разработчика соединителей в браузере.

##### <a name="messaging-extensions"></a>Расширения для сообщений

[Расширения обмена сообщениями](~/messaging-extensions/what-are-messaging-extensions.md) — это мощный способ привлечения пользователей к работе с приложением в Microsoft Teams. Пользователи могут запрашивать информацию из вашей службы и отправлять эти сведения в виде карточек, прямо в канал или беседу в чате.

Расширения обмена сообщениями на базе Боты Framework, поэтому для них требуется настроенный для работы пользователь. Если у вас есть имя и идентификатор платформы Bot, на которой вы хотите включить питание для расширения обмена сообщениями, введите его. В противном случае нажмите *зарегистрировать* , чтобы создать его и ввести сведения позже. Укажите, может ли пользователь изменять конфигурацию расширения системы обмена сообщениями.

После настройки основного робота определите команды и параметры, которые может принимать расширение обмена сообщениями.

Каждая команда требует указания заголовка и идентификатора. Команда может дополнительно содержать описание для пользователя. Каждая команда может поддерживать до пяти параметров, каждая из которых требует:

* Имя параметра в том виде, в котором оно отображается в клиенте Teams и включенное в запрос пользователя
* Понятное название
* Необязательное описание

#### <a name="test-and-distribute"></a>Тестирование и распределение

По завершении определения приложения раздел "тестирование и распространение" позволяет экспортировать определение приложения в виде ZIP-файла, который затем может быть предоставлен в клиенте Teams для тестирования. При нажатии кнопки Экспорт ZIP-файл скачивается как *appname.zip* в каталоге загрузки по умолчанию.

На домашней странице проекта вы можете отправить свое приложение в группу, отправить свое приложение в пользовательскую магазин приложений для пользователей в вашей организации или отправить свое приложение в источник приложений для всех пользователей Teams. ИТ ИТ ИТ, просматривая эти отправки. Вы можете вернуться на страницу *публикации* , чтобы проверить состояние отправки, а также узнать, было ли ваше приложение утверждено или отклонено вашим ИТ-администратором. Кроме того, здесь вы будете передавать обновления в свое приложение или отменять все активные в данный момент отправки.

### <a name="card-editor"></a>Редактор карточек

Карточка — это контейнер для коротких или связанных частей информации. Microsoft Teams поддерживает карты, у которых может быть несколько свойств и вложений. Карточки — это ключевой способ, с помощью которого Боты и соединители ретрансляции могут получить доступ к данным. 

Чтобы упростить и уменьшить вероятность возникновения ошибки, на вкладке Редактор карточки можно создавать карточки главный Имиджевый баннер или карты эскизов с помощью формы, а также проверять и тестировать полученную карточку (точно так же, как пользователь видит ее) с помощью Bot. Кроме того, он предоставляет соответствующий код JSON, C# или Node.js для карточки, которую можно скопировать или вставить в исходный код приложения.

Если у вас уже есть карточка, которую вы хотите проверить внутри Teams, вы можете вставить JSON для этой карточки на вкладку JSON в разделе *добавить сведения о карте* и отправить ее себе, чтобы увидеть, как она выглядит в чате.
