# <a name="web-apps"></a>Веб-приложения 

Вы можете сделать веб-приложения подходящими Teams социальных и совместных функций, правильно интегрируя их с Teams.
  
Различные типы приложений, которые вы можете интегрировать с Teams, следующие:
* **Автономные приложения**: Автономное приложение является одностраничное или большое и сложное приложение. Пользователь может использовать некоторые аспекты его в Teams.
* **Приложения для совместной** работы : приложение, уже построенное для социальных и совместных функций, присущих Teams.
* **SharePoint**: страница SharePoint, которую вы хотите, чтобы поверхность в Teams.

Вы можете составить карту и следовать соответствующим рекомендациям, применимым к вашему сценарию интеграции.
Этот документ дает обзор возможностей Teams, требований к точкам обмена файлами и данными, требований API, аутентификации и глубокой связи вашего приложения с Teams.

## <a name="understand-authentication-options"></a>Понимание вариантов проверки подлинности

***Сценарии интеграции**: Автономные приложения, приложения для совместной работы, SharePoint»

Azure Active Directory (AD) является поставщиком идентификационных данных для Teams. Если ваше приложение использует другого поставщика идентификационных данных, необходимо либо выработать упражнение по отображению идентификации, либо объединиться с Azure AD.

Teams имеет единые механизмы ва-а (SSO) с Azure AD для сторонних приложений. Он также предоставляет рекомендации для потоков аутентификации другим поставщикам идентификационных данных, используя стандарты, такие как OAuth и Open ID Подключение, известный как OIDC.

Для SharePoint страниц можно использовать только SSO и не добавлять еще один идентификатор Azure AD, если вы хотите, чтобы SSO работал на другое приложение, так как идентификатор является SharePoint приложением.

Подробнее об [аутентификации можно узнать в Teams.](../concepts/authentication/authentication.md)

## <a name="be-smart-when-messaging-users"></a>Будьте умны при обмене сообщениями пользователей

***Сценарии интеграции**: Автономные приложения, приложения для совместной работы, SharePoint»

Используйте [бота](../bots/what-are-bots.md) в вашем Teams для многотемного разговора, так как он предлагает больше гибкости, [чем webhook.](../webhooks-and-connectors/what-are-webhooks-and-connectors.md)

Боты также позволяют отправлять **упреждающие сообщения отдельным** пользователям или каналам. Упреждающие сообщения являются необумежденно сообщениями, вызванными внешним событием, а не сообщением, отправленным боту. Например, ваш бот отправляет приветственное сообщение при установке или новый пользователь присоединяется к каналу. 

Отправка упреждающих сообщений требует Teams конкретных идентификаторов. Вы можете захватить информацию путем [извлечения реестра или данных профиля пользователя,](../bots/how-to/get-teams-context.md#fetch-the-roster-or-user-profile) [подписки на события разговора,](../bots/how-to/conversations/subscribe-to-conversation-events.md)или с [помощью Microsoft Graph](/graph/teams-proactive-messaging).

Не спам пользователей с чрезмерными сообщениями. Если Teams поддерживает его, пользователи могут настроить настройки уведомлений для вашего приложения.
