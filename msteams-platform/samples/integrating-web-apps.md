# <a name="considerations-for-teams-integration"></a>Вопросы интеграции Teams

Вы можете сделать веб-приложения подходящими для Teams и совместной работы, правильно интегрировав их с Teams.
  
Ниже приведены различные типы приложений, которые можно интегрировать с Teams приложениями.

* **Автономные приложения**: автономное приложение — это однострайное или крупное и сложное приложение. Пользователь может применять некоторые его аспекты в Teams.
* **Приложения для** совместной работы: приложение, уже созданное для социальных сетей и функций совместной работы, которые Teams.
* **SharePoint**: SharePoint, которую вы хотите отобразить в Teams.

Вы можете сопоставить и следовать соответствующим рекомендациям, применимым к сценарию интеграции.
В этом документе представлен обзор возможностей Teams, требований к общей точке для хранения файлов и данных, требований к API, проверки подлинности и глубокой компоновки приложения с Teams.

## <a name="understand-sharepoint-requirements-and-options"></a>Общие SharePoint и параметры

***Сценарии интеграции**: SharePoint*

Чтобы интегрировать существующую [SharePoint как](/sharepoint/dev/general-development/overview-of-the-sharepoint-page-model) вкладку Teams, необходимо учитывать следующее:

* Это должна быть *современная SharePoint* веб-страницы.
* Поддерживаются только личные вкладки. Вы не можете интегрировать страницу как вкладку канала.

Кроме того, можно создать вкладку Teams [с помощью SharePoint Framework](/sharepoint/dev/spfx/integrate-with-teams-introduction).

## <a name="understand-authentication-options"></a>Общие сведения о вариантах проверки подлинности

***Сценарии интеграции**: автономные приложения, приложения для совместной работы, SharePoint*

Azure Active Directory является поставщиком удостоверений для Teams. Если приложение использует другой поставщик удостоверений, необходимо выполнить упражнение по сопоставлению удостоверений или объединить его с Microsoft Azure Active Directory (Azure AD).

Teams имеет механизмы единого входа (SSO) с Azure AD для сторонних приложений. Он также предоставляет рекомендации по потокам проверки подлинности для других поставщиков удостоверений, использующих такие стандарты, как OAuth и Open ID Подключение, известные как OIDC.

> [!IMPORTANT]
> В настоящее время сторонние приложения доступны в облако сообщества для государственных организаций (GCC), но недоступны для GCC-High и Министерства обороны США (DOD). Сторонние приложения по умолчанию отключены для GCC. Сведения о том, как включить сторонние приложения для GCC, см. в статье об управлении политиками разрешений [приложений и](/microsoftteams/teams-app-permission-policies) [управлении приложениями](/microsoftteams/manage-apps).

Для SharePoint вы можете использовать единый вход и не можете добавить другой идентификатор Azure AD, если хотите, чтобы единый вход можно было использовать для другого приложения, так как идентификатор является SharePoint приложения.

Дополнительные сведения о [проверке подлинности см. в Teams](../concepts/authentication/authentication.md).

## <a name="be-smart-when-messaging-users"></a>Будьте интеллектуальными при обмене сообщениями с пользователями

***Сценарии интеграции**: автономные приложения, приложения для совместной работы, SharePoint*

Используйте бот [в](../bots/what-are-bots.md) приложении Teams для многопотоковой беседы, так как он обеспечивает большую гибкость, чем [веб-перехватчик](../webhooks-and-connectors/what-are-webhooks-and-connectors.md).

Боты также позволяют отправлять **упреждающие сообщения** отдельным пользователям или каналам. Упреждающие сообщения — это незащищенные сообщения, активируемые внешним событием, а не сообщения, отправляемые боту. Например, бот отправляет приветственное сообщение, когда он установлен или новый пользователь присоединяется к каналу.

Для отправки упреждающих сообщений Teams идентификаторов. Вы можете записать информацию, извлекая данные о списке или профиле [пользователя, подписку](../bots/how-to/get-teams-context.md#fetch-the-roster-or-user-profile) на события беседы или используя [microsoft Graph](/microsoftteams/platform/graph-api/proactive-bots-and-messages/graph-proactive-bots-and-messages?context=graph/context#proactive-messaging-in-teams). [](../bots/how-to/conversations/subscribe-to-conversation-events.md)

Не используйте спам для пользователей с чрезмерным количеством сообщений. Если Teams поддерживает эту возможность, пользователи могут настроить параметры уведомлений для вашего приложения.

## <a name="use-sharepoint-for-file-and-data-storage"></a>Использование SharePoint для хранения файлов и данных

***Сценарии интеграции:** Автономные приложения, приложения для совместной работы, SharePoint страниц*
