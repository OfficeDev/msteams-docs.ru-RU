# <a name="integrate-web-apps"></a>Интеграция веб-приложений

Вы можете обеспечить обогащенное взаимодействие с пользователем, интегрировав функции существующего веб-приложения в Microsoft Teams платформе. Обязательно следуйте [Teams проектирования](~/concepts/design/understand-use-cases.md), чтобы ваше приложение было Teams.
В этом документе приводятся общие сведения о предварительных требованиях для интеграции веб-приложений с Teams, power platform для создания приложений Power, Power Virtual Agents, Виртуальный помощник, шаблонов приложений, соединителей Shift, Moodle LMS, создания кнопки "Общий доступ к Teams" для веб-сайта, добавления Microsoft Teams в SharePoint, создания прямых ссылок и интеграции возможностей устройств.

## <a name="prerequisites"></a>Предварительные требования

Для эффективной интеграции убедитесь, что у вас есть более полное представление о следующих предварительных требованиях:

* Teams возможностей.
* SharePoint требования к хранилищу файлов и данных.
* Требования к API.
* Проверки подлинности.
* Прямая компоновка приложения с помощью Teams.
* Сопоставьте варианты использования приложения с Teams платформы.
* Определите точки входа приложения, такие как личное использование, совместная работа или и то, и другое.

## <a name="low-code-platforms"></a>Платформы с низким уровнем кода

Низкоуровневые платформы кода предоставляют интуитивно понятный подход к разработке программного обеспечения и не требуют написания кода для создания приложений и процессов. Вы можете легко создавать пользовательские приложения с низкой платформой кода. Эти платформы состоят из визуального интерфейса, соединителей для серверных служб и встроенной системы управления жизненным циклом приложений для создания, отладки, развертывания и обслуживания приложений. Корпорация Майкрософт предоставляет следующие инновационные шлюзы для быстрой сборки Teams совместимых приложений с использованием атрибутов низкого уровня кода:

* Платформа Microsoft Power
* Microsoft Teams шаблонов приложений

## <a name="microsoft-power-platform"></a>Платформа Microsoft Power

Платформа Microsoft Power объединяет четыре надежные технологии Майкрософт, такие как Power BI, Power Apps, Power Automate и Power Virtual Agents в одной мощной платформе приложений. Эти технологии позволяют создавать решения, автоматизировать процессы, анализировать данные и создавать виртуальные агенты в единой и интегрированной среде.

>[!NOTE]
>Не следует использовать Microsoft Power Platform для создания приложений, которые должны быть опубликованы в Teams app Store. Приложения Microsoft Power Platform можно публиковать только в магазине приложений организации.

### <a name="power-apps"></a>Power Apps

С Power Apps вы можете создавать бизнес-приложения, которые подключаются к бизнес-данным и адаптированы к потребностям вашей организации. Power Apps широкий спектр сценариев приложений для решения бизнес-задач с помощью приложений на основе холста. После создания приложения его можно экспортировать с портала Power Apps maker и внедрить в Microsoft Teams.

### <a name="power-virtual-agents"></a>Power Virtual Agents

Power Virtual Agent — это не код, а интерактивное графическое решение. Она построена на платформе Microsoft Power Platform и Bot Framework. Она позволяет каждому участнику вашей команды создавать и поддерживать полнофункциональные чат-боты, которые легко интегрируются с Teams платформой. Вы можете проектировать, разрабатывать и публиковать интеллектуальные виртуальные агенты для Teams без необходимости настройки среды разработки, создания веб-службы или непосредственной регистрации в Bot Framework.

### <a name="create-virtual-assistant"></a>Создание виртуального помощника

Виртуальный помощник — это шаблон Майкрософт с открытым исходным кодом, который позволяет создать надежное диалоговое решение, сохраняя полный контроль над взаимодействием с пользователем, фирменной символикой организации и необходимыми данными.

## <a name="app-templates"></a>Шаблоны приложений

Шаблон приложения можно использовать для создания пользовательских приложений в соответствии с потребностями организации. Это готовые к работе приложения для Microsoft Teams, управляемые сообществом, с открытым кодом и доступные на GitHub. Каждый шаблон содержит подробные инструкции по развертыванию и установке приложения для организации. Он предоставляет готовое к использованию приложение, которое можно установить и начать использовать немедленно.

## <a name="teams-shifts-work-force-management-connectors"></a>Teams shifts Work Force Management connectors

Teams shifts Work Force Management — это готовые к работе интеграции с открытым кодом и интеграции на основе сообщества. Они обеспечивают простой интерфейс и быстрый процесс цифрового преобразования сотрудников без компьютеров с Teams смен.

## <a name="install-moodle-lms"></a>Установка Moodle LMS

Moodle — это популярная система управления Обучение с открытым кодом (LMS). Теперь он интегрирован с Microsoft Teams. Эта интеграция помогает преподавателям и преподавателям совместно работать над курсами Moodle, задавать вопросы о оценках и заданиях, а также получать уведомления непосредственно в Teams.

## <a name="create-a-share-to-teams-button-for-your-website"></a>Создайте кнопку «Поделиться в Teams» для своего сайта

Сторонние веб-сайты могут использовать сценарий средства запуска для внедрения share в Teams кнопки на своих веб-страницах. При нажатии кнопки запускается общий доступ Teams во всплывающем окне. Это позволяет предоставлять общий доступ к ссылке непосредственно любому пользователю или Microsoft Teams каналу без переключения контекста.

## <a name="add-a-microsoft-teams-tab-in-sharepoint"></a>Добавление вкладки Microsoft Teams в SharePoint

Вы можете получить широкие возможности интеграции между Microsoft Teams и SharePoint, добавив вкладку Microsoft Teams в SharePoint в качестве SPFx веб-части.

## <a name="create-deep-link"></a>Создание глубокой ссылки

Вы можете создавать глубокие ссылки на сущности в Teams. Вы можете создавать ссылки на сведения и функции в Teams. Эти прямые ссылки переходя к содержимому и сведениям на вкладке. Вы можете использовать прямые ссылки для связывания приложения с Teams, так как они объединяет несколько частей приложения для более собственного Teams взаимодействия.

## <a name="integrate-device-capabilities"></a>Интеграция возможностей устройства

Microsoft Teams постоянно совершенствует возможности разработчиков в соответствии со встроенными возможностями. Расширенная платформа Teams позволяет партнерам получать доступ к собственным возможностям устройства, таким как камера, QR или сканер штрихкодов, фотоальбом, микрофон и расположение, с помощью выделенных интерфейсов API, доступных в клиентском пакете SDK Microsoft Teams JavaScript.

## <a name="integrate-people-picker"></a>Интеграция средства "Выбор людей"

Вы можете интегрировать Teams средства выбора людей, который позволяет пользователям выполнять поиск и выбирать людей в веб-приложении.

## <a name="integrate-teams-in-your-external-app"></a>Интеграция Teams во внешнее приложение

Вы можете внедрить собственные возможности в Microsoft Teams, создав Teams приложений. Если вы хотите отменить эту  модель и интегрировать Teams или другие возможности связи в собственный интерфейс внешнего [приложения](/azure/communication-services/overview), см. Службы коммуникации Azure. Службы коммуникации Azure — это облачные службы с интерфейсами REST API и пакетами SDK для клиентских библиотек, которые помогают интегрировать обмен данными в собственные пользовательские приложения. Вы можете внедрить универсальные или Teams веб-компоненты React для вызова и чата с помощью библиотеки [пользовательского интерфейса](https://azure.github.io/communication-ui-library/).

Службы коммуникации Azure приложения могут использовать функции общедоступной предварительной версии для [](/azure/communication-services/concepts/teams-interop) взаимодействия с Teams и разрешить пользовательскому приложению анонимно присоединяться к собраниям Teams собраний. Например, вы можете интегрировать видеозвонок в мобильное банковское приложение и разрешить конечным пользователям практические встречи с сотрудниками банка с помощью Microsoft Teams.

Вы также можете интегрировать Microsoft 365 для создания внешних приложений, которые внедряют видео и вызовы ТСОП от имени Teams пользователя. Если вы использовали [Skype для бизнеса пакеты SDK](/skype-sdk/appsdk/skypeappsdk) в прошлом, эти возможности в Службы коммуникации Azure рекомендуется в качестве замены.
