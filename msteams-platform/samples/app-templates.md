# <a name="app-templates-for-microsoft-teams"></a>Шаблоны приложений для Microsoft Teams

Шаблоны приложений — это примеры полных приложений для Microsoft Teams с открытым исходным кодом и доступных GitHub. Каждый шаблон приложения содержит подробные инструкции по развертыванию и установке этого приложения для организации. Он также предоставляет пример приложения, которое можно установить и начать использовать немедленно. Кроме того, доступен полный исходный код, который позволяет подробно изучить его или раздвоить код и изменить его, чтобы соответствовать определенным требованиям.
Все шаблоны приложений предоставляются в соответствии с [условиями лицензии MIT](https://github.com/OfficeDev/microsoft-teams-apps-eprescription/blob/master/LICENSE) .

> [!NOTE]
> Необходимо лицензировать и поддерживать приложения, созданные из шаблонов приложений для пользователей и организаций.

Основные преимущества использования шаблонов приложений:

* **Развертывание непосредственно в облаке:** Все шаблоны приложений включают сценарии развертывания, которые позволяют развертывать все необходимые службы в Microsoft Azure или платформе Power.
* **Рекомендуемый пример кода:** Шаблоны приложений соответствуют рекомендуемой рекомендации по безопасности и инфраструктуре. Все внесенные сообществом изменения в шаблоны приложений проверяются для обеспечения соответствия.
* **Настраиваемые и размязывные:** Несмотря на то, что все шаблоны приложений развернуты с минимальной конфигурацией, предоставляется вся база кода и сценарии развертывания, чтобы их можно было легко настроить или расширить, чтобы соответствовать вашим уникальным потребностям.
* **Подробные документы:** Все шаблоны приложений сопровождаются документацией по архитектуре решений, развертыванию и этапам конфигурации.  

## <a name="adoption-tool--champion-management-platform"></a>Платформа управления средствами принятия и чемпиона

Шаблон приложения Платформы управления чемпионами (CMP) помогает управлять, масштабировать и вдохновлять чемпионов командной работы, чтобы добиться большего. Этот шаблон приложения создается на SharePoint Framework и загружается на вкладку в команде. Группы могут использовать этот инструмент для управления членством в программе, предоставления лидеров и типов событий для ведения журнала, а также инструментов для наложения цифровых значков для участников программы.

## <a name="company-communicator"></a>Корпоративный Communicator

Приложение Communicator позволяет корпоративным командам создавать и отправлять сообщения, предназначенные для нескольких групп или большого количества сотрудников в чате, позволяя организации достигать сотрудников прямо там, где они сотрудничают. Используйте этот шаблон для нескольких сценариев, таких как новые объявления об инициативе, учет сотрудников, современное обучение и разработка или широкомасштабные трансляции.

Приложение обеспечивает простой интерфейс для назначенных пользователей для создания, предварительного просмотра, совместной работы и отправки сообщений.

Он создает основу для создания настраиваемой целевой системы связи, например настраиваемой телеметрии, о том, сколько пользователей было признано или взаимодействует с сообщением.

[Получите его на GitHub](https://github.com/OfficeDev/microsoft-teams-company-communicator-app)

## <a name="faq-plus"></a>Вопросы и ответы плюс

Диалоговая&боты — это простой способ предоставления ответов на часто задамые пользователями вопросы. Но большинство ботов не могут эффективно взаимодействовать с пользователями, так как при сбойе бота в цикле нет человека. FaQ bot — это дружественный Q&бот, который приводит человека в цикл, когда он не в состоянии помочь. Можно задать боту вопрос, и бот отвечает ответом, если он содержится в базе знаний. Если нет, бот позволяет пользователю отправлять запрос, который затем передается предварительно настроенной группе экспертов, которые помогают обеспечить поддержку, действуя на уведомления из самой группы.

> [!NOTE]
> Последний выпуск **FAQ Plus** поддерживает улучшение Q&A, позволяя группе экспертов выполнить следующие действия:
>
> &#x2714; добавить новые Q&Как непосредственно в базу знаний с помощью расширений сообщений.
>
> &#x2714; изменить и удалить Q&пары, добавленные ботом.
>
> &#x2714; отслеживать историю пересмотра Q&As.
>
> &#x2714; настройка ответа с дополнительными сведениями для отображения в качестве [адаптивной карты](../task-modules-and-cards/cards/cards-reference.md#adaptive-card).
>
[Получите его на GitHub](https://github.com/OfficeDev/microsoft-teams-apps-faqplusv2)

## <a name="icebreaker"></a>Точки соприкосновения

Icebreaker это Microsoft Teams[,](../bots/what-are-bots.md) который помогает вашей команде приблизиться, спарив двух случайных членов команды каждую неделю для встречи. Бот упрощает планирование, автоматически предлагая свободное время, которое работает для обоих участников. Укрепляйте личные связи и создайте тесное сообщество с помощью этого приложения.

В дополнение к поощрению личных подключений всей вашей команды приложение Icebreaker может помочь развивать сообщества, основанные на интересе в вашей организации. Например, это приложение можно использовать для группы DevOps, чтобы помочь идеям и лучшим практикам органично распространяться по всей организации.

[Получите его на GitHub](https://github.com/OfficeDev/microsoft-teams-icebreaker-app)
