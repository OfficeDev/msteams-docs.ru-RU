# <a name="app-templates-for-microsoft-teams"></a>Шаблоны приложений для Microsoft Teams

Шаблоны приложений — это примеры полных приложений для Microsoft Teams с открытым исходным кодом и доступных GitHub. Каждый шаблон приложения содержит подробные инструкции по развертыванию и установке этого приложения для вашей организации. Он также предоставляет пример приложения, которое можно установить и начать использовать немедленно. Кроме того, доступен полный исходный код, который позволяет подробно изучить его или создать вилку кода и изменить его в соответствии с конкретными требованиями.
Все шаблоны приложений предоставляются в соответствии с условиями [лицензии MIT](https://github.com/OfficeDev/microsoft-teams-apps-eprescription/blob/master/LICENSE) .

> [!NOTE]
> Для пользователей и организаций необходимо лицензирование и поддержка приложений, созданных на основе шаблонов приложений.

Основные преимущества использования шаблонов приложений:

* **Развертывание непосредственно в облаке:** Все шаблоны приложений включают сценарии развертывания, позволяющие разместить все необходимые службы в Microsoft Azure power Platform.
* **Рекомендуемый пример кода:** Шаблоны приложений соответствуют рекомендациям по безопасности и инфраструктуре. Все изменения, отправленные сообществом в шаблоны приложений, проверяются, чтобы обеспечить соответствие.
* **Настраиваемые и расширяемые возможности:** Хотя все шаблоны приложений развертываются с минимальной конфигурацией, предоставляется вся база кода и скрипты развертывания, что позволяет легко настраивать или расширять их в соответствии с вашими уникальными потребностями.
* **Подробная документация:** Все шаблоны приложений сопровождается сквозной документацией по архитектуре, развертыванию и настройке решения.  

## <a name="adoption-tool--champion-management-platform"></a>Средство внедрения — платформа управления "Лидеров"

Шаблон приложения "Платформа управления для лидеров" (CMP) помогает управлять, масштабировать и подавляет лидеров командной работы для достижения большего. Этот шаблон приложения создается на основе SharePoint Framework и загружается на вкладку в команде. Группы могут использовать это средство для управления членством в программе, предоставления списка лидеров и типов событий для ведения журнала, а также средств для наложения цифровых индикаторов для участников программы.

[Получить его на GitHub](https://github.com/OfficeDev/microsoft-teams-apps-champion-management)

## <a name="adoption-tool--microsoft-365-learning-pathways-get-started"></a>Средства внедрения — Microsoft 365 Обучение pathways (Начало работы)

Шаблон Начало работы позволяет использовать возможности Microsoft 365 схем обучения внутри Microsoft Teams. Этот шаблон приложения позволяет предоставить простой доступ к определенным страницам обучения или другим ресурсам интрасети и загрузить содержимое непосредственно в Teams. Вы также можете изменить имя приложения или логотип в соответствии с фирменной символикой вашей компании.

[Получить его на GitHub](https://github.com/msft-teams/tools/tree/master/M365%20Learning%20Pathways)

## <a name="company-communicator"></a>Корпоративный Communicator

Приложение корпоративного Communicator позволяет корпоративным командам создавать и отправлять сообщения, предназначенные для нескольких команд или большого количества сотрудников, через чат, позволяя организации связаться с сотрудниками прямо там, где они работают совместно. Используйте этот шаблон для нескольких сценариев, таких как объявления о новых инициативах, адаптация сотрудников, современное обучение, разработка или широковещательные трансляции на уровне организации.

Приложение предоставляет назначенным пользователям простой интерфейс для создания, предварительного просмотра, совместной работы и отправки сообщений.

Он предоставляет основу для создания настраиваемых целевых возможностей связи, таких как пользовательская телеметрия, о том, сколько пользователей подтверждает или взаимодействует с сообщением.

[Получить его на GitHub](https://github.com/OfficeDev/microsoft-teams-company-communicator-app)

## <a name="co-worker-appreciation"></a>Совместная рабочая роль

Используя шаблон совместной работы в Microsoft Teams, пользователи могут распознавать достижения коллег в Teams контексте. Когда коллеги выбирают для вознаграждения коллег, получатели и другие участники команды помечаются тегами в беседе канала и получают уведомление о сведениях о награде канала. Эти файлы записываются в Teams, которое является безопасным, переносимым и легкодоступным. Это считается версией шаблона приложения Open Badges на основе PowerApps с списком лидеров.

## <a name="faq-plus"></a>Вопросы и ответы плюс

Диалоговая&боты — это простой способ предоставления ответов на часто задаваемые вопросы пользователями. Но большинство ботов не могут взаимодействовать с пользователями осмысленным образом, так как в случае сбоя бота в цикле нет человека. Часто задаваемые вопросы — это понятный Q&бот, который переводит человека в цикл, когда он не может помочь. Можно задать боту вопрос, и бот ответит ответом, если он содержится в база знаний. В противном случае бот позволяет пользователю отправить запрос, который затем отправляется предварительно настроенной группе экспертов, которые помогают предоставлять поддержку, действуя на основе уведомлений из самой команды.

> [!NOTE]
> Последний выпуск часто задаваемых **вопросов и ответов plus** поддерживает улучшенное решение Q&A, позволяя команде экспертов выполнить следующие действия:
>
> &#x2714; Q&как непосредственно в база знаний с помощью расширений сообщений.
>
> &#x2714; изменения и удаления Q&A, добавленных ботом.
>
> &#x2714; журнал исправлений Q&As.
>
> &#x2714; ответ с дополнительными сведениями для отображения в виде [адаптивной карточки](../task-modules-and-cards/cards/cards-reference.md#adaptive-card).
>
[Получить его на GitHub](https://github.com/OfficeDev/microsoft-teams-apps-faqplusv2)

## <a name="icebreaker"></a>Точки соприкосновения

Icebreaker — это [Microsoft Teams](../bots/what-are-bots.md) бот, который помогает вашей команде ближе связывать двух случайных участников команды каждую неделю для встречи. Бот упрощает планирование, автоматически предлагая свободное время для обоих участников. Укрепляйте личные подключения и создавайте тесное сообщество с помощью этого приложения.

Помимо поддержки личных подключений в вашей команде, приложение Icebreaker помогает создавать сообщества на основе интересов в вашей организации. Например, это приложение можно использовать для группы DevOps, чтобы помочь идеям и рекомендациям органично распространяться по всей организации.

[Получить его на GitHub](https://github.com/OfficeDev/microsoft-teams-icebreaker-app)

![Icebreaker приложения](../assets/images/icebreaker.png)
