# <a name="app-templates-for-microsoft-teams"></a>Шаблоны приложений для Microsoft Teams

Шаблоны приложений — это примеры готовых приложений для Microsoft Teams с открытым исходным кодом, доступные на портале GitHub. Каждый шаблон приложения содержит подробные инструкции по развертыванию и установке этого приложения в вашей организации. В шаблоне также содержится образец приложения: можно немедленно установить его и начать использовать. Кроме того, доступен полный исходный код: вы можете изучать его в подробностях или создать собственное ответвление кода и изменить его в соответствии с вашими требованиями.
Все шаблоны приложений предоставляются здесь на условиях [лицензии МТИ](https://github.com/OfficeDev/microsoft-teams-apps-eprescription/blob/master/LICENSE).

> [!NOTE]
> Вы обязаны лицензировать и поддерживать приложения, созданные на основе шаблонов приложений для ваших пользователей и организаций.

Основные преимущества использования шаблонов приложений:

* **Развертывание непосредственно в облаке:** Все шаблоны приложений включают сценарии развертывания, которые позволяют разместить все необходимые службы в Microsoft Azure или Power Platform.
* **Рекомендуемый пример кода:** шаблоны приложений соответствуют рекомендациям в отношении безопасности и инфраструктуры. Все изменения шаблонов приложений, отправленные сообществом, проверяются для поддержания соответствия.
* **Возможность настройки и расширения:** все шаблоны приложений развертываются в минимальной конфигурации, но предоставляется вся база кода и сценарии развертывания, поэтому можно настраивать и расширять шаблоны в соответствии с вашими потребностями.
* **Подробная документация:** для всех шаблонов приложений предоставляется полный комплект документации, в которой описывается архитектура решения, его развертывание и действия по настройке.  

## <a name="adoption-tool--champion-management-platform"></a>Средство внедрения — Champion Management Platform

Шаблон приложения Champion Management Platform (CMP) дает возможность управлять работой энтузиастов в составе команд и помогать им добиваться более значимых результатов. Этот шаблон приложения создан на основе SharePoint Framework и загружается на вкладку в команде. С помощью этого инструмента группы могут управлять членством в программе, предоставлять список лидеров и типы событий для записи в журнал, а также предоставлять инструменты для отображения цифровых эмблем для участников программы.

[Получить на GitHub](https://github.com/OfficeDev/microsoft-teams-apps-champion-management)

## <a name="adoption-tool--microsoft-365-learning-pathways-get-started"></a>Средство внедрения — схемы обучения для Microsoft 365 Learning Pathways (Get Started)

Шаблон приложения Get Started позволяет использовать возможности схем обучения Microsoft 365 в Microsoft Teams. Этот шаблон приложения дает возможность предоставлять простой доступ к определенным учебным страницам и другим ресурсами в интрасети и загружать содержимое непосредственно в Teams. Также можно изменить имя этого приложения и его логотип в соответствии с фирменной символикой вашей компании.

[Получить на GitHub](https://github.com/msft-teams/tools/tree/master/M365%20Learning%20Pathways)

## <a name="company-communicator"></a>Корпоративный Communicator

С помощью приложения Company Communicator команды предприятий могут создавать и отправлять в чате сообщения, предназначенные для нескольких команд или большого количества сотрудников, чтобы организация могла передавать нужную информацию сотрудникам по используемым ими каналам совместной работы. Этот шаблон можно использовать для различных сценариев, включая объявления о новых инициативах, адаптация сотрудников, современное обучение, разработка, проведение трансляций в пределах организации.

В этом приложении назначенным пользователям доступен удобный интерфейс для создания, совместного редактирования, предварительного просмотра и отправки сообщений.

Это приложение можно использовать в качестве основы для создания специализированных функций обмена информацией. Например, можно создавать настраиваемые системы телеметрии, получающие данные о том, сколько пользователей подтвердили сообщение или взаимодействовали с ним.

[Получить на GitHub](https://github.com/OfficeDev/microsoft-teams-company-communicator-app)

## <a name="co-worker-appreciation"></a>Co-worker Appreciation

Используя шаблон Co-worker Appreciation в Microsoft Teams, пользователи могут отмечать достижения своих коллег в контексте Teams. Когда сотрудники решают наградить коллегу, получатели и другие участники команды отмечаются в беседе в канале и получают уведомление с подробными сведениями о награде в этом канале. Эти награды записываются в приложении Teams: за счет этого обеспечивается безопасность, переносимость и удобное предоставление доступа. Это решение можно считать версией шаблона приложения Open Badges на базе PowerApps со списком лидеров.

[Получить на GitHub](https://github.com/OfficeDev/microsoft-teams-apps-coworker-appreciation)

## <a name="faq-plus"></a>Вопросы и ответы плюс

Боты для ответов на вопросы в беседах — удобное средство предоставления ответов на частые вопросы пользователей. Но большинство ботов не могут взаимодействовать с пользователями осмысленным образом, так как в случае сбоя бота в цикле нет человека. Часто задаваемые вопросы — это понятный Q&бот, который выводит человека в цикл, когда он не может помочь. Можно задать боту вопрос, и бот ответит ответом, если он содержится в база знаний. В противном случае бот позволяет пользователю отправить запрос, который затем отправляется предварительно настроенной группе экспертов, которые помогают предоставлять поддержку, действуя на основе уведомлений из самой команды.

> [!NOTE]
> В последнем выпуске приложения **FAQ Plus** поддерживается улучшенное предоставление ответов на вопросы. Для этого команде экспертов предоставляются следующие возможности:
>
> &#x2714; добавление новых вопросов и ответов непосредственно в базу знаний с использованием расширений сообщений;
>
> &#x2714; редактирование и удаление пар вопросов и ответов, добавленных ботом;
>
> &#x2714; отслеживание журнала изменений вопросов и ответов;
>
> &#x2714; настройка ответов с дополнительными сведениями с отображением в виде [адаптивной карточки](../task-modules-and-cards/cards/cards-reference.md#adaptive-card).
>
[Получить на GitHub](https://github.com/OfficeDev/microsoft-teams-apps-faqplusv2)

## <a name="icebreaker"></a>Точки соприкосновения

Icebreaker — это бот [Microsoft Teams](../bots/what-are-bots.md), помогающий наладить более тесное общение в составе команды. Для этого бот каждую неделю устраивает встречу двух участников команды, выбранных случайным образом. Этот бот также упрощает планирование: он автоматически предлагает время, свободное для обоих участников. Это приложение укрепляет связи на личном уровне и способствует формированию более сплоченного сообщества.

Приложение Icebreaker не только поддерживает общение на личном уровне в пределах команды, но и помогает формировать сообщества по интересам в пределах организации. Например, с помощью этого приложения группа пользователей, интересующихся DevOps, может распространять идеи и рекомендации по всей организации.

[Получить на GitHub](https://github.com/OfficeDev/microsoft-teams-icebreaker-app)

## <a name="new-employee-onboarding"></a>New Employee Onboarding

New Employee Onboarding — это интегрированное решение Microsoft Teams и [SharePoint для подключения новых сотрудников](https://lookbook.microsoft.com/details/75e60a32-9849-4ed4-b83e-b2b08983ad19). С помощью этого приложения отделы кадров и менеджеры по найму могут предоставлять важную информацию в процессе приема на работу и адаптации, а новые сотрудники могут оставлять отзывы, предоставлять нужные сведения и выполнять задачи, связанные с началом работы.
