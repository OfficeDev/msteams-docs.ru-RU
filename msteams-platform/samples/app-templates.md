# <a name="app-templates-for-microsoft-teams"></a>Шаблоны приложений для Microsoft Teams

Шаблоны приложений — это примеры полных приложений для Microsoft Teams с открытым исходным кодом и доступных GitHub. Каждый шаблон приложения содержит подробные инструкции по развертыванию и установке этого приложения для организации. Он также предоставляет пример приложения, которое можно установить и начать использовать немедленно. Кроме того, доступен полный исходный код, который позволяет подробно изучить его или раздвоить код и изменить его, чтобы соответствовать определенным требованиям.
Все шаблоны приложений предоставляются в соответствии с [условиями лицензии MIT.](https://github.com/OfficeDev/microsoft-teams-apps-eprescription/blob/master/LICENSE)

> [!NOTE] 
> Необходимо лицензировать и поддерживать приложения, созданные из шаблонов приложений для пользователей и организаций.

**&#9734; указывает только что выпущенные шаблоны приложений.**

### <a name="key-benefits"></a>Основные преимущества

* **Развертывание непосредственно в облаке:** Все шаблоны приложений включают сценарии развертывания, которые позволяют развертывать все необходимые службы в Microsoft Azure или на платформе Power. 
* **Рекомендуемый пример кода:** Шаблоны приложений соответствуют рекомендуемой рекомендации по безопасности и инфраструктуре. Все внесенные сообществом изменения в шаблоны приложений проверяются для обеспечения соответствия.
* **Настраиваемые и размязывные:** Несмотря на то, что все шаблоны приложений развернуты с минимальной конфигурацией, предоставляется вся база кода и сценарии развертывания, чтобы их можно было легко настроить или расширить, чтобы соответствовать вашим уникальным потребностям.
* **Подробные документы:** Все шаблоны приложений сопровождаются документацией по архитектуре решений, развертыванию и этапам конфигурации.  

## <a name="adoption-bot"></a>Бот по внедрению 

Adoption Bot — это чат-бот по уходу за пользователями, построенный с помощью виртуального агента power для Teams PVA. Он считается PVA-версией FAQ Plus. Бот-принятия отвечает на более 100 общих вопросов о Microsoft 365 и Teams. Можно редактировать существующие темы, добавлять собственные темы и гнать существующие вопросы. Если пользователям нужна дополнительная помощь, Бот-пользователь может подключить их к экспертам или даже расширить доступ к билетам на обслуживание с соединиттелями потока премиум-класса. Этот бот самостоятельно установлен или встроен в пользовательское приложение, например концентратор [принятия.](https://github.com/akporzondek/adoption_hub)

## <a name="appointment-manager"></a>Менеджер по встречам 

Диспетчер назначений — это шаблон Teams, который помогает предприятиям создавать, управлять и проводить виртуальные встречи с потребителями через Teams. Новые запросы на прием от потребителей видны в Teams каналах, где они быстро назначены и назначены сотрудникам в команде. Запросы на назначение рассматриваются на командном или личном уровнях с помощью настраиваемой вкладки. Каждое собрание связано с Teams, поэтому сотрудники и потребители могут легко присоединиться к собранию в запланированное время.

Шаблон приложения интегрируется с Microsoft Bookings для простого управления назначением. Запланированные встречи автоматически отображаются в календарях назначенного персонала, а потребители получают настраиваемые уведомления электронной почты и напоминания со встроенными ссылками на собрания.

[Получите его на GitHub](https://github.com/OfficeDev/microsoft-teams-apps-appointment-manager)

## <a name="employee-ideas"></a>Идеи сотрудников

* Просмотрите советы лидеров идей.
* Голосуйте за и делитесь приоритетными идеями.
* Отправка идей для кампании.
* Просмотр идеи другого члена группы.
* Голосуйте за наиболее понравились идеи.
* Просмотрите производительность их идей по сравнению с другими в рамках кампании.

## <a name="e-prescriptions"></a>Электронные рецепты

>

>
> &#x2714; изменить и удалить Q&пары, добавленные ботом.
>
> &#x2714; отслеживание истории пересмотра Q&As.
>
> &#x2714; настройка ответа с дополнительными сведениями для отображения в качестве [адаптивной карты.](../task-modules-and-cards/cards/cards-reference.md#adaptive-card)

## <a name="sharepoint-list-search"></a>Поиск в списке SharePoint

Совместная Microsoft Teams часто ссылается на сведения, содержащиеся в элементов в SharePoint списке. Вклей ссылку на этот элемент заставляет всех переключать контекст из беседы, находить необходимые сведения, а затем возвращаться Teams для продолжения беседы. По мере того как беседа продолжается, люди должны несколько раз переключаться на справочный элемент, чтобы проверить новые комментарии и обновить воспоминания о сведениях, содержащихся в элементе. Это переключение контекста создает барьер для плавного взаимодействия.
Для решения этой проблемы используется шаблон приложения поиска списков. Многие пользователи используют SharePoint для питания некоторых основных процессов в своих организациях. Однако совместная работа со списками затруднена. Используя шаблон приложения поиска списка в Microsoft Teams, пользователи могут вставлять сведения из SharePoint элементов списка непосредственно в чате, чтобы облегчить переключение контекста, вызванное при простом включении ссылки в чат. Эти сведения вставляются в виде легко читаемой автоформатной карты, помогая пользователям оставаться в стороне от беседы.

[Получите его на GitHub](https://github.com/OfficeDev/microsoft-teams-list-search-app)

![Приложение Поиска списка](../assets/images/list-search-template.png)

## <a name="time-tally"></a>Время Tally 

Проект может включать несколько задач, а сотрудникам могут быть назначены различные проекты. Руководители получают возможность просматривать использование проекта на основе ресурсов и могут утверждать или отклонить записи. Уведомления о напоминаниях отправляются для обеспечения соответствия времени. Кроме того, для аналитики доступны исторические данные и использование.

## <a name="water-cooler-9734"></a>Охладители &#9734;

Water Cooler — это настраиваемое приложение Teams, которое позволяет корпоративным командам создавать, приглашать и присоединяться к случайным беседам между одноклубниками, такими как те, которые проходят в кулере воды или в комнате перерывов. Это основа для создания настраиваемой целевой возможности связи, способствов взаимодействия между коллегами, которые в противном случае не могут получить возможность общения во время перерывов.

### <a name="key-features"></a>Основные возможности

**Домашняя страница** Water Cooler: Вы можете просматривать существующие комнаты, в которых члены группы взаимодействуют в существующих беседах с определенными людьми или темами, которые интересуют. Активные беседы на **домашней странице** показывают имя комнаты, краткое описание, продолжительность вызова и изображение комнаты. 

![Домашняя страница кулера воды](../assets/images/home-page.png)

**Присоединиться к** комнате. Используйте **функцию Присоединиться к комнате,** чтобы немедленно присоединиться к текущей беседе. Выберите **Присоединиться к** активным беседам, чтобы присоединиться к комнате.

![Присоединиться к комнате](../assets/images/joinRoom.gif)

**Создание комнаты.** Используйте функцию создания комнаты для создания Teams или чата для взаимодействия всех участников.  Создайте комнаты легко, указав имя комнаты, краткое описание, до пяти коллег в качестве начальной группы и выбрав из предоставленного набора изображений комнаты. 

![Создание комнат](../assets/images/createRoom.gif)

**Find room:** Используйте функцию **Find room** для поиска ключевого слова, которое совпадает с темой или краткими описаниями текущих бесед.

![Приглашение участника](../assets/images/attendeeInvitation.gif)
