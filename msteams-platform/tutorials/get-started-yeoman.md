# <a name="create-your-first-microsoft-teams-app-using-the-yeoman-generator"></a>Создайте свое первое Microsoft Teams с помощью генератора Yeoman

> [!Note]
> Этот учебник исходит от [генератора Yeoman для Teams вики](https://github.com/OfficeDev/generator-teams/wiki/Build-Your-First-Microsoft-Teams-App).

В этом учебнике мы пройдитесь по созданию вашего самого первого Microsoft Teams с помощью Microsoft Teams генератора Yeoman. Он также проходит через процесс модернизации вашего Teams с помощью генератора Yeoman. Предпосылкой для начала с этого учебника является то, что у вас есть Teams, которая позволяет [приложение sideloading](~/concepts/build-and-test/prepare-your-o365-tenant.md).

![генератор yeoman git](~/assets/yeoman-demo.gif)

## <a name="setup-and-prepare-your-machine"></a>Настройка и подготовка машины

Вы должны установить следующее на вашей машине, прежде чем начать использовать генератор Yeoman.

### <a name="install-nodejs"></a>Установите Node.js.

Вы должны иметь Node.js установлен на вашей машине. Вы должны использовать последнюю [версию LTS](https://nodejs.org).

### <a name="install-a-code-editor"></a>Установка редактора кода

Вам нужен редактор кода. Большая часть этой документации и изображений относится к [использованию Visual Studio Code](https://code.visualstudio.com). Тем не менее, не стесняйтесь использовать любой текстовый редактор вы предпочитаете.

### <a name="install-yeoman-and-gulp-cli"></a>Установка Yeoman и Gulp CLI

Для строительных лесов проектов с помощью генератора, вы должны установить инструмент Yeoman и Gulp CLI менеджер задач.

Откройте подсказку команды и ввеми следующие:

```bash
npm install yo gulp-cli --global
```

## <a name="install-the-generator"></a>Установка генератора

Установите Teams Yeoman со следующей командой:

```bash
npm install generator-teams --global
```

Установите предварительную версию генератора со следующей командой:

```bash
npm install generator-teams@preview --global
```

## <a name="generate-your-project"></a>Создание проекта

Этот раздел проготовит шаги по созданию проекта.

**Создание проекта**

1. Откройте подсказку команды и создайте новый каталог, где вы хотите создать свой проект, и в этом каталоге запустите `yo teams` команду. Генератор запускается.
1. Ответьте на набор вопросов, подсказав генератором:

   ![йо команды](~/assets/yeoman-images/teams-first-app-1.png)

   1. Первый вопрос о названии проекта, вы можете оставить его как есть, нажав введите.
   1. Следующий вопрос задает вам, хотите ли вы создать новый каталог или использовать текущий. Как вы уже в каталоге вы хотите, просто нажмите введите.
   1. В следующем вопросе ввемите название проекта. Это название будет использовано в манифесте и описании вашего приложения. 
   1. Далее вас попросят название компании, которое также будет использовано в манифесте.
   1. Пятый вопрос спрашивает вас о том, какую версию манифеста вы хотите использовать. Для этого учебника `v1.5` выберите , который является текущей общей доступной схеме.
   1. Далее генератор спросит вас, какие элементы вы хотите добавить в свой проект. Вы можете выбрать одну или любую комбинацию элементов. Для этого учебники, просто *выберите Tab:*

    ![выбор элемента](~/assets/yeoman-images/teams-first-app-2.png)

1. Ответьте на следующий набор последующих вопросов, которые отображаются на основе элементов, выбранных в шаге 2.
1. Введите URL-адрес, где вы будете разместить свое решение. 

   > [!NOTE]
   > URL может быть любым URL, но по умолчанию генератор предлагает URL веб-узла Azure.

1. В следующем вопросе подтвердите, хотите ли вы включить в свое решение унитарное тестирование. Ответ по умолчанию *да*. Если вы решите включить в него унитарное тестирование, то генерируемый проект будет иметь рамку тестирования единицы и некоторые единицы тестов по умолчанию для различных элементов, которые вымолись под строительные леса. 
   > [!NOTE]
   > * Для этого учебника не включать тестовую основу.
   > * Генератор имеет много встроенных расширенных функций, которые вы можете отказаться или отказаться от.

1. Для того, чтобы сделать регистрацию легкой для вас, вас также попросят, хотите ли вы использовать Azure Application Insights для регистрации. Если вы *выберете* Да, вам нужно будет предоставить ключ Azure Application Insights. 

   > [!NOTE]
   > Для этого учебника отказаться от использования приложения Исследования.

Следующий набор вопросов будет основываться на ранее выбранных пунктах. Для вкладки вам нужно только узнать имя и дополнительно выбрать, если вы хотите иметь возможность использовать это приложение в качестве SharePoint интернет-части. После предоставления имени генератор будет генерировать проект и устанавливать все зависимости. Это займет минуту или две.

## <a name="add-some-code-to-your-tab"></a>Добавить код в вкладку

После того, как генератор будет сделан, вы можете открыть решение в любимом редакторе кода. Возьмите минуту или две и ознакомитесь с тем, как организован код. Для получения дополнительной информации [с Project м.](https://github.com/OfficeDev/generator-teams/wiki/Project-Structure)

Ваша вкладка в `./src/app/scripts/myFirstAppTab/MyFirstAppTab.tsx` файле. Это класс TypeScript React основе для вашей вкладки. Найдите `render()` метод и добавьте строку кода внутри `<PanelBody>` управления, чтобы она выглядела так:

``` TypeScript
<PanelBody>
    <div style={styles.section}>
    Hello World! Yo Teams rocks!
    </div>
</PanelBody>
```

Сохраните файл и вернитесь к подсказке команды.

## <a name="build-your-app"></a>Создание приложения

Теперь вы можете построить свой проект. Это делается в два этапа (или один шаг, см. ниже).

Сначала вам нужно создать Teams app манифест файл, который вы загружаете / sideload в Microsoft Teams. Это делается по задаче Gulp `gulp manifest` . Это позволит проверить манифест и создать почтовый файл в `./package` каталоге.

Для создания решения используется `gulp build` команда. Это перекапит ваше решение в `./dist` папку. 

## <a name="run-your-app"></a>Запуск приложения

Для запуска приложения используется `gulp serve` команда. Это позволит создать и начать локальный веб-сервер для тестирования приложения. Команда также восстанавливает приложение всякий раз, когда вы сохраните файл в проекте. 

Теперь вы должны быть в состоянии просматривать, `http://localhost:3007/myFirstAppTab/` чтобы убедиться, что ваша вкладка рендеринга. Однако, еще Microsoft Teams:

![просмотр вашего сайта в браузере](~/assets/yeoman-images/teams-first-app-3.png)

## <a name="run-your-app-in-microsoft-teams"></a>Запустите приложение в Microsoft Teams

Microsoft Teams не позволяет вам иметь ваше приложение размещено на localhost, так что вам нужно либо опубликовать его на общедоступный URL или использовать прокси, такие как ngrok.

Хорошей новостью является то, что леса проекта имеет этот встроенный. При запуске `gulp ngrok-serve` ngrok служба будет запущена в фоновом режиме, с уникальной и публичной записи DNS, и он также пакет манифест с этим уникальным URL, а затем сделать то же самое, как `gulp serve` .

После запуска `gulp ngrok-serve` создайте новую команду Microsoft Teams и когда она будет создана, нажмите на имя команды, чтобы перейти к настройкам команд, а затем выбрать *Apps.* В правом нижнем углу вы должны увидеть ссылку *Upload пользовательское приложение,* выберите его, а затем просматривать папку проекта и subfolder называется `package` . Выберите файл zip в этой папке и выберите открытый. Теперь ваше приложение загружено в Microsoft Teams:

![боковое приложение](~/assets/yeoman-images/teams-first-app-4.png)

Вернитесь к *Общему* каналу и *+* выберите добавление новой вкладки. Вы должны увидеть свою вкладку в списке вкладок:

![настройка вкладки](~/assets/yeoman-images/teams-first-app-5.png)

Выберите вкладку и следуйте инструкциям, чтобы добавить ее. Обратите внимание, что у вас есть пользовательский диалог конфигурации, для которого вы можете редактировать источник. Выберите *Сохранить,* чтобы добавить вкладку в канал. После того, как сделали вкладку должны быть загружены Microsoft Teams!

![работает вкладка в командах](~/assets/yeoman-images/teams-first-app-6.png)

## <a name="upgrade-microsoft-teams"></a>Обновление Microsoft Teams

Вы также можете обновить текущую Microsoft Teams версию до последней версии с помощью Microsoft Teams генератора Yeoman.

**Для обновления Microsoft Teams**

1. Получите текущую версию Teams со следующей командой:

   ```PowerShell
    yo teams --version
   ```
2. Используйте следующую команду, чтобы выбрать обновление генератора:

   ```PowerShell
    yo
   ```
3. Используйте клавиши **стрелки, чтобы выбрать обновление генераторов:**

   ![изображение YoSelectUpdatGen](~/assets/images/Update-Teams/YoSelectUpdateGen.png)

4. Выберите генератор, который вы хотите из списка генераторов:
   > [!NOTE]
   > Используйте планку пространства, чтобы выбрать или очистить выбранную Teams версию из доступных опций.

    ![изображение UseSpaceToSelectGenerators](~/assets/images/Update-Teams/UseSpaceToSelectGenerators.png)
    

   > [!NOTE]
   > Установка системы занимает от нескольких секунд Teams минут.

5. После завершения установки используйте следующую команду для проверки установленной версии:

   ```PowerShell
    yo teams --version
   ```
   
**Поздравляю! Вы создали и развернули свое первое Microsoft Teams приложение. Вы также обновили Microsoft Teams.**
