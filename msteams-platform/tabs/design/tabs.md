# <a name="content-and-conversations-all-at-once-using-tabs"></a>Контент и беседы, все сразу с помощью вкладок

> [!Important]
> **Вкладки на мобильных клиентах**
>
> Следуйте [указаниям для вкладок на странице Mobile](./tabs-mobile.md) при создании вкладок. Если вкладка использует проверку подлинности, необходимо обновить пакет SDK для Teams для Teams до версии 1.4.1 или более поздней, иначе проверка подлинности завершится неуспешно.
>
> **Личные (статические) вкладки на мобильных устройствах:**
>
> * Статические вкладки (личное приложение) доступны в [предварительной версии для разработчиков](~/resources/dev-preview/developer-preview-intro.md).
> * При создании статических вкладок обязательно следуйте [указаниям для вкладок на мобильных устройствах](~/tabs/design/tabs-mobile.md) .
>
> **Вкладки каналов и групп (настраиваемых) на мобильных устройствах:**
>
> * Для `websiteUrl`мобильных клиентов отображаются только вкладки со значением. Если вы хотите, чтобы вкладка отображалась в клиентах Teams для мобильных устройств, необходимо задать `websiteUrl`значение.
> * Поведение по умолчанию при открытии на мобильном устройстве можно открыть снаружи `websiteUrl`в браузере с помощью. Для приложений, опубликованных в магазине общедоступных приложений, если вы хотите, чтобы вкладка каналы открывалась внутри Teams по умолчанию, следуйте [указаниям по использованию вкладок на мобильном устройстве](~/tabs/design/tabs-mobile.md), чтобы получить доступ к внесены.

Вкладки — это холсты, которые можно использовать для обмена контентом, удержания бесед и размещения сторонних служб в рабочем процессе. Когда вы создаете вкладку в Microsoft Teams, он помещает веб-приложение спереди и в центре, где оно легко доступно из ключевых бесед.

## <a name="guidelines"></a>Рекомендации

На хорошей вкладке должны отображаться следующие характеристики:

### <a name="focused-functionality"></a>Ориентированные функциональные возможности

Вкладки лучше подходят для конкретных нужд. Сосредоточьтесь на небольшом наборе задач или подмножестве данных, относящихся к каналу, в котором находится вкладка.

### <a name="reduced-chrome"></a>Сокращенный хром

Избегайте создания нескольких панелей на вкладке, добавления слоев навигации или необходимости прокручивать их по вертикали и горизонтали на одной вкладке. Другими словами, старайтесь не использовать вкладки на вкладке.

### <a name="integration"></a>Интеграция

Узнайте о том, как уведомлять пользователей об активности вкладок путем разноски карточек в беседу, например.

### <a name="conversational"></a>Разговорному

Найдите способ для упрощения общения вокруг вкладки. Это гарантирует, что центр бесед на содержании, данные или процесс в любой момент.

### <a name="streamlined-access"></a>Упрощенный доступ

Убедитесь, что вы предоставляете доступ к нужным людям в нужное время. Сохранение процесса входа в систему не позволит создавать барьеры для участия и совместной работы.

### <a name="responsiveness-to-window-sizing"></a>Скорость отклика при изменении размера окна

Teams можно использовать в размерах окон как небольшие 720px, поэтому убедитесь, что вкладка используется в небольшом окне так же, как и удобство при высоком разрешении.

### <a name="flat-navigation"></a>Плоская Навигация

Мы просим разработчиков не добавлять весь портал на вкладку. хранение сравнительно плоской структуры помогает поддерживать более простую модель беседы. Другими словами, беседа представляет собой список вещей, таких как рассмотренные рабочие элементы, или одна вещь, например Spec.

В цепочке обсуждений есть проблемы, связанные с навигацией с углубленной иерархией навигации. Для оптимального взаимодействия с пользователем вы можете сохранить навигацию на вкладке по меньшей мере и иметь следующий вид:

> [!div class="checklist"]
>
> * **Открывает модуль задачи, такой как отдельный рабочий элемент или сущность**. Это исключает возможность полностью отменять чат и является лучшим вариантом, чтобы отследить обсуждение вкладки, а не вложенных сущностей или возможностей редактирования.
>* **Открывает псевдо диалоговое окно в IFRAME**. При использовании с экранным фоном рекомендуется использовать более светлый цвет, а не темный. `app-gray-10 at 30%` Прозрачность работает хорошо.
>* **Открывает страницу браузера**.

### <a name="personality"></a>Индивидуальные

На холсте вкладок можно получить прекрасную возможность для фирменного стиля работы. Ваш логотип является важной частью вашего удостоверения и подключения к вашим пользователям. Убедитесь, что он должен быть включен:

> [!div class="checklist"]
>
>* Размещение логотипа в левом или правом углу или вдоль нижнего края
> * Хранение и ненавязчивая эмблема

Включение собственных цветов и макетов твилл также помогает в общении.

## <a name="configuration-page-height"></a>Высота страницы конфигурации

>[!IMPORTANT]
>В сентябре 2018 высота [страницы настройки](~/tabs/how-to/create-tab-pages/configuration-page.md) вкладки была увеличена, а ширина осталась неизменной. Если ваше приложение предназначено для устаревшего размера вкладки, на странице Конфигурация вкладки будет много вертикальных пробелов. Устаревшие приложения хранилища, исключенные из этого изменения, должны обратиться в корпорацию Майкрософт после обновления, чтобы вместить новые измерения.

### <a name="guidelines-for-tab-configuration-page-format"></a>Рекомендации по формату страницы настройки вкладки

* Основа минимальной высоты области контента на странице настройки вкладки для графических элементов с фиксированной высотой.
* Рассчитайте доступное вертикальное пространство (высоту области контента на странице Конфигурация) с помощью `window.innerHeight`. При этом возвращается размер страницы конфигурации `<iframe>` , в которой размещается страница конфигурации, которая может измениться в будущих выпусках. При использовании этого значения контент будет автоматически скорректирован до будущих изменений.
* Выделение вертикального пространства для элементов переменной высоты за вычетом того, что требуется для элементов фиксированной высоты.
* Для состояния *входа* по вертикали и горизонтали по центру содержимого.
* Если вы хотите использовать фоновое изображение, вам потребуется новое изображение, в соответствии с областью (предпочтительный), или можно оставить одно и то же изображение и выбрать один из следующих вариантов:
  * Выравнивание по верхнему левому углу.
  * масштабирование изображения по размеру.

## <a name="best-practices"></a>Рекомендации

### <a name="always-include-a-default-state"></a>Всегда включать состояние по умолчанию

Включите состояние по умолчанию, чтобы упростить настройку вкладок, даже если вкладка настраиваемая.

### <a name="deep-linking"></a>Глубокая компоновка

По мере возможности карточки и боты должны иметь глубокую ссылку на Расширенные данные на вкладке размещения. Например, карточка может показывать сводную информацию об ошибках, но при щелчке она может отобразить всю ошибку на вкладке.

### <a name="naming"></a>Переименовывает

Во многих случаях имя приложения будет иметь отличное название вкладки. Кроме того, рекомендуется называть вкладки в соответствии с функциями, которые они предоставляют.

## <a name="notifications-for-tabs"></a>Уведомления для вкладок

Существует два режима уведомления об изменениях содержимого на вкладке:

> [!div class="checklist"]
>
> * **Использование API приложений для уведомления пользователей об изменениях**. Это сообщение будет отображаться в веб-канале активности пользователя и имеет глубокую ссылку на вкладку. *Дополнительные*  [ссылки на материалы и функции в Microsoft Teams](../../concepts/build-and-test/deep-links.md?view=msteams-client-js-latest)
> * **Использование ленты**. Этот метод является предпочтительным, особенно если выбран целевой поток вкладок. В результате цепочка вкладок будет перемещена в представление "недавно активные". Этот метод также позволяет использовать некоторые сложности при отправке уведомления.

  При отправке сообщения в поток вкладок увеличивается осведомленность о действиях для всех пользователей без явного уведомления всех. Это относится без шума. Кроме того, если для `@mention` определенных пользователей одно и то же уведомление будет размещаться в своем канале, то детально привязать их к потоку вкладок напрямую.
