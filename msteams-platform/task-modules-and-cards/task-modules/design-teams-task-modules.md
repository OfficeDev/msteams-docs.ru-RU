# <a name="designing-task-modules-for-your-microsoft-teams-app"></a>Разработка модулей задач для приложения Microsoft Teams

Вы можете создавать модальные контекстные меню в приложении Teams с помощью модулей задач. Эта возможность предназначена для отображения мультимедийных данных и данных или выполнения сложной задачи.

## <a name="open-a-task-module"></a>Открытие модуля задачи

Модули задач можно запускать практически из любого места в приложении.

* **Tab**: модуль задачи можно запустить из любой ссылки на вкладке или в IFRAME. Используйте в сценариях, где пользователь должен сосредоточиться на взаимодействии.
* **Bot**: модуль задачи можно запустить из ссылки внутри сообщения Bot.
* **Адаптивная карта**: модуль задачи можно запустить из адаптивной карточки (отправляемой с помощью расширения обмена сообщениями или с помощью Bot), когда пользователь нажимает кнопку.
* **Расширение системы обмена сообщениями (команды действий)**: расширения обмена сообщениями позволяют выполнять определенные действия с содержимым сообщения. При выборе действия открывается модуль задачи.
* **Расширение системы обмена сообщениями (контекст окна создания)**: в поле Создать можно создать расширение для обмена сообщениями, чтобы открыть модуль задач вместо типичного всплывающего меню. Резервирование модулей задач для сложных взаимодействий, таких как заполнение формы.

## <a name="anatomy"></a>Составляющие

:::image type="content" source="../../assets/images/task-module/task-module-anatomy.png" alt-text="Иллюстрация, демонстрирующая пользовательский интерфейс Анатомия модуля задачи." border="false":::

Модули задач являются очень гибкими поверхностями. Они могут быть созданы с помощью iframes, изменяя другие шаблоны пользовательского интерфейса для размещения опыта, созданного партнером. Это предпочтительно для наиболее комфортного взаимодействия.

Кроме того, они могут быть созданы с помощью инфраструктуры [адаптивных карт](../../task-modules-and-cards/cards/design-effective-cards.md) , что может быть простым и быстрым способом выполнения распространенных сценариев (таких как формы).

|Счетчик|Описание|
|----------|-----------|
|1|**Значок приложения**|
|2 |**Имя приложения**: полное имя приложения.|
|3 |**Верхний колонтитул**: Сделайте заголовки четкими и краткими. Опишите задачу, которую должны выполнить пользователи.
|4 |**Кнопка "Закрыть"**: позволяет пользователям находить контент приложений, которые они хотят вставить.|
|5 |**IFRAME**: пространство для ответа, в котором размещается контент вашего приложения.|
|6 |**Actions (необязательно)**: кнопки, связанные с контентом приложения.|

## <a name="designing-with-ui-templates"></a>Разработка с помощью шаблонов пользовательского интерфейса

Рекомендуется использовать шаблоны для общих макетов внутри модулей задач. Каждый из них состоит из меньших компонентов, чтобы создать элегантный, быстро реагирующий проект, который можно использовать для вашего сценария или для вашего фирменного оформления и оформления.

* [Список](../../concepts/design/design-teams-app-ui-templates.md#list): списки могут отображать связанные элементы в формате с возможностью записи, а пользователи выполнять действия для всего списка или отдельных элементов.
* [Форма](../../concepts/design/design-teams-app-ui-templates.md#form): формы предназначены для сбора, проверки и отправки пользовательских данных структурированным способом.
* [Пустое состояние](../../concepts/design/design-teams-app-ui-templates.md#empty-state): шаблон пустое состояние можно использовать для многих сценариев, в том числе входа в систему, первого запуска, сообщений об ошибках и многого другого.

### <a name="simplicity"></a>Простоты

#### <a name="do-keep-it-short"></a>Do: оставьте это краткое

Вы можете легко создать многошаговый мастер, но это не обязательно означает! Многоэкранный модуль задач может вызывать проблемы, так как входящие сообщения отправляются из-за нежелательных пользователей. Если ваша задача на самом деле затронута, выйдите на всю веб-страницу, а не на ее модуль.

   :::column-end:::
   :::column span="":::
:::image type="content" source="../../assets/images/task-module/simplicity-dont.png" alt-text="Пример, в котором показана рекомендуемая практика для модуля задачи." border="false":::

#### <a name="dont-do-long-interactions"></a>Не: выполнение длительных действий

Постарайтесь оставить свои взаимодействия короче и в точке.
